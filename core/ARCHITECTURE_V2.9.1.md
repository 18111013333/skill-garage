# 终极精简架构 V2.9.1

## 设计原则

1. **六层固定** - L1-L6 核心架构不变
2. **目录极简** - 从36个目录精简到6个
3. **文件聚合** - 同类文件合并，减少碎片
4. **按需加载** - 启动只加载L1，其他按需

---

## 六层架构 (固定不变)

```
┌─────────────────────────────────────────────────────────┐
│  L1: 核心认知层 - 身份、规则、引导 (立即加载)            │
├─────────────────────────────────────────────────────────┤
│  L2: 记忆上下文层 - 记忆存储、检索 (按需加载)            │
├─────────────────────────────────────────────────────────┤
│  L3: 任务编排层 - 路由、工作流、调度 (按需加载)          │
├─────────────────────────────────────────────────────────┤
│  L4: 能力执行层 - 技能执行、结果验证 (按类别加载)        │
├─────────────────────────────────────────────────────────┤
│  L5: 稳定治理层 - 安全、权限、审计 (敏感操作加载)        │
├─────────────────────────────────────────────────────────┤
│  L6: 基础设施层 - 路径、插件、性能 (系统操作加载)        │
└─────────────────────────────────────────────────────────┘
```
