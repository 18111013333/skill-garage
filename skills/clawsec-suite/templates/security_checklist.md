# 安全检查清单

## 一、代码安全

### 输入验证
- [ ] 所有用户输入都经过验证
- [ ] 使用白名单验证而非黑名单
- [ ] 验证数据类型、长度、格式、范围
- [ ] 防止SQL注入（使用参数化查询）
- [ ] 防止XSS攻击（转义输出）
- [ ] 防止命令注入
- [ ] 防止路径遍历

### 认证授权
- [ ] 使用强密码策略
- [ ] 密码加密存储（bcrypt/argon2）
- [ ] 实现账户锁定机制
- [ ] 使用多因素认证
- [ ] Session安全配置
- [ ] Token安全存储
- [ ] 实现权限最小化原则

### 敏感数据
- [ ] 敏感数据加密存储
- [ ] 传输使用HTTPS
- [ ] 不在日志中记录敏感信息
- [ ] 不在URL中传递敏感信息
- [ ] 不在错误信息中暴露敏感信息
- [ ] 定期轮换密钥

## 二、API安全

### 认证
- [ ] 使用标准认证协议（OAuth2/JWT）
- [ ] Token有过期时间
- [ ] 实现Token刷新机制
- [ ] 支持Token撤销

### 授权
- [ ] 实现RBAC/ABAC
- [ ] 每个接口都有权限检查
- [ ] 资源归属验证
- [ ] 防止越权访问

### 限流
- [ ] 实现API限流
- [ ] 防止暴力破解
- [ ] 防止DDoS攻击
- [ ] 配置合理的限流阈值

### 输入输出
- [ ] 验证请求参数
- [ ] 验证请求体大小
- [ ] 过滤响应中的敏感字段
- [ ] 统一错误响应格式

## 三、基础设施安全

### 网络安全
- [ ] 使用防火墙
- [ ] 只开放必要端口
- [ ] 内网隔离
- [ ] VPN访问内网
- [ ] DDoS防护

### 服务器安全
- [ ] 及时更新系统补丁
- [ ] 禁用不必要的服务
- [ ] 使用非root用户运行服务
- [ ] 配置SSH安全（禁用密码登录）
- [ ] 安装入侵检测系统

### 容器安全
- [ ] 使用可信基础镜像
- [ ] 定期扫描镜像漏洞
- [ ] 不以root运行容器
- [ ] 限制容器资源
- [ ] 使用只读文件系统

## 四、数据库安全

### 访问控制
- [ ] 使用最小权限账户
- [ ] 禁用远程root访问
- [ ] 限制IP访问
- [ ] 使用SSL连接

### 数据保护
- [ ] 敏感数据加密
- [ ] 定期备份
- [ ] 备份加密存储
- [ ] 测试恢复流程

### 审计
- [ ] 开启审计日志
- [ ] 记录所有DDL操作
- [ ] 记录敏感数据访问
- [ ] 定期审查日志

## 五、应用安全

### 依赖管理
- [ ] 定期更新依赖
- [ ] 扫描依赖漏洞
- [ ] 锁定依赖版本
- [ ] 移除未使用的依赖

### 配置安全
- [ ] 不在代码中硬编码密钥
- [ ] 使用环境变量
- [ ] 配置文件不提交到版本控制
- [ ] 生产配置与开发配置分离

### 错误处理
- [ ] 不暴露堆栈信息
- [ ] 统一错误处理
- [ ] 记录错误日志
- [ ] 友好的错误提示

## 六、安全测试

### 自动化测试
- [ ] SAST（静态分析）
- [ ] DAST（动态分析）
- [ ] 依赖扫描
- [ ] 容器扫描

### 渗透测试
- [ ] 定期进行渗透测试
- [ ] 修复发现的问题
- [ ] 验证修复效果

### 安全审计
- [ ] 代码审计
- [ ] 配置审计
- [ ] 权限审计
- [ ] 日志审计

## 七、应急响应

### 准备
- [ ] 制定安全事件响应计划
- [ ] 建立应急响应团队
- [ ] 准备联系方式清单
- [ ] 定期演练

### 响应流程
1. **发现**：监控告警、用户报告
2. **确认**：验证安全事件
3. **遏制**：隔离受影响系统
4. **根除**：修复漏洞
5. **恢复**：恢复正常服务
6. **总结**：复盘改进
