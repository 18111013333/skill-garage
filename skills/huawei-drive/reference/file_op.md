# 文件操作

描述如何操作云盘文件，目前支持上传、列举文件。

## 上传文件

上传文件到云盘。

### 执行步骤

1. 根据用户输入的指定文件或从上文中找到用户最后一次生成的文件，得到文件路径为`{file-path}`
2. 计算需上传的文件大小 得到`{file_size}`
3. 查看云盘剩余可用空间 得到`{available_space}`
4. 若云盘剩余可用空间`{available_space}`小于需上传的文件大小`{file_size}`，则提醒用户购买云空间套餐，反之继续后续步骤
5. 查询云盘文件夹 `小艺claw` 是否存在，若不存在则创建 `小艺claw` 文件夹
6. 查询云盘文件是否已存在，若存在则询问用户已什么方式上传。可选上传方式：覆盖/重命名
7. 调用 `huawei_drive.py` 并指定文件路径`{file-path}`及上传方式，完成文件上传
8. 如果用户有多个文件待上传，循环执行上面的操作过程逐个完成上传，最后输出一个汇总报告
9. 若上传失败，则根据错误日志，告诉用户失败原因。若上传失败是因为AT失效或者鉴权失败，则提示用户退出并重新进入小艺Claw后重试操作。

> 参考：文件上传成功后会保存到远程 `/root/小艺claw` 目录，不支持子文件夹。

### 输出格式
#### 单个文件上传
上传成功输出格式
```
文件上传成功！
云盘路径：小艺claw/test.txt
文件大小：2.45MB
```
上传失败按”常见错误“的要求进行提示。

#### 多个文件上传
上传完成后，汇总报告输出格式
```
文件上传完成！
| 云盘路径 | 文件大小 | 状态 | 
|-----|-----|-----|
|小艺claw/test1.txt|2.45MB| ✅ 成功 |
|小艺claw/test2.txt|4.0KB| ✅ 成功 | 
|小艺claw/test3.txt|4.0KB| ❌ 失败 | 
```
上传失败的文件按”常见错误“的要求进行提示。

### CLI 命令示例

```bash
# 上传文件到云盘
python huawei_drive.py --command upload --mode <upload_mode> --path <file_path>
```

huawei_drive.py脚本命令行能力说明，参考[huawei_drive_commands.md](./huawei_drive_commands.md)
