# 手机操作错误处理完整方案

## 概述

本技能整合了以下5个技能的内容：
1. xiaoyi-gui-agent-troubleshoot（故障排查）
2. xiaoyi-gui-agent-fallback（自动降级）
3. xiaoyi-gui-agent-error-handling（错误处理）
4. xiaoyi-gui-agent-timeout-solution（超时解决）
5. xiaoyi-gui-agent-captcha-ip-solution（验证码和IP限制）

---

## 第一部分：错误类型分类

### 1. 超时问题

**占比**：40%

**原因分析**：
| 原因 | 占比 | 检测方法 |
|-----|------|---------|
| 手机锁屏/休眠 | 60% | 超时后询问用户 |
| 网络连接不稳定 | 20% | 尝试网页版验证 |
| 服务未开启 | 15% | 超时后询问用户 |
| APP响应慢 | 5% | 重试验证 |

**解决方案**：
| 超时类型 | 解决方案 | 成功率 |
|---------|---------|-------|
| 网络超时 | 重试3次，然后切换 | 80% |
| 操作超时 | 简化任务，拆分步骤 | 70% |
| 加载超时 | 切换到网页版 | 90% |

**用户引导**：
```
⚠️ 检测到手机可能锁屏或休眠

请按以下步骤操作：
1. 解锁手机
2. 保持屏幕常亮（不要让手机休眠）
3. 回复"已解锁"，我将继续执行任务
```

## 第三部分：综合错误处理流程

```
执行任务
    ↓
检测错误类型
    ↓
├─ 超时 → 引导用户解锁/检查网络/检查服务
├─ 验证码 → 切换网页版/其他网站
├─ IP限制 → 切换国内网站
├─ 登录问题 → 提醒用户登录
├─ APP问题 → 切换网页版
└─ 网络问题 → 提醒用户检查
    ↓
自动尝试替代方案
    ↓
├─ 网页版 → 尝试执行
├─ 替代APP → 尝试执行
├─ 替代网站 → 尝试执行
└─ 全部失败 → 告知用户
```

## 第四部分：预防措施

### 任务开始前检查

```
1. 发送提示："即将操作手机，请确保手机已解锁并保持屏幕常亮"
2. 等待用户确认
3. 开始执行任务
```

### 任务执行中监控

```
1. 每30秒发送心跳检测
2. 如果心跳失败，立即通知用户
3. 等待恢复后继续
```

### 超时时间设置

| 操作类型 | 超时时间 |
|---------|---------|
| 手机操作 | 3分钟 |
| 网页操作 | 30秒 |
| 图片搜索 | 10秒 |
