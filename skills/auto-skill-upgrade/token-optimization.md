## 🚀 Token 优化策略

### 1. 懒加载机制 (预计减少 60%)

**当前问题**: 会话启动时加载所有技能元数据

**优化方案**:
```
启动时只加载:
- P0 核心技能 (20个, ~80K tokens)
- 技能索引文件 (1KB)

按需加载:
- P1-P3 技能仅在触发时加载
- 预计减少: ~400K tokens/会话
```

**实施**:
```bash
# 快速加载清单
quick-load.txt: 只包含 P0 技能

# 技能索引
skill-index.json: 所有技能的元数据摘要
```

### 2. SKILL.md 精简 (预计减少 50%)

**当前问题**: SKILL.md 包含过多示例和详细说明

**优化方案**:
```
SKILL.md 结构:
- Frontmatter (name, description) - 必须
- 快速开始 (3-5行) - 必须
- 核心指令 (10-20行) - 必须
- 详细文档 → references/

目标: 每个 SKILL.md < 2KB
```

**已优化**: 43 个大文件技能已精简

### 3. 技能链合并 (预计减少 30%)

**当前问题**: 多个技能独立加载，重复内容

**优化方案**:
```
工作流链合并:
- 文档链: 5技能 → 1个入口
- 图像链: 4技能 → 1个入口
- 搜索链: 4技能 → 1个入口

预计减少: ~100K tokens
```

### 4. 记忆压缩 (预计减少 40%)

**当前问题**: MEMORY.md 和会话历史累积

**优化方案**:
```
MEMORY.md:
- 只保留关键决策和偏好
- 详细日志 → memory/YYYY-MM-DD.md
- 定期归档旧记录

会话历史:
- 自动摘要超过 50 轮的对话
- 保留最近 20 轮完整上下文
```

### 5. 工具定义优化 (预计减少 20%)

**当前问题**: 所有工具定义始终加载

**优化方案**:
```
分层加载:
- P0 工具: 始终可用
- P1-P3 工具: 按需注册

工具描述精简:
- 移除冗余示例
- 压缩参数说明
```
