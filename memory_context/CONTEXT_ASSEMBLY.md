# CONTEXT_ASSEMBLY.md - 上下文组装流程

## 目的
定义上下文组装流程，确保能复盘"这轮为什么拿了这些上下文"。

## 适用范围
所有上下文构造过程。

## 组装流程

```
┌─────────────────────────────────────────────────────────┐
│ 1. 收集 (Collect)                                       │
│    - 收集所有可用上下文来源                              │
│    - 标记每个来源的类型和优先级                          │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 2. 去重 (Deduplicate)                                   │
│    - 检测重复内容                                        │
│    - 保留最高优先级版本                                  │
│    - 记录去重日志                                        │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 3. 评分 (Score)                                         │
│    - 计算每个片段的相关性分数                            │
│    - 计算时效性分数                                      │
│    - 计算可信度分数                                      │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 4. 排序 (Rank)                                          │
│    - 按综合分数排序                                      │
│    - 确保高优先级内容在前                                │
│    - 处理同分情况                                        │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 5. 截断 (Truncate)                                      │
│    - 按Token预算截断                                     │
│    - 确保关键内容不被截断                                │
│    - 记录截断日志                                        │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 6. 注入 (Inject)                                        │
│    - 按格式注入上下文                                    │
│    - 标记每个片段的来源                                  │
│    - 生成组装报告                                        │
└─────────────────────────────────────────────────────────┘
```

## 内容合并规则

### 系统规则合并
| 规则类型 | 合并方式 | 冲突处理 |
|----------|----------|----------|
| 安全规则 | 必须包含 | 安全优先 |
| 执行策略 | 按需包含 | 新覆盖旧 |
| 格式要求 | 合并 | 用户优先 |

### 用户要求合并
| 要求类型 | 合并方式 | 冲突处理 |
|----------|----------|----------|
| 硬约束 | 必须包含 | 用户优先 |
| 软偏好 | 按需包含 | 新覆盖旧 |
| 格式要求 | 合并 | 用户优先 |

### 记忆合并
| 记忆类型 | 合并方式 | 冲突处理 |
|----------|----------|----------|
| 会话记忆 | 按相关性 | 新覆盖旧 |
| 长期记忆 | 按需检索 | 标记冲突 |
| 场景记忆 | 按场景 | 场景优先 |

### 检索结果合并
| 结果类型 | 合并方式 | 冲突处理 |
|----------|----------|----------|
| 网页搜索 | 按相关性 | 标记来源 |
| 文件搜索 | 按相关性 | 标记来源 |
| 记忆搜索 | 按相关性 | 标记来源 |

### 工具返回合并
| 返回类型 | 合并方式 | 冲突处理 |
|----------|----------|----------|
| 执行结果 | 直接注入 | 无冲突 |
| 错误信息 | 标记后注入 | 无冲突 |
| 状态更新 | 更新上下文 | 新覆盖旧 |

## 冲突标记规则

### 显式标记格式
```
[CONFLICT: {冲突类型}]
├── Source: {来源A} (confidence: {可信度})
│   └── Content: {内容A}
├── Source: {来源B} (confidence: {可信度})
│   └── Content: {内容B}
└── Resolution: {处理方式}
```

### 禁止行为
- ❌ 禁止静默覆盖冲突信息
- ❌ 禁止忽略用户硬约束
- ❌ 禁止隐式修改系统规则

## 异常处理

| 异常 | 处理 |
|------|------|
| 收集失败 | 使用缓存上下文 |
| 去重失败 | 保留所有版本 |
| 评分失败 | 使用默认分数 |
| 截断关键信息 | 标记警告 |

## 维护方式
- 调整合并规则: 更新合并规则表
- 新增来源类型: 添加到合并规则
- 调整冲突处理: 更新冲突标记规则
