# GOAL_INFERENCE.md - 目标推断规则

## 目的
定义从用户输入推断真实目标的规则，确保理解用户真正想要什么。

## 适用范围
所有用户输入的目标识别和推断。

## 目标推断层次

### 显式目标
| 特征 | 说明 | 示例 |
|------|------|------|
| 直接陈述 | 用户直接说明目标 | "帮我写一篇文章" |
| 明确指令 | 用户给出明确指令 | "搜索XX信息" |
| 清晰需求 | 用户需求清晰 | "创建一个日程" |

### 隐式目标
| 特征 | 说明 | 示例 |
|------|------|------|
| 模糊表达 | 用户表达模糊 | "有点麻烦" → 需要帮助 |
| 问题暗示 | 通过问题暗示目标 | "这个怎么做？" → 需要指导 |
| 情绪表达 | 通过情绪暗示目标 | "太烦了" → 需要简化 |

### 深层目标
| 特征 | 说明 | 示例 |
|------|------|------|
| 根本需求 | 用户的根本需求 | "提高效率" → 自动化 |
| 长期目标 | 用户的长期目标 | "学习AI" → 系统学习 |
| 价值追求 | 用户的价值追求 | "更安全" → 风险控制 |

## 推断流程

```
用户输入
    ↓
┌─────────────────────────────────────┐
│ 1. 提取信号                          │
│    - 关键词提取                      │
│    - 意图信号提取                    │
│    - 情绪信号提取                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 匹配规则                          │
│    - 关键词规则匹配                  │
│    - 上下文规则匹配                  │
│    - 意图规则匹配                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 计算置信度                        │
│    - 规则置信度                      │
│    - 上下文加成                      │
│    - 历史加成                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 生成目标模型                      │
│    - 主要目标                        │
│    - 子目标                          │
│    - 约束条件                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 确认或执行                        │
│    - 高置信度: 直接执行              │
│    - 中置信度: 确认后执行            │
│    - 低置信度: 询问确认              │
└─────────────────────────────────────┘
```

## 置信度计算

### 计算公式
```
confidence = (keyword_score × 0.4) 
           + (context_score × 0.3) 
           + (history_score × 0.2) 
           + (explicit_score × 0.1)
```

### 置信度级别
| 级别 | 范围 | 处理方式 |
|------|------|----------|
| 高 | ≥ 0.8 | 直接执行 |
| 中 | 0.6-0.8 | 确认后执行 |
| 低 | < 0.6 | 询问确认 |

## 目标确认

### 确认模板
| 置信度 | 确认模板 |
|--------|----------|
| 高 | 无需确认 |
| 中 | "我理解您想要XX，对吗？" |
| 低 | "您是想要XX还是YY？" |

### 确认示例
```
中置信度:
用户: "这个文件改一下"
系统: "我理解您想修改这个文件，是要修改内容还是格式？"

低置信度:
用户: "帮我看看"
系统: "您是想让我查看什么内容？是文件、日志还是其他？"
```

## 目标修正

### 修正触发
| 触发条件 | 说明 |
|----------|------|
| 用户纠正 | 用户明确纠正理解 |
| 执行失败 | 执行结果不符合预期 |
| 新信息 | 获得新的上下文信息 |
| 用户反馈 | 用户表达不满意 |

### 修正流程
```
检测到修正信号
    ↓
重新分析用户输入
    ↓
更新目标模型
    ↓
确认新目标
    ↓
继续执行
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 无法推断 | 询问用户 |
| 多目标冲突 | 询问优先级 |
| 置信度过低 | 询问确认 |

## 维护方式
- 新增推断规则: 添加到规则表
- 调整置信度权重: 修改计算公式
- 新增信号类型: 更新信号提取
