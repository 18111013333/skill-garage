# ANSWER_VALIDATION.md - 答案验证规则

## 目的
定义答案验证的具体规则和流程，确保答案质量可检验。

## 适用范围
所有生成答案的验证。

## 验证维度

### 事实验证
| 验证项 | 检查方式 | 通过条件 |
|--------|----------|----------|
| 事实准确性 | 与来源比对 | 100%匹配 |
| 数据准确性 | 数值验证 | 误差<1% |
| 时间准确性 | 时间戳验证 | 在有效期内 |
| 名称准确性 | 实体验证 | 正确无误 |

### 逻辑验证
| 验证项 | 检查方式 | 通过条件 |
|--------|----------|----------|
| 推理正确 | 逻辑检查 | 无逻辑错误 |
| 因果合理 | 因果分析 | 因果清晰 |
| 结论合理 | 结论验证 | 结论可推导 |

### 完整性验证
| 验证项 | 检查方式 | 通过条件 |
|--------|----------|----------|
| 问题回应 | 问题匹配 | 回应所有问题点 |
| 要点覆盖 | 要点检查 | 覆盖率≥90% |
| 信息完整 | 完整度检查 | 完整度≥80% |

### 安全性验证
| 验证项 | 检查方式 | 通过条件 |
|--------|----------|----------|
| PII保护 | 敏感信息检测 | 无泄露 |
| 有害内容 | 内容审核 | 无有害内容 |
| 注入防护 | 注入检测 | 无注入攻击 |

### 格式验证
| 验证项 | 检查方式 | 通过条件 |
|--------|----------|----------|
| 结构完整 | 结构检查 | 结构正确 |
| 格式规范 | 格式检查 | 符合要求 |
| 编码正确 | 编码检查 | UTF-8正确 |

## 验证流程

### 事实验证流程
```
事实陈述
    ↓
┌─────────────────────────────────────┐
│ 1. 提取事实                          │
│    - 识别事实陈述                    │
│    - 提取关键信息                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 查找来源                          │
│    - 匹配引用来源                    │
│    - 查找验证来源                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 比对验证                          │
│    - 内容比对                        │
│    - 数值比对                        │
│    - 时间比对                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 输出结果                          │
│    - 验证通过/失败                   │
│    - 差异说明                        │
└─────────────────────────────────────┘
```

### 逻辑验证流程
```
推理过程
    ↓
┌─────────────────────────────────────┐
│ 1. 提取推理                          │
│    - 识别推理链                      │
│    - 提取前提和结论                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 检查前提                          │
│    - 前提是否成立                    │
│    - 前提是否充分                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 检查推理                          │
│    - 推理是否合理                    │
│    - 是否有跳跃                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 检查结论                          │
│    - 结论是否可推导                  │
│    - 结论是否过度                    │
└─────────────────────────────────────┘
```

## 验证失败处理

### 失败级别
| 级别 | 说明 | 处理 |
|------|------|------|
| error | 严重错误 | 必须修正 |
| warning | 警告 | 建议修正 |
| info | 提示 | 可选修正 |

### 失败处理
| 失败类型 | 处理方式 |
|----------|----------|
| 事实错误 | 重新生成或修正 |
| 逻辑错误 | 重新生成 |
| 不完整 | 补充内容 |
| 安全问题 | 拒绝或脱敏 |
| 格式错误 | 修正格式 |

## 验证统计

### 统计指标
| 指标 | 说明 |
|------|------|
| 验证通过率 | 通过验证的答案比例 |
| 平均验证分数 | 平均验证得分 |
| 问题分布 | 各类问题的分布 |
| 修正率 | 需要修正的比例 |

### 统计周期
| 周期 | 统计内容 |
|------|----------|
| 实时 | 单次验证结果 |
| 每小时 | 通过率、平均分 |
| 每天 | 问题分布、修正率 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 验证超时 | 使用简化验证 |
| 来源缺失 | 标记无法验证 |
| 验证失败 | 重新生成 |

## 维护方式
- 新增验证规则: 添加到验证规则表
- 调整验证级别: 修改失败级别定义
- 新增验证维度: 更新验证流程
