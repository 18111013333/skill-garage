## 三层画像模型

### L1 - 会话层 (Session Layer)
- **数据源**: 当前对话实时流
- **提取内容**: 意图、情绪、临时偏好、即时需求
- **存储**: 会话上下文 (短期记忆)
- **生命周期**: 单次会话

### L2 - 场景层 (Scenario Layer)
- **数据源**: L1 聚合 + 场景识别
- **提取内容**: 场景偏好、任务模式、交互习惯
- **存储**: `memory/scenarios/{scenario_type}.md`
- **生命周期**: 场景周期 (如工作项目周期)

### L3 - 长期层 (Long-term Layer)
- **数据源**: L2 沉淀 + 显式声明
- **提取内容**: 核心身份、长期偏好、价值观、能力边界
- **存储**: `USER.md` + `MEMORY.md` + `ontology/`
- **生命周期**: 持久

## 提取机制

### 实时提取触发器

| 触发条件 | 提取动作 | 目标层 |
|---------|---------|--------|
| 用户表达偏好 | 记录偏好 | L1 → L2 |
| 用户纠正行为 | 更新偏好 | L2 → L3 |
| 场景切换 | 聚合场景画像 | L2 |
| 显式声明 | 直接写入 | L3 |
| 周期性回顾 | 沉淀长期画像 | L2 → L3 |

```yaml
用户画像维度:
  核心身份:
    - 姓名/称呼

    - 技术栈偏好
    - 工具使用习惯
    - 专业领域

    - 输出格式偏好
    - 沟通基调
    - 反馈风格

    - 健康限制

  场景画像:
    - 工作场景偏好
    - 学习场景偏好
    - 生活场景偏好
    - 创作场景偏好
```

## 技术实现

### 1. 场景识别
使用 LLM 分析对话内容，识别当前场景类型：
- 工作场景 (coding, writing, research, meeting)
- 学习场景 (learning, exploring, practicing)
- 生活场景 (planning, shopping, health, entertainment)
- 创作场景 (designing, brainstorming, creating)

### 2. 信息提取
结合 ontology 技能，结构化存储提取的信息：
- Person 实体：用户基本信息
- Preference 实体：偏好设置
- Scenario 实体：场景画像
- Action 实体：行为记录

### 3. 画像更新
使用 self-improving-agent 机制：
- 检测用户纠正 → 触发画像更新
- 检测偏好变化 → 更新偏好权重
- 检测新场景 → 创建场景画像

## 与现有技能集成

| 技能 | 集成方式 |
|------|---------|
| self-improving-agent | 复用学习机制，画像更新触发学习记录 |

| best-minds | 根据用户画像选择专家视角 |

## 自动化流程

### 会话结束时
1. 提取本次会话关键信息
2. 更新 L1 会话画像
3. 聚合到 L2 场景画像
4. 检查是否需要更新 L3 长期画像

### 心跳触发时
1. 回顾近期会话
2. 识别画像变化趋势
3. 沉淀稳定的偏好到 L3
4. 清理过时的临时画像

### 用户显式更新时
1. 直接更新 L3 长期画像
2. 同步更新 ontology 实体
3. 记录更新原因和时间
