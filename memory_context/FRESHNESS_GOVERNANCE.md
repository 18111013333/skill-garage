# FRESHNESS_GOVERNANCE.md - 知识时效治理

**版本: V27.0**

## 目的
定义知识时效治理规则，确保旧知识不会长期冒充最新知识，实现"知识有时效、过期有标记、降权有规则"。

## 适用范围
所有知识时效管理场景，包括：
- 稳定知识管理
- 周期更新知识管理
- 高频变化知识管理
- 过期知识处理

---

## 一、知识时效分类

| 分类 | 代码 | 说明 | 典型示例 |
|------|------|------|----------|
| 稳定知识 | STABLE | 长期有效，极少变化 | 基础概念、原理 |
| 周期更新 | PERIODIC | 定期更新，有明确周期 | 政策法规、标准规范 |
| 高频变化 | VOLATILE | 频繁变化，时效性强 | 价格、库存、状态 |
| 实时知识 | REALTIME | 实时数据，秒级变化 | 行情、流量、状态 |

---

## 二、稳定知识管理

### 2.1 定义标准

| 标准 | 说明 |
|------|------|
| 变化频率 | < 1次/年 |
| 有效期 | > 5年 |
| 来源稳定性 | 来源稳定可靠 |
| 验证方式 | 可独立验证 |

### 2.2 管理规则

| 规则 | 说明 |
|------|------|
| 复查周期 | 每年复查一次 |
| 更新触发 | 来源变更或发现错误 |
| 过期处理 | 标记待验证，不删除 |
| 降权规则 | 不降权 |

### 2.3 示例

| 知识类型 | 时效分类 | 复查周期 |
|----------|----------|----------|
| 数学公式 | 稳定知识 | 5年 |
| 物理定律 | 稳定知识 | 5年 |
| 编程语法 | 稳定知识 | 2年 |
| 设计模式 | 稳定知识 | 3年 |

## 三、周期更新知识管理

### 3.1 定义标准

| 标准 | 说明 |
|------|------|
| 变化频率 | 1-12次/年 |
| 有效期 | 有明确有效期 |
| 更新周期 | 有明确更新周期 |
| 来源稳定性 | 来源定期发布 |

### 3.2 管理规则

| 规则 | 说明 |
|------|------|
| 复查周期 | 按更新周期复查 |
| 更新触发 | 来源发布新版本 |
| 过期处理 | 标记过期，提示新版本 |
| 降权规则 | 过期后降权 |

### 3.3 更新周期配置

| 知识类型 | 更新周期 | 过期阈值 |
|----------|----------|----------|
| 法律法规 | 按发布 | 发布新版本即过期 |
| 行业标准 | 按发布 | 发布新版本即过期 |
| 公司政策 | 季度 | 超过更新周期2倍过期 |
| 技术规范 | 半年 | 超过更新周期2倍过期 |
| 产品文档 | 月度 | 超过更新周期3倍过期 |

## 四、高频变化知识管理

### 4.1 定义标准

| 标准 | 说明 |
|------|------|
| 变化频率 | > 12次/年 |
| 有效期 | 短期有效 |
| 更新方式 | 自动或半自动更新 |
| 来源类型 | 动态数据源 |

### 4.2 管理规则

| 规则 | 说明 |
|------|------|
| 复查周期 | 按需或自动 |
| 更新触发 | 数据源变化 |
| 过期处理 | 自动标记或删除 |
| 降权规则 | 时效性衰减 |

### 4.3 时效性衰减

```
时效性衰减公式:
freshness_score = base_score * exp(-decay_rate * age)

参数:
- base_score: 基础分数 (0-1)
- decay_rate: 衰减率 (按知识类型)
- age: 知识年龄 (天)

示例:
- 价格信息: decay_rate = 0.1, 7天后分数 < 0.5
- 库存信息: decay_rate = 0.05, 14天后分数 < 0.5
- 状态信息: decay_rate = 0.2, 3天后分数 < 0.5
```

## 五、刷新周期

### 5.1 刷新周期定义

| 知识分类 | 刷新周期 | 刷新方式 |
|----------|----------|----------|
| 稳定知识 | 1年 | 人工复查 |
| 周期更新 | 按更新周期 | 自动 + 人工确认 |
| 高频变化 | 按配置 | 自动刷新 |
| 实时知识 | 实时 | 实时同步 |

### 5.2 刷新流程

```
刷新触发
    ↓
┌─────────────────────────────────────┐
│ 1. 来源检查                          │
│    - 检查来源是否更新                │
│    - 获取最新版本                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 变更检测                          │
│    - 对比新旧版本                    │
│    - 识别变更内容                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 更新执行                          │
│    - 更新知识内容                    │
│    - 更新时效标记                    │
│    - 重建索引                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 通知与审计                        │
│    - 通知相关方                      │
│    - 记录审计日志                    │
└─────────────────────────────────────┘
```

## 六、过期标记

### 6.1 过期判定

| 判定条件 | 说明 |
|----------|------|
| 时间过期 | 超过有效期 |
| 版本过期 | 有新版本发布 |
| 来源失效 | 来源不可用 |
| 内容失效 | 内容不再适用 |

### 6.2 过期标记类型

| 标记类型 | 代码 | 说明 | 检索影响 |
|----------|------|------|----------|
| 有效 | VALID | 知识有效 | 正常返回 |
| 即将过期 | EXPIRING | 即将过期 | 正常返回 + 提示 |
| 已过期 | EXPIRED | 已过期 | 降权返回 + 警告 |
| 已废弃 | DEPRECATED | 已废弃 | 不返回 + 可查询 |
| 已删除 | DELETED | 已删除 | 不返回 |

### 6.3 过期标记流程

```
过期检测
    ↓
┌─────────────────────────────────────┐
│ 1. 自动检测                          │
│    - 定时扫描知识库                  │
│    - 识别过期知识                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 标记更新                          │
│    - 更新过期标记                    │
│    - 记录过期原因                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 通知处理                          │
│    - 通知知识 Owner                  │
│    - 生成过期报告                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 后续处理                          │
│    - 等待更新或                      │
│    - 进入废弃流程                    │
└─────────────────────────────────────┘
```

## 七、降权规则

### 7.1 降权因素

| 因素 | 权重 | 说明 |
|------|------|------|
| 时效性 | 40% | 知识新鲜度 |
| 准确性 | 30% | 知识准确性 |
| 完整性 | 20% | 知识完整性 |
| 来源可信度 | 10% | 来源可靠性 |

### 7.2 降权计算

```
综合分数 = 时效分 * 0.4 + 准确分 * 0.3 + 完整分 * 0.2 + 来源分 * 0.1

时效分计算:
- 有效: 1.0
- 即将过期: 0.8
- 已过期: 0.3
- 已废弃: 0.1
```

### 7.3 降权影响

| 综合分数 | 检索排序 | 返回方式 |
|----------|----------|----------|
| > 0.8 | 正常排序 | 正常返回 |
| 0.5-0.8 | 降权排序 | 正常返回 + 提示 |
| 0.3-0.5 | 大幅降权 | 警告返回 |
| < 0.3 | 不参与排序 | 仅按需返回 |
