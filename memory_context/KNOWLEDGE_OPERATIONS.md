# KNOWLEDGE_OPERATIONS.md - 知识运营机制

**版本: V27.0**

## 目的
定义知识运营机制，确保知识不再只是存储和检索，而是被持续运营、优化、更新，保持知识库的高质量和时效性。

## 适用范围
所有知识库管理、知识更新、知识质量监控、知识生命周期管理场景。

---

## 一、知识运营原则

| 原则 | 说明 |
|------|------|
| 主动运营 | 知识需要主动维护，而非被动积累 |
| 质量优先 | 知识质量比数量更重要 |
| 时效管理 | 知识有时效性，需定期更新或下架 |
| 价值驱动 | 高价值知识优先维护 |
| 可追溯 | 知识来源和变更可追溯 |

---

## 三、热点问题补库

### 3.1 热点识别

| 指标 | 阈值 | 说明 |
|------|------|------|
| 查询频率 | > 100次/周 | 高频查询 |
| 零命中次数 | > 10次/周 | 无结果查询 |
| 低满意度 | < 60% | 用户反馈差 |
| 重复提问 | > 5次 | 同类问题 |

### 3.2 补库流程

```
热点识别
    ↓
┌─────────────────────────────────────┐
│ 1. 问题分析                          │
│    - 问题类型分类                    │
│    - 问题意图识别                    │
│    - 关联问题聚合                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 答案准备                          │
│    - 自动草拟                        │
│    - 专家审核                        │
│    - 来源标注                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 知识入库                          │
│    - 质量检查                        │
│    - 分类索引                        │
│    - 发布上线                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 效果验证                          │
│    - 命中率提升验证                  │
│    - 满意度提升验证                  │
│    - 持续监控                        │
└─────────────────────────────────────┘
```

### 3.3 补库优先级

| 优先级 | 条件 | 处理时限 |
|--------|------|----------|
| P0 | 业务关键 + 零命中 | 24 小时 |
| P1 | 高频 + 低满意度 | 3 天 |
| P2 | 中频 + 零命中 | 7 天 |
| P3 | 低频 + 有改进空间 | 14 天 |

## 四、低命中高价值知识修补

### 4.1 识别标准

| 条件 | 说明 |
|------|------|
| 命中率低 | < 5% 查询命中 |
| 价值高 | 业务关键或专家知识 |
| 时效性好 | 内容仍然有效 |

### 4.2 修补策略

| 策略 | 适用场景 | 方法 |
|------|----------|------|
| 关键词优化 | 检索词不匹配 | 添加同义词、扩展关键词 |
| 内容重构 | 表达不清晰 | 重写内容、优化结构 |
| 标签调整 | 分类不准确 | 调整分类、添加标签 |
| 向量优化 | 语义匹配差 | 优化向量化参数 |
| 关联增强 | 孤立知识 | 建立知识关联 |

### 4.3 修补流程

```
识别低命中高价值知识
    ↓
分析原因（检索词/内容/分类/向量）
    ↓
制定修补方案
    ↓
执行修补
    ↓
效果验证（命中率提升 > 20%）
    ↓
持续监控
```

## 五、失效知识下架

### 5.1 失效判定标准

| 标准 | 说明 | 处理方式 |
|------|------|----------|
| 内容过时 | 信息已不适用 | 标记过时 + 归档 |
| 政策变更 | 与新政策冲突 | 标记失效 + 替换 |
| 来源失效 | 来源不可用 | 标记待验证 |
| 低价值 | 长期无命中且价值低 | 归档或删除 |
| 负反馈 | 持续负反馈 | 下架审查 |

### 5.2 下架流程

```
失效检测
    ↓
┌─────────────────────────────────────┐
│ 1. 失效确认                          │
│    - 自动检测 + 人工确认             │
│    - 影响评估                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 替代准备（如有）                  │
│    - 准备新知识                      │
│    - 建立重定向                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 下架执行                          │
│    - 标记失效                        │
│    - 移出检索索引                    │
│    - 归档保存                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 通知与审计                        │
│    - 通知相关方                      │
│    - 记录审计日志                    │
└─────────────────────────────────────┘
```

## 六、术语统一

### 6.1 术语管理

| 管理项 | 说明 |
|--------|------|
| 术语库 | 维护标准术语库 |
| 同义词映射 | 建立同义词关系 |
| 缩写扩展 | 定义缩写全称映射 |
| 术语变更 | 管理术语变更流程 |

### 6.2 统一规则

| 规则 | 说明 |
|------|------|
| 新知识入库 | 必须使用标准术语 |
| 旧知识更新 | 逐步统一术语 |
| 检索扩展 | 自动扩展同义词 |
| 歧义处理 | 标注歧义术语 |

### 6.3 术语变更流程

```
术语变更申请
    ↓
影响评估（影响哪些知识）
    ↓
审批（知识管理员）
    ↓
批量更新（自动 + 人工审核）
    ↓
索引重建
    ↓
效果验证
```

## 七、Owner 指派

### 7.1 Owner 职责

| 职责 | 说明 |
|------|------|
| 质量保证 | 确保知识准确完整 |
| 时效维护 | 定期检查更新 |
| 反馈处理 | 处理用户反馈 |
| 关联管理 | 维护知识关联 |

### 7.2 指派规则

| 知识类型 | 指派规则 |
|----------|----------|
| 业务知识 | 业务领域负责人 |
| 技术知识 | 技术领域专家 |
| 通用知识 | 知识运营团队 |
| 外部知识 | 来源对接人 |

### 7.3 Owner 管理

| 管理项 | 说明 |
|--------|------|
| Owner 变更 | 记录变更历史 |
| 代理机制 | Owner 缺席时代理 |
| 绩效考核 | 知识质量纳入考核 |
| 培训支持 | 提供 Owner 培训 |

## 八、复查节奏

### 8.1 复查周期

| 知识类型 | 复查周期 | 复查内容 |
|----------|----------|----------|
| 高频变化 | 每周 | 时效性、准确性 |
| 中频变化 | 每月 | 完整性、关联性 |
| 稳定知识 | 每季度 | 全面复查 |
| 归档知识 | 每年 | 是否需要恢复 |

### 8.2 复查流程

```
复查计划生成
    ↓
自动检查（时效、命中率、反馈）
    ↓
生成复查清单
    ↓
Owner 复查
    ↓
复查结果处理（更新/下架/保持）
    ↓
复查报告生成
```

### 8.3 复查指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 复查完成率 | > 95% | 按计划完成复查 |
| 问题发现率 | > 5% | 复查发现问题比例 |
| 问题修复率 | > 90% | 发现问题修复比例 |
| 复查及时性 | > 90% | 按时完成复查 |

---

## 九、知识运营指标

### 9.1 质量指标

| 指标 | 计算方式 | 目标 |
|------|----------|------|
| 命中率 | 命中次数/查询次数 | > 80% |
| 满意度 | 正反馈/总反馈 | > 85% |
| 准确率 | 正确答案/总答案 | > 95% |
| 完整率 | 完整知识/总知识 | > 90% |

### 9.2 运营指标

| 指标 | 计算方式 | 目标 |
|------|----------|------|
| 补库及时性 | 按时完成/总补库 | > 90% |
| 复查完成率 | 完成复查/计划复查 | > 95% |
| 失效处理率 | 处理失效/发现失效 | > 95% |
| Owner 覆盖率 | 有 Owner 知识/总知识 | 100% |

---

## 十、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 热点补库超时 | 升级 + 临时答案 |
| 修补效果不佳 | 专家介入 + 方案调整 |
| 失效知识误判 | 恢复 + 原因分析 |
| Owner 缺席 | 代理 + 告警 |
| 复查超期 | 告警 + 自动标记 |

---

## 十一、完成标准

| 标准 | 验证方式 |
|------|----------|
| 热点补库 | 热点问题命中率提升 |
| 知识修补 | 低命中知识命中率提升 |
| 失效下架 | 无过时知识返回 |
| 术语统一 | 术语一致性检查通过 |
| Owner 覆盖 | 所有知识有 Owner |
| 复查执行 | 复查报告完整 |
