# ANSWER_POLICY.md - 答案生成策略

## 目的
定义答案生成的规则和约束，确保答案质量可控、风险可识别。

## 适用范围
所有返回给用户的答案生成。

## 答案类型

### 直接回答
| 条件 | 说明 | 示例 |
|------|------|------|
| 常识问题 | 通用知识 | "水的沸点是100°C" |
| 系统规则 | 已知规则 | "最大步骤数是30" |
| 用户指令 | 明确指令 | 按用户要求执行 |

### 检索后回答
| 条件 | 说明 | 示例 |
|------|------|------|
| 事实问题 | 需要验证 | "最新价格是..." |
| 时效问题 | 需要最新信息 | "今天天气..." |
| 专业问题 | 需要权威来源 | "法律规定..." |

### 拒绝回答
| 条件 | 说明 | 处理 |
|------|------|------|
| 超出能力 | 无法完成的任务 | 说明原因 |
| 安全风险 | 高风险操作 | 需确认 |
| 权限不足 | 无权限操作 | 说明限制 |
| 信息不足 | 缺乏必要信息 | 请求补充 |

### 延迟回答
| 条件 | 说明 | 处理 |
|------|------|------|
| 需要检索 | 需要外部信息 | 先检索再回答 |
| 需要确认 | 高风险操作 | 等待用户确认 |
| 需要规划 | 复杂任务 | 先规划再执行 |

## 答案结构

### 标准结构
```
1. 直接回答 (核心结论)
2. 详细说明 (可选)
3. 来源依据 (如需要)
4. 注意事项 (如需要)
5. 后续建议 (可选)
```

### 简洁结构
```
1. 直接回答
```

### 详细结构
```
1. 摘要
2. 详细说明
3. 数据/证据
4. 来源引用
5. 风险提示
6. 建议行动
```

## 答案质量要求

### 完整性
| 要求 | 说明 |
|------|------|
| 回应问题 | 必须回应用户的问题 |
| 覆盖要点 | 覆盖问题的所有要点 |
| 提供结论 | 给出明确结论 |

### 准确性
| 要求 | 说明 |
|------|------|
| 事实准确 | 事实陈述必须准确 |
| 数据准确 | 数据必须来自可靠来源 |
| 引用准确 | 引用必须与内容匹配 |

### 相关性
| 要求 | 说明 |
|------|------|
| 切题 | 回答必须与问题相关 |
| 无冗余 | 避免无关内容 |
| 聚焦 | 围绕核心问题 |

### 可读性
| 要求 | 说明 |
|------|------|
| 结构清晰 | 使用标题、列表、表格 |
| 语言流畅 | 表达清晰易懂 |
| 格式规范 | 符合输出格式要求 |

## 答案验证

### 验证项
| 验证项 | 检查内容 | 通过条件 |
|--------|----------|----------|
| 完整性 | 是否回应问题 | 100% |
| 准确性 | 事实是否准确 | ≥ 95% |
| 相关性 | 是否切题 | ≥ 90% |
| 安全性 | 是否有风险 | 无风险 |
| 格式 | 是否符合要求 | 100% |

### 验证流程
```
生成答案
    ↓
┌─────────────────────────────────────┐
│ 1. 完整性检查                        │
│    - 是否回应问题                    │
│    - 是否覆盖要点                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 准确性检查                        │
│    - 事实验证                        │
│    - 数据验证                        │
│    - 引用验证                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 安全性检查                        │
│    - PII检测                         │
│    - 风险检测                        │
│    - 权限检查                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 格式检查                          │
│    - 结构检查                        │
│    - 格式检查                        │
└─────────────────────────────────────┘
```

## 答案分级

### A级答案
- 完整性: 100%
- 准确性: ≥ 95%
- 相关性: ≥ 95%
- 有完整引用
- 无风险

### B级答案
- 完整性: ≥ 90%
- 准确性: ≥ 90%
- 相关性: ≥ 85%
- 有部分引用
- 低风险

### C级答案
- 完整性: ≥ 80%
- 准确性: ≥ 80%
- 相关性: ≥ 80%
- 有来源标记
- 中风险

### D级答案
- 完整性: < 80%
- 准确性: < 80%
- 相关性: < 80%
- 无引用
- 需改进

## 答案降级

### 降级触发
| 条件 | 降级动作 |
|------|----------|
| 验证失败 | 重新生成 |
| 信息不足 | 标注不确定性 |
| 风险过高 | 添加警告 |
| 格式错误 | 修正格式 |

### 降级处理
```
A级 → 验证失败 → 重新生成
B级 → 信息不足 → 标注"基于有限信息"
C级 → 风险过高 → 添加风险警告
D级 → 需改进 → 重新生成或拒绝
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 生成失败 | 返回错误信息 |
| 验证失败 | 重新生成 |
| 超时 | 返回部分结果 |
| 资源不足 | 降级回答 |

## 维护方式
- 调整质量要求: 修改质量要求表
- 新增答案类型: 添加到答案类型表
- 调整验证规则: 更新验证项
