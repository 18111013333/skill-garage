# RETRIEVAL_SCORING.md - 检索评分规则

## 目的
定义检索结果评分维度和排序逻辑，确保检索结果按规则选择。

## 适用范围
所有检索结果的评分和排序。

## 评分维度

### 相关性 (Relevance)
| 因素 | 权重 | 说明 |
|------|------|------|
| 关键词匹配 | 30% | 查询关键词出现频率 |
| 语义相似度 | 40% | 向量相似度分数 |
| 主题相关性 | 30% | 主题模型匹配度 |

### 时效性 (Freshness)
| 因素 | 权重 | 说明 |
|------|------|------|
| 发布时间 | 50% | 内容发布/更新时间 |
| 时效类型 | 30% | 内容时效性分类 |
| 过期风险 | 20% | 信息过期可能性 |

### 可信度 (Credibility)
| 因素 | 权重 | 说明 |
|------|------|------|
| 来源权威性 | 40% | 来源可信度评级 |
| 引用数量 | 30% | 被引用/参考次数 |
| 验证状态 | 30% | 是否经过验证 |

### 来源质量 (Source Quality)
| 因素 | 权重 | 说明 |
|------|------|------|
| 来源类型 | 40% | 官方/媒体/个人等 |
| 历史准确率 | 30% | 该来源历史准确率 |
| 更新频率 | 30% | 来源更新频率 |

### 信息完整度 (Completeness)
| 因素 | 权重 | 说明 |
|------|------|------|
| 信息覆盖 | 40% | 是否覆盖查询要点 |
| 细节程度 | 30% | 信息详细程度 |
| 结构完整 | 30% | 信息结构完整性 |

### 冲突程度 (Conflict)
| 因素 | 权重 | 说明 |
|------|------|------|
| 与已知冲突 | 50% | 与已有信息冲突程度 |
| 内部矛盾 | 30% | 信息内部矛盾程度 |
| 来源冲突 | 20% | 不同来源冲突程度 |

## 综合评分公式

```
total_score = (relevance × 0.30) + (freshness × 0.15) + (credibility × 0.20) + (source_quality × 0.15) + (completeness × 0.15) + (conflict_penalty × 0.05)
```

### 冲突惩罚
```
conflict_penalty = 1 - conflict_degree
```

## 排序逻辑

### 主排序
1. 按综合评分降序排列
2. 同分时按可信度降序排列
3. 仍同分时按时效性降序排列

### 分组排序
| 分组 | 说明 | 排序规则 |
|------|------|----------|
| 高可信来源 | 官方、权威来源 | 按时效性优先 |
| 一般来源 | 媒体、博客等 | 按相关性优先 |
| 低可信来源 | 个人、未验证 | 按完整度优先 |

## 评分示例

```json
{
  "resultId": "res_001",
  "scores": {
    "relevance": 0.85,
    "freshness": 0.90,
    "credibility": 0.80,
    "source_quality": 0.75,
    "completeness": 0.70,
    "conflict_degree": 0.10
  },
  "total_score": 0.82,
  "rank": 1
}
```

## 特殊处理

### 高风险任务
| 处理 | 说明 |
|------|------|
| 仅保留高可信来源 | 可信度 > 0.8 |
| 多源交叉验证 | 至少3个独立来源 |
| 标注来源 | 必须标注来源 |

### 时效敏感任务
| 处理 | 说明 |
|------|------|
| 时效性权重加倍 | freshness × 0.30 |
| 过滤过期信息 | freshness < 0.5 过滤 |
| 标注时效 | 必须标注发布时间 |

### 事实密集任务
| 处理 | 说明 |
|------|------|
| 完整度权重加倍 | completeness × 0.30 |
| 要求完整覆盖 | completeness < 0.6 过滤 |
| 标注引用 | 必须标注引用来源 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 评分计算失败 | 使用默认分数 0.5 |
| 无有效结果 | 返回空，标注原因 |
| 结果过多 | 限制返回数量，按评分截断 |

## 维护方式
- 调整权重: 修改评分公式权重
- 新增维度: 添加到评分维度表
- 调整排序: 更新排序逻辑
