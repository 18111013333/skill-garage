# CONTEXT_POLICY.md - 上下文治理策略

## 目的
定义上下文装配规则，确保每轮上下文构造有统一规则。

## 适用范围
所有LLM调用的上下文构造。

## 上下文优先级

### 装配优先级（从高到低）
| 优先级 | 内容类型 | 说明 | 最大占比 |
|--------|----------|------|----------|
| P0 | 当前用户请求 | 用户本轮输入 | 20% |
| P1 | 当前轮约束 | 用户明确约束条件 | 15% |
| P2 | 高相关会话记忆 | 与当前任务高度相关的历史 | 20% |
| P3 | 必要系统规则 | 安全边界、执行策略等 | 15% |
| P4 | 外部检索内容 | 搜索、文件等检索结果 | 20% |
| P5 | 补充上下文 | 其他辅助信息 | 10% |

### 禁止规则
- ❌ 禁止把低相关历史内容强塞进当前任务
- ❌ 禁止跳过系统安全规则
- ❌ 禁止覆盖用户硬约束
- ❌ 禁止隐式修改用户意图

## 上下文长度限制

### Token预算分配
| 组件 | 预算占比 | 说明 |
|------|----------|------|
| 系统提示 | 15% | 固定系统规则 |
| 用户输入 | 20% | 当前用户请求 |
| 会话记忆 | 25% | 历史上下文 |
| 检索结果 | 25% | 外部信息 |
| 输出预留 | 15% | 预留给输出 |

### 长度受限时的裁剪顺序
```
1. 删除 P5 补充上下文
2. 压缩 P4 外部检索内容（保留最高相关）
3. 压缩 P2 会话记忆（保留关键决策）
4. 简化 P3 系统规则（保留核心安全规则）
5. 永不裁剪 P0 用户请求和 P1 硬约束
```

## 上下文类型

### 按来源分类
| 类型 | 来源 | 处理方式 |
|------|------|----------|
| 用户输入 | 用户消息 | 原样保留 |
| 系统规则 | 系统配置 | 按需加载 |
| 会话记忆 | session-summaries | 按相关性筛选 |
| 长期记忆 | MEMORY.md | 按需检索 |
| 检索结果 | web/file搜索 | 按相关性排序 |
| 工具返回 | 工具执行结果 | 按需注入 |

### 按时效分类
| 类型 | 时效性 | 处理方式 |
|------|--------|----------|
| 实时 | 当前轮 | 必须包含 |
| 会话级 | 当前会话 | 按需包含 |
| 长期 | 跨会话 | 按需检索 |
| 静态 | 永久 | 系统规则 |

## 上下文冲突处理

### 冲突检测
- 用户约束 vs 系统规则 → 系统规则优先（安全）
- 新信息 vs 旧记忆 → 新信息优先（时效）
- 多来源冲突 → 标记冲突，显式处理

### 冲突标记格式
```
[CONFLICT DETECTED]
- Source A: {信息内容}
- Source B: {信息内容}
- Resolution: {处理方式}
```

## 上下文注入规则

### 注入时机
| 时机 | 注入内容 |
|------|----------|
| 任务开始 | 系统规则 + 用户请求 |
| 检索后 | 检索结果 |
| 工具调用后 | 工具返回 |
| 记忆更新后 | 新记忆片段 |

### 注入格式
```
--- [CONTEXT: {类型}] ---
{内容}
--- [END CONTEXT] ---
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 上下文超长 | 按裁剪顺序压缩 |
| 关键信息丢失 | 标记警告，请求用户确认 |
| 冲突无法解决 | 标记冲突，请求用户澄清 |

## 维护方式
- 调整优先级: 修改优先级表
- 调整预算分配: 修改Token预算表
- 新增上下文类型: 添加到类型表
