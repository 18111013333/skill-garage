# KNOWLEDGE_QUALITY_SCORE.md - 知识质量评分

## 目的
给知识条目做质量评分，影响检索排序与引用优先级。

## 适用范围
所有知识库内容的质量评估。

## 评分维度

### 维度定义
| 维度 | 权重 | 说明 |
|------|------|------|
| 来源可靠性 | 25% | 来源的可信程度 |
| 结构完整性 | 20% | 内容结构是否完整 |
| 时效性 | 20% | 内容是否最新 |
| 复用频率 | 15% | 被引用/使用频率 |
| 冲突情况 | 10% | 是否与其他知识冲突 |
| 人工确认状态 | 10% | 是否经过人工审核 |

## 评分规则

### 1. 来源可靠性 (25%)
| 来源类型 | 分数 | 说明 |
|----------|------|------|
| 官方文档 | 100 | 官方发布文档 |
| 权威机构 | 95 | 权威机构发布 |
| 专家审核 | 90 | 领域专家审核 |
| 内部文档 | 80 | 内部正式文档 |
| 用户贡献 | 60 | 用户提交内容 |
| 外部抓取 | 50 | 外部网页抓取 |
| 未标注来源 | 30 | 来源不明 |

### 2. 结构完整性 (20%)
| 结构要素 | 分值 | 说明 |
|----------|------|------|
| 标题清晰 | 20 | 有明确标题 |
| 内容完整 | 40 | 内容无缺失 |
| 分类正确 | 15 | 分类准确 |
| 标签合理 | 15 | 标签覆盖关键点 |
| 关联完整 | 10 | 相关知识已关联 |

### 3. 时效性 (20%)
| 时效状态 | 分数 | 说明 |
|----------|------|------|
| 最新 | 100 | 7天内更新 |
| 较新 | 90 | 30天内更新 |
| 正常 | 80 | 90天内更新 |
| 接近过期 | 60 | 180天内更新 |
| 过期 | 40 | 超过有效期 |
| 严重过期 | 20 | 长期未更新 |

### 4. 复用频率 (15%)
| 频率等级 | 分数 | 说明 |
|----------|------|------|
| 高频 | 100 | 每日使用 |
| 较高频 | 85 | 每周使用 |
| 中频 | 70 | 每月使用 |
| 低频 | 50 | 季度使用 |
| 极少 | 30 | 半年使用 |
| 未使用 | 10 | 从未使用 |

### 5. 冲突情况 (10%)
| 冲突状态 | 分数 | 说明 |
|----------|------|------|
| 无冲突 | 100 | 无任何冲突 |
| 轻微冲突 | 80 | 有轻微不一致 |
| 中度冲突 | 50 | 有明显矛盾 |
| 严重冲突 | 20 | 内容严重冲突 |
| 已解决冲突 | 90 | 冲突已解决 |

### 6. 人工确认状态 (10%)
| 确认状态 | 分数 | 说明 |
|----------|------|------|
| 专家确认 | 100 | 领域专家确认 |
| 管理员确认 | 90 | 知识管理员确认 |
| 创建者确认 | 80 | 创建者确认 |
| 待确认 | 50 | 等待确认 |
| 未确认 | 30 | 未经过确认 |

## 评分计算

### 计算公式
```
总分 = Σ(维度分数 × 权重)

总分 = (来源可靠性 × 0.25) + (结构完整性 × 0.20) + (时效性 × 0.20) + (复用频率 × 0.15) + (冲突情况 × 0.10) + (人工确认状态 × 0.10)
```

### 评分示例
```json
{
  "entry_id": "kb_001_e001",
  "quality_score": {
    "overall": 85,
    "dimensions": {
      "source_reliability": 90,
      "structure_completeness": 85,
      "freshness": 80,
      "reuse_frequency": 70,
      "conflict_status": 100,
      "human_verified": true
    },
    "calculation": {
      "source_reliability": 90 × 0.25 = 22.5,
      "structure_completeness": 85 × 0.20 = 17.0,
      "freshness": 80 × 0.20 = 16.0,
      "reuse_frequency": 70 × 0.15 = 10.5,
      "conflict_status": 100 × 0.10 = 10.0,
      "human_verified": 100 × 0.10 = 10.0,
      "total": 86.0
    }
  },
  "grade": "A",
  "evaluated_at": "2026-04-06T18:50:00+08:00"
}
```

|------|----------|------|----------|

## 评分影响

### 对检索的影响
```yaml

  high_quality:
    score_range: "A"
    boost: 1.5
    priority: "high"
  
  medium_quality:
    score_range: "B-C"
    boost: 1.0
    priority: "normal"
  
  low_quality:
    score_range: "D-F"
    boost: 0.5
    priority: "low"
```

## 评分更新

### 更新触发
| 触发条件 | 说明 |
|----------|------|
| 内容更新 | 知识内容变更 |
| 定期评估 | 周期性重评 |
| 使用统计 | 复用频率变化 |
| 冲突检测 | 发现新冲突 |
| 人工审核 | 审核完成 |

### 更新流程
```
评分更新触发
    ↓
┌─────────────────────────────────────┐
│ 1. 收集评分数据                      │
│    - 来源信息                        │
│    - 结构分析                        │
│    - 时效状态                        │
│    - 使用统计                        │
│    - 冲突状态                        │
│    - 审核状态                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 计算各维度分数                    │
│    - 应用评分规则                    │
│    - 计算维度分数                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 计算总分                          │
│    - 加权计算                        │
│    - 确定等级                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 更新索引                          │
│    - 更新质量索引                    │
│    - 更新检索权重                    │
│    - 记录评分历史                    │
└─────────────────────────────────────┘
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 数据缺失 | 使用默认值 |
| 计算异常 | 标记待人工评估 |
| 维度冲突 | 取最低分 |
