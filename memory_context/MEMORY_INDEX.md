# MEMORY.md - 记忆系统总索引

## 目的
作为记忆系统的总入口，定义记忆层级、读写流程、治理规则入口。

## 适用范围
所有记忆操作，包括读取、写入、更新、删除。

## 记忆层级

```
┌─────────────────────────────────────────────────────────┐
│                    记忆系统架构                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  L1 会话层 (短期)                                        │
│  ├── 存储: sessions/*.jsonl                             │
│  ├── 生命周期: 单次会话                                  │
│  └── 用途: 实时上下文                                    │
│                                                         │
│          ↓ 聚合                                         │
│                                                         │
│  L2 场景层 (中期)                                        │
│  ├── 存储: memory/scenarios/*.md                        │
│  ├── 生命周期: 场景周期                                  │
│  └── 用途: 场景画像                                      │
│                                                         │
│          ↓ 沉淀                                         │
│                                                         │
│  L3 长期层 (永久)                                        │
│  ├── 存储: MEMORY.md, USER.md, ontology/                │
│  ├── 生命周期: 永久                                      │
│  └── 用途: 核心记忆、知识图谱                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 写入流程

```
新信息产生
    ↓
┌─────────────────────────────────────┐
│ 1. 判断信息类型                      │
│    - 会话信息 → L1                   │
│    - 场景信息 → L2                   │
│    - 长期信息 → L3                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行写入前检查                    │
│    - 权限检查                        │
│    - 容量检查                        │
│    - 冲突检查                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行写入                          │
│    - 写入对应层级                    │
│    - 更新索引                        │
│    - 同步知识图谱                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 写入后验证                        │
│    - 完整性检查                      │
│    - 索引更新确认                    │
└─────────────────────────────────────┘
```

详细规则: `governance/MEMORY_POLICY.md`

## 读取流程

```
读取请求
    ↓
┌─────────────────────────────────────┐
│ 1. 确定读取范围                      │
│    - 单层读取                        │
│    - 跨层读取                        │
│    - 全层搜索                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行读取                          │
│    - L1: 直接读取                    │
│    - L2: 场景匹配                    │
│    - L3: 语义搜索                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 结果合并                          │
│    - 去重                            │
│    - 排序                            │
│    - 格式化                          │
└─────────────────────────────────────┘
```

详细规则: `governance/MEMORY_POLICY.md`

## 冲突处理入口

当记忆信息发生冲突时，按以下规则处理：

| 冲突类型 | 处理规则 | 详细文档 |
|----------|----------|----------|
| 同层冲突 | 时间优先 | `governance/MEMORY_CONFLICT_RULES.md` |
| 跨层冲突 | 层级优先 | `governance/MEMORY_CONFLICT_RULES.md` |
| 用户纠正 | 用户优先 | `governance/MEMORY_CONFLICT_RULES.md` |

## 删除入口

| 删除类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 会话清理 | 会话结束 | 归档到L2 |
| 场景归档 | 场景结束 | 归档到L3 |
| 长期删除 | 用户请求 | 需确认 |
| 知识图谱清理 | 实体过期 | 标记删除 |

详细规则: `governance/MEMORY_POLICY.md`

| 存储 | 说明 |
|------|------|

## 索引系统

### 向量索引
- 数据库: `~/.openclaw/memory/main.sqlite` (4.3MB)
- 向量化: Voyage API (1024维)
- 检索: `memory_search` 工具

### 知识图谱
- 存储: `memory/ontology/graph.jsonl`
- 实体数: 182
- 关系类型: Skill, Category, Learning, Error

## 维护方式
- 新增记忆类型: 更新层级定义
- 调整写入流程: 修改 `MEMORY_POLICY.md`
- 调整冲突规则: 修改 `MEMORY_CONFLICT_RULES.md`
- 本文件仅作为索引，不承载细则

## 引用文件
- `governance/MEMORY_POLICY.md` - 记忆治理策略
- `governance/MEMORY_SCHEMA.md` - 记忆结构定义
- `governance/MEMORY_CONFLICT_RULES.md` - 冲突处理规则
