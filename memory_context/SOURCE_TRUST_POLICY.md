# SOURCE_TRUST_POLICY.md - 来源可信度策略

## 目的
定义来源可信度分级，确保系统知道"什么来源可以支撑什么级别的结论"。

## 适用范围
所有信息来源的可信度评估。

## 来源可信度分级

### 一级：最高可信度 (1.0)
| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 用户明确输入 | 用户直接陈述的信息 | 用户说"我叫张三" |
| 系统长期记忆 | 已验证的长期记忆 | 用户画像中的偏好 |
| 官方文档 | 官方发布的技术文档 | 官方API文档 |
| 权威机构 | 权威机构发布的信息 | 政府公告、学术论文 |

### 二级：高可信度 (0.8)
| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 内部文件 | 用户授权的内部文件 | 用户上传的文档 |
| 主流媒体 | 主流媒体报道 | 新华社、BBC等 |
| 专业网站 | 专业领域网站 | 技术博客、专业论坛 |
| 已验证来源 | 历史准确率高的来源 | 可靠的新闻源 |

### 三级：中等可信度 (0.6)
| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 一般网页 | 普通网页内容 | 一般网站文章 |
| 社交媒体 | 社交媒体信息 | 微博、Twitter等 |
| 用户评论 | 用户生成内容 | 评论、评价等 |
| 未验证来源 | 未经验证的来源 | 新发现的来源 |

### 四级：低可信度 (0.4)
| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 个人博客 | 个人发布的内容 | 个人博客文章 |
| 论坛帖子 | 论坛讨论内容 | 论坛帖子 |
| 匿名来源 | 匿名发布的信息 | 匿名评论 |
| 广告内容 | 商业推广内容 | 广告、软文 |

### 五级：极低可信度 (0.2)
| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 谣言来源 | 已知谣言来源 | 被标记的假新闻源 |
| 利益相关 | 有明显利益倾向 | 商业推广、公关稿 |
| 过期信息 | 已过期的信息 | 过期的政策、价格 |
| 冲突信息 | 与已知信息冲突 | 未验证的冲突信息 |

## 可信度与结论级别

### 高风险结论
| 要求 | 说明 |
|------|------|
| 最低可信度 | ≥ 0.8 |
| 来源数量 | ≥ 3个独立来源 |
| 交叉验证 | 必须交叉验证 |
| 适用场景 | 医疗、法律、财务、安全 |

### 中风险结论
| 要求 | 说明 |
|------|------|
| 最低可信度 | ≥ 0.6 |
| 来源数量 | ≥ 2个来源 |
| 交叉验证 | 建议交叉验证 |
| 适用场景 | 事实陈述、数据引用 |

### 低风险结论
| 要求 | 说明 |
|------|------|
| 最低可信度 | ≥ 0.4 |
| 来源数量 | ≥ 1个来源 |
| 交叉验证 | 可选 |
| 适用场景 | 一般信息、观点引用 |

### 不适用结论
| 要求 | 说明 |
|------|------|
| 可信度 | < 0.4 |
| 处理 | 标注不确定性 |
| 适用场景 | 仅供参考 |

## 来源可信度评估

### 评估因素
| 因素 | 权重 | 说明 |
|------|------|------|
| 来源类型 | 40% | 来源的基本类型 |
| 历史准确率 | 30% | 该来源历史准确率 |
| 验证状态 | 20% | 是否经过验证 |
| 时效性 | 10% | 信息时效性 |

### 动态调整
| 条件 | 调整 |
|------|------|
| 连续准确 | 可信度 +0.1 |
| 连续错误 | 可信度 -0.2 |
| 用户反馈 | 根据反馈调整 |
| 官方认证 | 可信度设为 0.9 |

## 来源冲突处理

### 冲突检测
| 情况 | 处理 |
|------|------|
| 高可信 vs 低可信 | 采用高可信来源 |
| 同级冲突 | 标注冲突，请求用户确认 |
| 多源一致 | 增强可信度 |

### 冲突标注
```
[CONFLICT]
- Source A (trust: 0.8): {内容A}
- Source B (trust: 0.6): {内容B}
- Resolution: 采用Source A
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 来源无法评估 | 默认可信度 0.5 |
| 可信度不足 | 标注不确定性 |
| 来源冲突 | 标注冲突，请求确认 |

## 维护方式
- 新增来源类型: 添加到分级表
- 调整可信度: 更新分级标准
- 新增评估因素: 更新评估因素表
