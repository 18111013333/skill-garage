# UNCERTAINTY_HANDLING.md - 不确定性处理

## 目的
定义不确定信息的处理规则，确保不会把猜测当成事实。

## 适用范围
所有包含不确定性的信息处理。

## 不确定性类型

### 信息不确定性
| 类型 | 说明 | 示例 |
|------|------|------|
| 来源不确定 | 来源可信度低 | "据传言..." |
| 时效不确定 | 信息可能过期 | "截至2024年..." |
| 完整性不确定 | 信息不完整 | "部分信息显示..." |
| 冲突不确定 | 来源冲突 | "不同来源说法不一..." |

### 推理不确定性
| 类型 | 说明 | 示例 |
|------|------|------|
| 前提不确定 | 前提可能不成立 | "如果A成立，那么..." |
| 推理不确定 | 推理可能不严谨 | "可能意味着..." |
| 结论不确定 | 结论可能不准确 | "推测..." |

### 知识不确定性
| 类型 | 说明 | 示例 |
|------|------|------|
| 知识缺失 | 缺乏相关知识 | "我不确定..." |
| 知识模糊 | 知识边界不清 | "可能涉及..." |
| 知识冲突 | 知识之间存在冲突 | "存在不同观点..." |

## 不确定性标记

### 标记类型
| 标记 | 含义 | 使用场景 |
|------|------|----------|
| `[不确定]` | 信息不确定 | 信息可信度<70% |
| `[推测]` | 基于推理 | 无直接来源支持 |
| `[可能]` | 可能性陈述 | 多种可能性 |
| `[待验证]` | 需要验证 | 无法确认 |
| `[争议]` | 存在争议 | 多种观点 |

### 标记规则
```json
{
  "uncertaintyMarkers": {
    "high": {
      "confidence": "< 50%",
      "marker": "[不确定]",
      "action": "明确说明不确定性"
    },
    "medium": {
      "confidence": "50-70%",
      "marker": "[可能]",
      "action": "标记可能性"
    },
    "low": {
      "confidence": "70-90%",
      "marker": "[推测]",
      "action": "说明推理依据"
    }
  }
}
```

## 不确定性处理流程

```
不确定信息
    ↓
┌─────────────────────────────────────┐
│ 1. 识别不确定性                      │
│    - 信息不确定性                    │
│    - 推理不确定性                    │
│    - 知识不确定性                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 评估不确定性程度                  │
│    - 高度不确定 (< 50%)              │
│    - 中度不确定 (50-70%)             │
│    - 低度不确定 (70-90%)             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 选择处理策略                      │
│    - 拒绝回答                        │
│    - 标记不确定                      │
│    - 提供多种可能                    │
│    - 请求更多信息                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 执行处理                          │
│    - 添加标记                        │
│    - 说明原因                        │
│    - 提供替代方案                    │
└─────────────────────────────────────┘
```

## 处理策略

### 高度不确定 (< 50%)
| 策略 | 说明 | 示例 |
|------|------|------|
| 拒绝回答 | 不提供不确定信息 | "我无法确定这个信息" |
| 明确说明 | 说明不确定性 | "这个信息非常不确定" |
| 请求补充 | 请求更多信息 | "需要更多信息才能回答" |

### 中度不确定 (50-70%)
| 策略 | 说明 | 示例 |
|------|------|------|
| 标记可能 | 标记为可能性 | "可能是..." |
| 提供来源 | 说明信息来源 | "根据XX来源..." |
| 说明局限 | 说明信息局限 | "信息可能不完整" |

### 低度不确定 (70-90%)
| 策略 | 说明 | 示例 |
|------|------|------|
| 标记推测 | 标记为推测 | "推测..." |
| 说明依据 | 说明推理依据 | "基于XX推断..." |
| 提供置信度 | 说明置信度 | "有较高可能性..." |

## 不确定性表达

### 表达模板
| 置信度 | 表达模板 |
|--------|----------|
| < 50% | "我无法确定...，建议..." |
| 50-70% | "根据现有信息，可能是...，但不确定" |
| 70-90% | "推测是...，基于..." |
| > 90% | "..." (正常表达) |

### 表达示例
```
高度不确定:
"我无法确定这个信息的准确性，建议您查阅官方来源。"

中度不确定:
"根据网络搜索结果，可能是XX，但信息来源未经证实。"

低度不确定:
"推测是XX，基于YY的推理，建议进一步验证。"
```

## 不确定性报告

### 报告格式
```json
{
  "uncertaintyId": "unc_20260406_111200",
  "timestamp": "2026-04-06T11:12:00+08:00",
  "type": "information_uncertainty",
  "level": "medium",
  "confidence": 0.65,
  "reason": "来源可信度较低",
  "handling": {
    "strategy": "mark_possible",
    "marker": "[可能]",
    "explanation": "根据网络信息，但来源未经证实"
  }
}
```

## 禁止行为

| 禁止类型 | 说明 |
|----------|------|
| 隐瞒不确定性 | 不说明信息不确定 |
| 夸大确定性 | 把不确定说成确定 |
| 编造信息 | 编造不存在的信息 |
| 忽略冲突 | 忽略信息冲突 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 无法评估不确定性 | 按高度不确定处理 |
| 不确定性过高 | 拒绝回答 |
| 用户要求确定答案 | 说明限制 |

## 维护方式
- 新增不确定性类型: 添加到类型表
- 调整置信度阈值: 修改阈值配置
- 新增处理策略: 更新策略表
