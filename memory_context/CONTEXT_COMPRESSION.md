# CONTEXT_COMPRESSION.md - 上下文压缩策略

## 目的
定义长上下文压缩策略，确保长任务不会因为压缩而丢关键限制。

## 适用范围
上下文长度超限时的内容压缩。

## 压缩类型

### 按内容类型压缩
| 类型 | 压缩策略 | 保留内容 |
|------|----------|----------|
| 事实压缩 | 提取关键事实，删除细节 | 核心事实陈述 |
| 步骤压缩 | 合并相似步骤，保留关键节点 | 开始、转折、结束 |
| 偏好压缩 | 合并相似偏好，保留明确偏好 | 用户硬约束 |
| 对话压缩 | 摘要对话内容，保留关键决策 | 决策、承诺、约束 |

### 按重要性压缩
| 级别 | 内容 | 压缩方式 |
|------|------|----------|
| 必须保留 | 用户硬约束 | 不压缩 |
| 必须保留 | 任务目标 | 不压缩 |
| 必须保留 | 未完成事项 | 不压缩 |
| 必须保留 | 风险边界 | 不压缩 |
| 可压缩 | 详细过程 | 摘要压缩 |
| 可压缩 | 历史对话 | 摘要压缩 |
| 可压缩 | 检索结果 | 选择性保留 |

## 必须保留内容

### 用户硬约束
```
[CONSTRAINT: HARD]
- {约束内容}
- 来源: {用户输入时间}
- 状态: ACTIVE
```

### 任务目标
```
[GOAL: ACTIVE]
- {目标描述}
- 进度: {完成百分比}
- 阻塞: {阻塞项}
```

### 未完成事项
```
[TODO: PENDING]
- {事项内容}
- 优先级: {P0-P3}
- 依赖: {依赖项}
```

### 风险边界
```
[RISK: BOUNDARY]
- {风险类型}
- 阈值: {阈值}
- 当前: {当前值}
```

## 压缩算法

### 事实压缩
```
原始: "用户说他们住在北京市海淀区中关村大街1号，这是一个科技园区附近"
压缩: "用户位置: 北京海淀区中关村"
```

### 步骤压缩
```
原始: 
1. 用户请求搜索
2. 系统调用搜索工具
3. 返回10个结果
4. 用户选择第3个
5. 系统打开详情
压缩: "搜索→选择结果#3→查看详情"
```

### 对话压缩
```
原始: [完整对话历史]
压缩: 
[SUMMARY]
- 用户目标: {目标}
- 已完成: {完成项}
- 待处理: {待办项}
- 关键决策: {决策列表}
[/SUMMARY]
```

## 压缩触发条件

| 条件 | 阈值 | 动作 |
|------|------|------|
| Token超限 | > 80%预算 | 触发压缩 |
| 上下文过长 | > 10000 tokens | 触发压缩 |
| 历史过多 | > 20轮对话 | 触发压缩 |
| 检索过多 | > 5个结果 | 选择性保留 |

## 压缩流程

```
检测超限
    ↓
┌─────────────────────────────────────┐
│ 1. 识别可压缩内容                    │
│    - 标记必须保留                    │
│    - 标记可压缩                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 选择压缩策略                      │
│    - 按内容类型选择                  │
│    - 按重要性选择                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行压缩                          │
│    - 生成摘要                        │
│    - 保留关键信息                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 验证压缩结果                      │
│    - 检查必须保留内容                │
│    - 检查压缩后长度                  │
└─────────────────────────────────────┘
    ↓
继续或再次压缩
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 压缩后仍超限 | 再次压缩或请求用户 |
| 关键信息丢失 | 回滚压缩，标记警告 |
| 压缩失败 | 使用原始内容，标记超限 |

## 维护方式
- 新增压缩类型: 添加到压缩类型表
- 调整保留规则: 更新必须保留内容
- 调整触发条件: 修改触发条件表
