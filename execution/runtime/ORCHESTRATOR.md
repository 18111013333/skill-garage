# ORCHESTRATOR.md - 系统总调度手册

## 目的
定义用户请求进入系统后的总执行顺序，确保任务有序、可控、可追踪。

## 适用范围
所有进入系统的用户请求，包括直接对话、API调用、定时任务触发。

## 执行主流程

```
用户请求

└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 2. 任务分类 (TASK_CLASSIFIER.json)                      │
│    - 匹配任务类型                                        │
│    - 评估风险级别                                        │
│    - 确定执行路径                                        │

    ↓

│    - 中风险 → 记录后执行                                 │

│    - 极高风险 → 拒绝或双重确认                           │

    ↓
```

## 三种主路径

### 简单任务路径
- **触发条件**: 任务类型为 qa、search、creative_generation
- **执行方式**: 直接执行，无需规划
- **最大步骤**: 3-5步
- **超时**: 30秒

### 复杂任务路径
- **触发条件**: 任务类型为 planning、multi_step_agent、document_edit
- **执行方式**: 先规划，再分步执行
- **最大步骤**: 15-30步
- **超时**: 180秒
- **要求**: 必须生成执行计划

### 高风险任务路径
- **触发条件**: 任务类型为 high_risk_request
- **执行方式**: 审批 → 确认 → 执行 → 审计
- **最大步骤**: 5步
- **超时**: 60秒
- **要求**: 双重确认 + 审计日志

## 何时直接答复
1. 任务类型为 qa 且置信度 > 0.8
2. 无需外部工具调用
3. 无需记忆更新
4. 风险级别为 low

## 何时进入多步工作流
1. 任务类型为 multi_step_agent
2. 需要多个技能协作
3. 需要工具链调用
4. 用户明确要求多步骤执行

## 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 分类失败 | 使用 fallback 任务类型 |
| 技能不可用 | 尝试备选技能 |
| 工具调用失败 | 执行回退方案 |
| 超时 | 终止并返回部分结果 |
| 连续失败 | 触发降级模式 |

## 维护方式
- 新增任务类型: 修改 `TASK_CLASSIFIER.json`
- 新增技能路由: 修改 `SKILL_ROUTER.json`
- 调整执行策略: 修改 `EXECUTION_POLICY.md`
- 新增异常处理: 修改本文件异常处理表
