# PLANNER.md - 任务规划规则

## 目的
定义何时需要拆解任务、如何拆解、如何排序，确保复杂任务有序执行。

## 适用范围
- 任务类型为 planning、multi_step_agent、document_edit
- 预估步骤数 > 5 的任务
- 用户明确要求规划的任务

## 规划决策

### 不需要规划
| 条件 | 说明 |
|------|------|
| 任务类型为 qa、search | 单步即可完成 |
| 预估步骤 ≤ 3 | 直接执行更高效 |
| 用户要求快速响应 | 跳过规划阶段 |
| 置信度 > 0.9 | 意图明确无需规划 |

### 需要规划
| 条件 | 说明 |
|------|------|
| 任务类型为 multi_step_agent | 明确的多步任务 |
| 预估步骤 > 5 | 需要协调多个技能 |
| 涉及文件操作 | 需要确保顺序 |
| 涉及外部调用 | 需要处理依赖 |

## 拆解规则

### 最大拆解层数
```
max_depth = 3
```
- 第1层: 主任务
- 第2层: 子任务
- 第3层: 原子操作

超过3层时，第3层不再拆解，直接作为原子操作执行。

### 拆解原则
1. **单一职责**: 每个子任务只做一件事
2. **可验证性**: 每个子任务有明确的完成标准
3. **可回滚性**: 每个子任务失败后可独立回滚
4. **依赖明确**: 子任务间的依赖关系必须显式声明

## 子任务排序

### 拓扑排序
1. 构建依赖图
2. 检测循环依赖
3. 按依赖关系排序
4. 无依赖的任务可并行

### 优先级规则
| 优先级 | 条件 |
|--------|------|
| P0 | 阻塞其他任务的前置任务 |
| P1 | 用户明确要求优先的任务 |
| P2 | 默认优先级 |
| P3 | 可延后执行的任务 |

## 依赖关系

### 依赖类型
| 类型 | 说明 | 示例 |
|------|------|------|
| sequential | 必须顺序执行 | A → B → C |
| parallel | 可并行执行 | A, B, C 同时执行 |
| conditional | 条件执行 | if A.success then B |
| fallback | 失败时执行 | A.fail → B |

### 依赖声明格式
```json
{
  "taskId": "task_001",
  "dependsOn": ["task_000"],
  "dependencyType": "sequential",
  "condition": null
}
```

## 可并行条件

### 可并行
- 无依赖关系
- 读写不同文件
- 调用不同API
- 用户无顺序要求

### 不可并行
- 有显式依赖
- 读写同一文件
- 共享状态修改
- 用户要求顺序

## 规划失败降级

### 降级策略
| 失败原因 | 降级方案 |
|----------|----------|
| 循环依赖 | 打破循环，改为顺序 |
| 依赖不可解 | 移除依赖，独立执行 |
| 拆解超限 | 合并子任务 |
| 规划超时 | 使用默认顺序 |

### 降级流程
```
规划失败
    ↓
记录失败原因
    ↓
选择降级方案
    ↓
生成简化计划
    ↓
继续执行
```

## 规划输出格式

```json
{
  "planId": "plan_20260406_001",
  "taskId": "task_001",
  "totalSteps": 5,
  "estimatedTime": 60000,
  "subtasks": [
    {
      "id": "subtask_001",
      "name": "搜索资料",
      "skill": "xiaoyi-web-search",
      "dependsOn": [],
      "priority": 0,
      "timeout": 30000
    }
  ],
  "executionOrder": ["subtask_001", "subtask_002", "subtask_003"],
  "parallelGroups": [["subtask_004", "subtask_005"]]
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 循环依赖检测 | 自动打破，记录警告 |
| 依赖任务失败 | 跳过依赖任务，继续独立任务 |
| 规划超时 | 使用简化计划 |
| 子任务超限 | 合并为更大粒度任务 |

## 维护方式
- 调整最大拆解层数: 修改 `max_depth`
- 新增依赖类型: 更新依赖类型表
- 调整优先级规则: 更新优先级表
- 新增降级策略: 更新降级策略表
