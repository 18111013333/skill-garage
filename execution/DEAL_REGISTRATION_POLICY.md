# DEAL_REGISTRATION_POLICY.md - 终极鸽子王商机报备规则

## 目的
定义商机报备的规则，避免渠道销售归属冲突。

## 适用范围
所有渠道伙伴的商机报备与管理。

---

## 一、报备规则总览

```
商机报备流程
    │
    ├── 报备申请
    │   ├── 提交商机信息
    │   └── 平台审核
    │
    ├── 保护期
    │   ├── 独家跟进权
    │   └── 定期更新
    │
    └── 成交确认
        ├── 归属确认
        └── 分成结算
```

---

## 二、谁可报备

### 2.1 报备资格
| 伙伴等级 | 报备资格 | 报备数量限制 |
|----------|----------|--------------|
| 注册级 | 可报备 | 5个/月 |
| 认证级 | 可报备 | 15个/月 |
| 高级级 | 可报备 | 50个/月 |
| 战略级 | 可报备 | 无限制 |

### 2.2 报备条件
| 条件 | 说明 |
|------|------|
| 伙伴状态正常 | 未被暂停或终止 |
| 无逾期欠款 | 财务状态正常 |
| 报备额度充足 | 未超月度限制 |

---

## 三、报备保护期

### 3.1 保护期定义
| 商机类型 | 保护期 | 可延长 |
|----------|--------|--------|
| 新客户 | 90天 | 可延长1次(30天) |
| 存量客户扩容 | 60天 | 不可延长 |
| 续费商机 | 30天 | 不可延长 |

### 3.2 保护期权益
| 权益 | 说明 |
|------|------|
| 独家跟进权 | 保护期内其他伙伴不可跟进 |
| 优先支持权 | 平台优先支持报备伙伴 |
| 归属保障权 | 成交后归属报备伙伴 |

### 3.3 保护期义务
| 义务 | 说明 |
|------|------|
| 定期更新 | 每30天更新商机状态 |
| 积极跟进 | 需证明积极跟进 |
| 信息真实 | 报备信息需真实有效 |

---

## 四、冲突商机处理

### 4.1 冲突类型
| 冲突类型 | 说明 | 处理方式 |
|----------|------|----------|
| 同时报备 | 多伙伴同时报备同一客户 | 先报备者优先 |
| 交叉报备 | 不同伙伴报备同一客户不同项目 | 分别保护 |
| 存量客户 | 客户已有其他伙伴服务 | 原服务者优先 |

### 4.2 冲突处理流程
```
冲突处理流程
    ↓
┌─────────────────────────────────────┐
│ 1. 冲突识别                          │
│    - 系统自动检测                    │
│    - 人工申诉识别                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 证据收集                          │
│    - 报备时间证据                    │
│    - 跟进记录证据                    │
│    - 客户关系证据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 归属判定                          │
│    - 按规则自动判定                  │
│    - 复杂情况人工判定                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 结果通知                          │
│    - 通知相关方                      │
│    - 说明判定依据                    │
└─────────────────────────────────────┘
```

## 五、归属判定

### 5.1 判定规则
| 规则 | 优先级 | 说明 |
|------|--------|------|
| 先报备原则 | 1 | 先报备者优先 |
| 积极跟进原则 | 2 | 积极跟进者优先 |
| 客户意愿原则 | 3 | 客户指定优先 |
| 原服务原则 | 4 | 原服务伙伴优先 |

### 5.2 判定证据
| 证据类型 | 说明 | 权重 |
|----------|------|------|
| 报备时间 | 系统记录的报备时间 | 高 |
| 跟进记录 | 拜访记录、沟通记录 | 高 |
| 客户确认 | 客户书面确认 | 最高 |
| 合同历史 | 历史合作记录 | 中 |

## 六、例外审批

### 6.1 例外场景
| 场景 | 说明 | 审批人 |
|------|------|--------|
| 战略客户 | 平台战略客户 | 销售总监 |
| 冲突申诉 | 对归属判定有异议 | 渠道经理 |
| 特殊保护 | 申请延长保护期 | 渠道经理 |
| 客户指定 | 客户明确指定伙伴 | 自动通过 |

### 6.2 例外审批流程
```
例外审批流程
    ↓
┌─────────────────────────────────────┐
│ 1. 提交申请                          │
│    - 说明例外原因                    │
│    - 提供支持材料                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 审批审核                          │
│    - 审核申请材料                    │
│    - 必要时调查核实                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 审批决定                          │
│    - 批准/拒绝                       │
│    - 说明决定理由                    │
└─────────────────────────────────────┘
```

## 七、报备信息要求

### 7.1 必填信息
| 信息项 | 说明 |
|--------|------|
| 客户名称 | 全称，不可缩写 |
| 客户联系人 | 关键决策人信息 |
| 项目需求 | 项目概述和需求 |
| 预算范围 | 预估项目金额 |
| 预计时间 | 预计成交时间 |
| 竞争情况 | 竞争对手情况 |

### 7.2 信息更新要求
| 更新项 | 频率 | 说明 |
|--------|------|------|
| 商机状态 | 每30天 | 需更新跟进状态 |
| 预计金额 | 有变化时 | 及时更新 |
| 预计时间 | 有变化时 | 及时更新 |
| 关键进展 | 有进展时 | 记录关键节点 |

---

## 八、违规处理

### 8.1 违规类型
| 违规类型 | 说明 | 处理 |
|----------|------|------|
| 虚假报备 | 报备虚假商机 | 取消报备+警告 |
| 恶意抢报 | 恶意抢报他人商机 | 取消报备+降级 |
| 信息不更新 | 长期不更新商机 | 自动取消保护 |
| 重复报备 | 重复报备同一商机 | 保留最早报备 |

### 8.2 处理流程
```
违规处理流程
    ↓
┌─────────────────────────────────────┐
│ 1. 违规识别                          │
│    - 系统检测                        │
│    - 人工举报                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 违规核实                          │
│    - 调查核实                        │
│    - 听取申诉                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 处理决定                          │
│    - 警告/取消报备/降级              │
│    - 记录违规档案                    │
└─────────────────────────────────────┘
```
