# GOAL_EXTRACTION_RULES.md - 目标提取规则

## 目的
定义从用户表达中抽取目标、约束、成功标准、风险偏好的规则。

## 适用范围
所有复杂任务前的目标提取。

## 提取字段

### 必须提取
| 字段 | 说明 | 示例 |
|------|------|------|
| goal | 用户目标 | "完成报告" |
| constraints | 约束条件 | "不超过500字" |
| deadline | 截止时间 | "明天之前" |
| budget | 预算限制 | "免费方案" |
| preferred_format | 偏好格式 | "表格形式" |
| unacceptable_outcomes | 不可接受结果 | "不能泄露隐私" |

### 可选提取
| 字段 | 说明 | 示例 |
|------|------|------|
| priority | 优先级 | "紧急" |
| quality_level | 质量要求 | "高质量" |
| risk_preference | 风险偏好 | "保守" |
| reference | 参考对象 | "像XX那样" |

## 提取规则

### 目标提取
| 规则 | 模式 | 提取方式 |
|------|------|----------|
| 直接陈述 | "我要/想/需要..." | 提取后续内容 |
| 问题形式 | "怎么/如何..." | 转换为目标 |
| 动作形式 | "帮我/给我..." | 提取动作对象 |
| 结果形式 | "实现/完成..." | 提取结果描述 |

### 约束提取
| 规则 | 模式 | 提取方式 |
|------|------|----------|
| 数量约束 | "不超过/至少/最多..." | 提取数量限制 |
| 时间约束 | "在...之前/...天内" | 提取时间限制 |
| 格式约束 | "用...格式/...形式" | 提取格式要求 |
| 内容约束 | "必须包含/不能有..." | 提取内容限制 |
| 资源约束 | "只用/仅用..." | 提取资源限制 |

### 成功标准提取
| 规则 | 模式 | 提取方式 |
|------|------|----------|
| 明确标准 | "需要满足/达到..." | 提取标准描述 |
| 验收条件 | "验收标准是..." | 提取验收条件 |
| 质量要求 | "要高质量/专业..." | 提取质量要求 |
| 功能要求 | "能实现/支持..." | 提取功能要求 |

### 风险偏好提取
| 规则 | 模式 | 提取方式 |
|------|------|----------|
| 保守倾向 | "稳妥/保守/安全..." | 标记为保守 |
| 激进倾向 | "激进/冒险/快速..." | 标记为激进 |
| 平衡倾向 | "平衡/折中/适中..." | 标记为平衡 |

## 提取流程

```
用户输入
    ↓
┌─────────────────────────────────────┐
│ 1. 分词与实体识别                   │
│    - 分词                           │
│    - 识别关键实体                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 模式匹配                         │
│    - 匹配目标模式                   │
│    - 匹配约束模式                   │
│    - 匹配标准模式                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 语义理解                         │
│    - 理解隐含目标                   │
│    - 理解隐含约束                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 结构化输出                       │
│    - 生成目标结构                   │
│    - 验证完整性                     │
└─────────────────────────────────────┘
    ↓
目标结构
```

## 隐含信息推断

### 隐含目标
| 情况 | 推断 |
|------|------|
| "这个报错怎么解决" | 目标: 解决报错，完成程序运行 |
| "帮我优化代码" | 目标: 提高代码性能/可读性 |
| "分析这个数据" | 目标: 得出分析结论 |

### 隐含约束
| 情况 | 推断 |
|------|------|
| 商务邮件 | 约束: 正式语气 |
| 技术文档 | 约束: 准确专业 |
| 创作内容 | 约束: 原创性 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 目标不明确 | 请求用户澄清 |
| 约束冲突 | 标注冲突，请求确认 |
| 信息不足 | 标注缺失，请求补充 |

## 维护方式
- 新增提取字段: 添加到提取字段表
- 新增提取规则: 更新提取规则表
- 新增推断规则: 更新隐含推断表
