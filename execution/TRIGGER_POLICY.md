# TRIGGER_POLICY.md - 事件触发规则

**版本: V27.0**

## 目的
定义事件触发规则，确保事件不会无控制触发执行链，实现"可控触发、分级响应、安全执行"。

## 适用范围
所有事件触发场景，包括：
- 自动分析触发
- 自动草拟触发
- 提醒触发
- 审批后执行触发

---

## 一、触发类型分类

| 触发类型 | 代码 | 说明 | 风险等级 |
|----------|------|------|----------|
| 自动分析 | AUTO_ANALYZE | 自动分析事件并输出结果 | 低 |
| 自动草拟 | AUTO_DRAFT | 自动草拟响应或文档 | 中 |
| 仅提醒 | NOTIFY_ONLY | 仅发送提醒，不执行操作 | 低 |
| 审批后执行 | APPROVE_THEN_EXEC | 审批通过后执行 | 高 |
| 立即执行 | IMMEDIATE_EXEC | 立即执行（仅限安全操作） | 极高 |

## 四、触发流程

### 4.1 自动分析流程

```
事件产生
    ↓
┌─────────────────────────────────────┐
│ 1. 事件验证                          │
│    - 验证事件格式                    │
│    - 验证事件来源                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 频率检查                          │
│    - 检查触发频率                    │
│    - 超限则排队或丢弃                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 分析执行                          │
│    - 调用分析模块                    │
│    - 生成分析结果                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 结果处理                          │
│    - 缓存结果                        │
│    - 通知相关方                      │
│    - 记录审计日志                    │
└─────────────────────────────────────┘
```

### 4.2 自动草拟流程

```
事件产生
    ↓
┌─────────────────────────────────────┐
│ 1. 事件验证                          │
│    - 验证事件格式                    │
│    - 验证草拟权限                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 草拟执行                          │
│    - 调用草拟模块                    │
│    - 生成草稿内容                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 质量检查                          │
│    - 检查草稿质量                    │
│    - 不合格则重新草拟                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 审核流程                          │
│    - 提交人工审核                    │
│    - 等待审核结果                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 发布处理                          │
│    - 审核通过后发布                  │
│    - 记录审计日志                    │
└─────────────────────────────────────┘
```

### 4.3 审批后执行流程

```
事件产生
    ↓
┌─────────────────────────────────────┐
│ 1. 事件验证                          │
│    - 验证事件格式                    │

    ↓

    ↓

    ↓

    ↓
```

## 五、触发限制

### 5.1 频率限制

| 限制类型 | 说明 | 默认值 |
|----------|------|--------|
| 全局频率 | 全局事件触发频率 | 1000/分钟 |
| 类型频率 | 单类型事件触发频率 | 100/分钟 |
| 租户频率 | 单租户事件触发频率 | 500/分钟 |
| 用户频率 | 单用户事件触发频率 | 50/分钟 |

### 5.2 时间限制

| 限制类型 | 说明 | 默认值 |

| 维护窗口 | 维护时段禁止触发 | 可配置 |
| 峰值避让 | 峰值时段降级触发 | 可配置 |

### 5.3 资源限制

| 限制类型 | 说明 | 默认值 |
|----------|------|--------|
| CPU 限制 | 触发处理 CPU 限制 | 80% |
| 内存限制 | 触发处理内存限制 | 80% |
| 队列限制 | 待处理队列长度 | 10000 |

## 六、触发降级

### 6.1 降级条件

| 条件 | 说明 | 降级策略 |
|------|------|----------|
| 资源不足 | CPU/内存超限 | 排队等待 |
| 队列满 | 待处理队列满 | 丢弃低优先级 |
| 错误率高 | 错误率超阈值 | 暂停触发 |
| 依赖异常 | 依赖服务异常 | 降级为提醒 |

### 6.2 降级策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 排队等待 | 排队等待处理 | 资源不足 |
| 丢弃 | 丢弃低优先级事件 | 队列满 |
| 降级为提醒 | 仅发送提醒 | 依赖异常 |
| 暂停触发 | 暂停所有触发 | 错误率高 |

---

## 七、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 触发失败 | 重试 + 告警 |
| 分析失败 | 降级为提醒 + 告警 |
| 草拟失败 | 通知人工处理 |
| 审批超时 | 自动拒绝 + 通知 |
| 执行失败 | 回滚 + 告警 |

---

## 八、完成标准

| 标准 | 验证方式 |
|------|----------|
| 触发可控 | 触发按规则执行 |
| 频率限制 | 频率限制生效 |
| 降级有效 | 降级策略正确执行 |
| 审计完整 | 所有触发有记录 |
