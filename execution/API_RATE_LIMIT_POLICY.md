# API_RATE_LIMIT_POLICY.md - API 限流与配额规则

## 目的
定义 API 限流策略，保护平台免受流量异常影响。

## 适用范围
所有对外公开的 API 接口。

---

## 一、限流维度

### 1.1 限流层级
```
租户级 → 应用级 → 端点级 → 风险级
```

### 1.2 限流维度说明
| 维度 | 说明 | 优先级 |
|------|------|--------|
| 租户级 | 按租户总限制 | 最高 |
| 应用级 | 按应用限制 | 高 |
| 端点级 | 按接口限制 | 中 |
| 风险级 | 按风险等级限制 | 低 |

---

## 二、租户限流

### 2.1 计划配额
| 计划 | 每分钟 | 每小时 | 每天 |
|------|--------|--------|------|
| Free | 10 | 100 | 100 |
| Standard | 60 | 1000 | 10000 |
| Premium | 300 | 5000 | 100000 |
| Enterprise | 1000 | 50000 | 无限 |

### 2.2 租户限流响应
```json
{
  "error": "tenant_rate_limit_exceeded",
  "message": "租户请求限制已达到",
  "limit": 100,
  "current": 100,
  "reset_at": "2026-04-07T13:00:00Z",
  "retry_after": 3600
}
```

## 三、应用限流

### 3.1 应用配额
| 应用类型 | 每分钟 | 说明 |
|----------|--------|------|
| Web 应用 | 100 | 前端应用 |
| 移动应用 | 50 | 移动端 |
| 服务应用 | 200 | 后端服务 |
| 批处理应用 | 500 | 批量任务 |

### 3.2 应用标识
```http
X-Application-Id: app_xxx
```

## 四、端点限流

### 4.1 端点分类
| 分类 | 每分钟限制 | 说明 |
|------|------------|------|
| 读操作 | 200 | GET 请求 |
| 写操作 | 100 | POST/PUT 请求 |
| 删除操作 | 20 | DELETE 请求 |
| 批量操作 | 10 | 批量接口 |
| 管理操作 | 5 | 管理接口 |

### 4.2 特殊端点
| 端点 | 限制 | 原因 |
|------|------|------|
| /v1/chat/stream | 10/分钟 | 流式响应资源占用 |
| /v1/execute | 5/分钟 | 复杂任务执行 |
| /v1/analytics/query | 30/分钟 | 计算密集 |

---

## 五、风险级别限流

### 5.1 风险等级
| 等级 | 说明 | 限制倍数 |
|------|------|----------|
| 低风险 | 正常用户 | 1x |
| 中风险 | 新用户/异常行为 | 0.5x |
| 高风险 | 可疑行为 | 0.1x |
| 封禁 | 违规用户 | 0x |

### 5.2 风险评估
| 因素 | 权重 |
|------|------|
| 账户年龄 | 20% |
| 历史行为 | 30% |
| 请求模式 | 30% |
| 异常检测 | 20% |

---

## 六、爆量保护

### 6.1 爆量检测
```
短时间内请求量突增 > 10x 平均值
```

### 6.2 保护措施
| 措施 | 触发条件 | 持续时间 |
|------|----------|----------|
| 软限制 | 5x 突增 | 1 分钟 |
| 硬限制 | 10x 突增 | 5 分钟 |
| 临时封禁 | 20x 突增 | 1 小时 |

### 6.3 爆量响应
```json
{
  "error": "burst_protection_activated",
  "message": "检测到异常流量，已触发保护",
  "cooldown": 300
}
```

---

## 七、突发额度

### 7.1 突发配额
| 计划 | 突发额度 | 说明 |
|------|----------|------|
| Free | 0 | 不支持 |
| Standard | 10% | 小幅突发 |
| Premium | 30% | 中等突发 |
| Enterprise | 50% | 大幅突发 |

### 7.2 突发恢复
```
突发使用后，配额逐步恢复
恢复速度 = 基础配额 / 恢复周期
```

---

## 八、降级策略

### 8.1 降级触发
| 触发条件 | 说明 |
|----------|------|
| 系统负载 > 80% | 资源紧张 |
| 错误率 > 5% | 服务异常 |
| 响应时间 > 5s | 性能下降 |

### 8.2 降级措施
| 措施 | 说明 |
|------|------|
| 限制新请求 | 优先处理现有请求 |
| 降低配额 | 临时降低所有限流 |
| 禁用非核心功能 | 只保留核心功能 |
| 只读模式 | 禁止写操作 |

### 8.3 降级恢复
```
负载恢复 → 逐步放开 → 监控稳定 → 完全恢复
```

---

## 九、限流响应头

### 9.1 响应头格式
```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1680844800
X-RateLimit-RetryAfter: 0
```

### 9.2 响应头说明
| 头部 | 说明 |
|------|------|
| Limit | 总限制 |
| Remaining | 剩余配额 |
| Reset | 重置时间戳 |
| RetryAfter | 重试等待秒数 |

---

## 十、限流监控

### 10.1 监控指标
| 指标 | 说明 |
|------|------|
| 限流触发次数 | 被限流的请求数 |
| 限流触发率 | 限流/总请求 |
| 平均等待时间 | Retry-After 平均值 |
| 降级触发次数 | 降级激活次数 |

### 10.2 告警规则
| 告警 | 触发条件 |
|------|----------|
| 高限流率 | 限流率 > 10% |
| 频繁降级 | 降级 > 3次/小时 |
| 异常流量 | 爆量检测触发 |
