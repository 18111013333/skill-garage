# CONFLICT_RESOLUTION.md - 冲突解决策略

## 目的
定义多agent输出冲突时的仲裁规则。

## 适用范围
所有多agent结果冲突场景。

## 冲突类型

### 事实冲突
| 类型 | 说明 | 示例 |
|------|------|------|
| 数值冲突 | 不同数值 | agentA: 25岁, agentB: 30岁 |
| 事实矛盾 | 事实矛盾 | agentA: 已完成, agentB: 未完成 |
| 时间冲突 | 时间不一致 | agentA: 2024年, agentB: 2025年 |

### 结论冲突
| 类型 | 说明 | 示例 |
|------|------|------|
| 判断冲突 | 判断不同 | agentA: 可行, agentB: 不可行 |
| 建议冲突 | 建议不同 | agentA: 方案A, agentB: 方案B |
| 评估冲突 | 评估不同 | agentA: 高风险, agentB: 低风险 |

### 格式冲突
| 类型 | 说明 | 示例 |
|------|------|------|
| 输出格式冲突 | 格式不同 | agentA: JSON, agentB: 表格 |
| 结构冲突 | 结构不同 | agentA: 3列, agentB: 5列 |
| 字段冲突 | 字段不同 | agentA: 有字段X, agentB: 无字段X |

### 优先级冲突
| 类型 | 说明 | 示例 |
|------|------|------|
| 执行顺序冲突 | 顺序不同 | agentA: 先A后B, agentB: 先B后A |
| 资源分配冲突 | 分配不同 | agentA: 50%给A, agentB: 30%给A |
| 时间分配冲突 | 分配不同 | agentA: 2小时, agentB: 4小时 |

### 工具结果冲突
| 类型 | 说明 | 示例 |
|------|------|------|
| 检索结果冲突 | 结果不同 | agentA: 结果集A, agentB: 结果集B |
| 工具输出冲突 | 输出不同 | agentA: 成功, agentB: 失败 |
| 状态报告冲突 | 状态不同 | agentA: 进行中, agentB: 已完成 |

## 仲裁规则

### 来源更强者优先
| 规则 | 说明 |
|------|------|
| 高可信来源优先 | 来源可信度高的优先 |
| 官方来源优先 | 官方来源优先于非官方 |
| 最新信息优先 | 时效性新的优先 |
| 多源一致优先 | 多来源一致的优先 |

### 校验agent优先
| 规则 | 说明 |
|------|------|
| validator结论优先 | 校验结果优先采用 |
| auditor结论优先 | 审计结果优先采用 |
| 专业agent优先 | 专业领域agent优先 |

### 主控最终裁决
| 规则 | 说明 |
|------|------|
| orchestrator最终裁决 | 主控agent有最终裁决权 |
| 用户确认优先 | 用户确认的结果优先 |
| 安全边界优先 | 安全边界不可违反 |

## 冲突处理流程

```
检测到冲突
    ↓
┌─────────────────────────────────────┐
│ 1. 分类冲突类型                      │
│    - 识别冲突类型                    │
│    - 评估冲突严重程度                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 选择仲裁规则                      │
│    - 按冲突类型选择规则              │
│    - 确定仲裁优先级                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行仲裁                          │
│    - 应用仲裁规则                    │
│    - 记录仲裁过程                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 输出仲裁结果                      │
│    - 确定最终结论                    │
│    - 标注仲裁依据                    │
└─────────────────────────────────────┘
    ↓
仲裁结果
```

## 仲裁结果格式

```json
{
  "resolution_id": "res_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "conflict_type": "fact_conflict",
  "conflicting_results": [
    {
      "agent": "retriever",
      "result": "用户年龄25岁",
      "confidence": 0.7,
      "source": "来源A"
    },
    {
      "agent": "validator",
      "result": "用户年龄30岁",
      "confidence": 0.9,
      "source": "用户明确输入"
    }
  ],
  "resolution": {
    "method": "source_priority",
    "winner": "validator",
    "final_result": "用户年龄30岁",
    "reason": "用户明确输入优先于推断"
  }
}
```

## 特殊冲突处理

### 安全相关冲突
| 处理 | 说明 |
|------|------|
| 保守处理 | 采用更安全的结论 |
| 阻断操作 | 存在安全风险时阻断 |
| 强制确认 | 需要用户确认 |

### 无法仲裁的冲突
| 处理 | 说明 |
|------|------|
| 升级主控 | 由orchestrator裁决 |
| 请求用户 | 请求用户确认 |
| 标注争议 | 标注存在争议 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 仲裁规则不适用 | 升级主控裁决 |
| 仲裁失败 | 请求用户确认 |
| 结果仍冲突 | 标注争议 |

## 维护方式
- 新增冲突类型: 添加到冲突类型表
- 新增仲裁规则: 更新仲裁规则表
- 调整优先级: 更新优先级规则
