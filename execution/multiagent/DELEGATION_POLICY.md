# DELEGATION_POLICY.md - 任务委派策略

## 目的
定义何时需要把任务委派给子agent，何时由主agent直接完成。

## 适用范围
所有任务委派决策。

## 委派决策矩阵

### 按任务复杂度决策
| 复杂度 | 判断条件 | 委派策略 |
|--------|----------|----------|
| 简单 | 单步可完成 | 主agent直接完成 |
| 中等 | 2-5步可完成 | 主agent协调，可选委派 |
| 复杂 | > 5步或需多技能 | 必须委派给子agent |
| 超复杂 | 多阶段、多依赖 | 分层委派 |

### 按任务类型决策
| 任务类型 | 委派策略 | 目标agent |
|----------|----------|------------|
| qa | 主agent直接完成 | - |
| search | 委派检索 | retriever |
| document_edit | 委派执行+写作 | executor + writer |
| planning | 委派规划 | planner |
| creative_generation | 委派写作 | writer |
| tool_use | 委派执行 | executor |
| memory_update | 主agent直接完成 | - |
| multi_step_agent | 分层委派 | 多个agent |
| high_risk_request | 委派校验+审计 | validator + auditor |

## 特殊场景委派规则

### 复杂任务委派
| 条件 | 委派规则 |
|------|----------|
| 步骤 > 5 | 必须委派 |
| 涉及多技能 | 委派对应专业agent |
| 需要外部信息 | 委派retriever |
| 需要工具调用 | 委派executor |

### 高风险任务委派
| 条件 | 委派规则 |
|------|----------|
| 风险等级 HIGH | 必须委派validator校验 |
| 风险等级 CRITICAL | 必须委派validator + auditor |
| 涉及删除操作 | 必须委派auditor审计 |
| 涉及批量操作 | 必须委派validator校验 |

### 长文档任务委派
| 条件 | 委派规则 |
|------|----------|
| 文档 > 5000字 | 委派writer分段生成 |
| 需要检索支撑 | 先委派retriever |
| 需要格式校验 | 委派validator |

### 多源检索任务委派
| 条件 | 委派规则 |
|------|----------|
| 需要多来源 | 委派retriever并行检索 |
| 来源可信度评估 | 委派validator评估 |
| 结果整合 | 主agent整合 |

### 多轮规划任务委派
| 条件 | 委派规则 |
|------|----------|
| 需要详细规划 | 委派planner |
| 规划需要调整 | 主agent协调 |
| 执行规划 | 委派executor |

## 委派前准备

### 上下文准备
| 内容 | 说明 |
|------|------|
| 任务目标 | 明确子目标 |
| 硬约束 | 必须遵守的约束 |
| 输入数据 | 相关输入数据 |
| 期望输出 | 期望的输出格式 |
| 时限要求 | 完成时限 |

### 约束传递
| 约束类型 | 传递方式 |
|----------|----------|
| 硬约束 | 必须完整传递 |
| 软约束 | 可选择性传递 |
| 安全边界 | 必须传递 |
| 预算限制 | 必须传递 |

## 委派后回收

### 结果回收
| 内容 | 说明 |
|------|------|
| 执行结果 | 子agent的输出 |
| 置信度 | 结果置信度 |
| 遇到问题 | 执行中的问题 |
| 资源消耗 | Token/时间消耗 |

### 结果验证
| 验证项 | 说明 |
|--------|------|
| 完整性 | 是否完成子目标 |
| 正确性 | 结果是否正确 |
| 约束满足 | 是否满足约束 |
| 质量评估 | 输出质量评估 |

## 委派流程

```
任务分析
    ↓
┌─────────────────────────────────────┐
│ 1. 判断是否需要委派                  │
│    - 评估复杂度                      │
│    - 评估风险                        │
│    - 评估资源需求                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 选择目标agent                     │
│    - 匹配agent能力                   │
│    - 检查agent可用性                 │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 准备委派上下文                    │
│    - 提取子目标                      │
│    - 整理约束                        │
│    - 准备输入数据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 执行委派                          │
│    - 发送委派请求                    │
│    - 等待结果                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 回收并验证结果                    │
│    - 接收结果                        │
│    - 验证结果                        │
│    - 整合结果                        │
└─────────────────────────────────────┘
```

## 异常处理

| 异常 | 处理 |
|------|------|
| agent不可用 | 尝试备选agent或主agent执行 |
| 委派超时 | 取消委派，主agent接管 |
| 结果不合格 | 要求重试或主agent修正 |

## 维护方式
- 新增委派场景: 添加到决策矩阵
- 调整委派规则: 更新委派规则表
- 新增agent: 更新目标agent表
