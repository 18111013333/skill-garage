# CONNECTOR_OBSERVABILITY.md - 连接器观测规则

## 目的
定义连接器观测指标，确保连接器状态可监控。

## 适用范围
所有连接器的运行时监控。

---

## 一、观测指标

### 1.1 核心指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 调用成功率 | 成功调用/总调用 | < 95% |
| 平均延迟 | 请求响应时间 | > 5s |
| P99 延迟 | 99 分位延迟 | > 10s |
| 错误率 | 错误/总请求 | > 5% |

### 1.2 权限指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 权限失败率 | 权限错误/总请求 | > 10% |
| Token 过期率 | Token 过期/总请求 | > 5% |
| 授权撤销率 | 授权撤销事件 | > 0 |

### 1.3 限流指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 限流触发率 | 限流/总请求 | > 5% |
| 配额使用率 | 已用/总额度 | > 80% |

## 二、外部错误监控

### 2.1 外部错误分类
| 错误类型 | 说明 | 处理 |
|----------|------|------|
| 网络错误 | 连接失败 | 重试 |
| 超时错误 | 响应超时 | 重试 |
| 认证错误 | 外部认证失败 | 通知用户 |
| 限流错误 | 外部限流 | 降级 |
| 服务错误 | 外部服务异常 | 降级 |

### 2.2 外部错误率
```
外部错误率 = 外部错误 / 总请求
告警阈值: > 20%
```

### 2.3 外部错误响应
```json
{
  "connector_id": "conn_slack",
  "error_type": "external_rate_limit",
  "external_error": {
    "code": "rate_limited",
    "message": "Slack API rate limit exceeded",
    "retry_after": 60
  }
}
```

## 三、数据同步监控

### 3.1 同步指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 同步延迟 | 数据更新滞后时间 | > 1h |
| 同步失败率 | 同步失败/总同步 | > 5% |
| 数据差异 | 本地与远程差异 | > 1% |

### 3.2 同步状态
| 状态 | 说明 |
|------|------|
| synced | 同步完成 |
| syncing | 同步中 |
| pending | 等待同步 |
| failed | 同步失败 |
| paused | 同步暂停 |

### 3.3 同步监控
```json
{
  "connector_id": "conn_google_drive",
  "sync_status": "syncing",
  "last_sync": "2026-04-07T11:00:00Z",
  "sync_lag": 3600,
  "pending_changes": 10
}
```

## 四、熔断状态

### 4.1 熔断器状态
| 状态 | 说明 | 触发条件 |
|------|------|----------|
| closed | 正常 | 错误率 < 阈值 |
| open | 熔断 | 错误率 > 阈值 |
| half-open | 探测 | 熔断后探测 |

### 4.2 熔断配置
```json
{
  "circuit_breaker": {
    "failure_threshold": 50,
    "success_threshold": 5,
    "timeout": 60,
    "half_open_requests": 3
  }
}
```

### 4.3 熔断监控
```json
{
  "connector_id": "conn_slack",
  "circuit_state": "open",
  "failure_count": 55,
  "last_failure": "2026-04-07T12:00:00Z",
  "next_retry": "2026-04-07T12:01:00Z"
}
```

## 五、监控仪表板

### 5.1 连接器健康度
```
健康度 = (成功率 × 0.4) + (延迟得分 × 0.3) + (错误率得分 × 0.3)
```

### 5.2 健康度等级
| 等级 | 分数范围 | 说明 |
|------|----------|------|
| 健康 | 90-100 | 正常运行 |
| 警告 | 70-89 | 需要关注 |
| 异常 | 50-69 | 需要处理 |
| 故障 | 0-49 | 需要紧急处理 |

### 5.3 仪表板展示
```

│ 连接器健康度仪表板                        │
├─────────────────────────────────────────┤

│ conn_mysql        █████████░ 90% 健康   │
└─────────────────────────────────────────┘
```

## 六、告警规则

### 6.1 告警级别
| 级别 | 说明 | 通知方式 |
|------|------|----------|
| info | 信息 | 日志 |
| warning | 警告 | 邮件 |
| error | 错误 | 邮件 + 短信 |
| critical | 严重 | 邮件 + 短信 + 电话 |

### 6.2 告警规则
| 规则 | 条件 | 级别 |
|------|------|------|
| 高错误率 | 错误率 > 20% | error |
| 熔断触发 | 熔断器打开 | critical |
| 同步延迟 | 延迟 > 1h | warning |
| 权限失败 | 权限失败率 > 10% | warning |

### 6.3 告警通知
```json
{
  "alert_id": "alert_xxx",
  "connector_id": "conn_slack",
  "level": "error",
  "message": "连接器错误率超过阈值",
  "metrics": {
    "error_rate": 0.25
  },
  "triggered_at": "2026-04-07T12:00:00Z"
}
```
