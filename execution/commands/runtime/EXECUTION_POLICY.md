# EXECUTION_POLICY.md - 执行策略

## 目的
定义执行边界，防止无限执行、资源耗尽、级联失败。

## 适用范围
所有任务执行过程。

## 执行边界

### 最大限制
| 指标 | 限制值 | 说明 |
|------|--------|------|
| maxSteps | 30 | 单任务最大步骤数 |
| maxToolCalls | 50 | 单任务最大工具调用次数 |
| maxRetries | 3 | 单操作最大重试次数 |
| maxParallelTasks | 5 | 最大并行任务数 |
| maxRecursionDepth | 3 | 最大递归深度 |
| singleTurnBudget | 100000 | 单轮Token预算 |
| executionTimeout | 180000 | 执行超时(ms) |

### 计数规则
- 每次工具调用计为1步
- 每次技能调用计为1步
- 每次LLM调用计为1步
- 重试不计入新步骤

## 降级触发

### 触发条件
| 条件 | 阈值 | 动作 |
|------|------|------|
| 连续失败 | ≥ 3次 | 进入降级模式 |
| 步骤超限 | > maxSteps | 终止并返回部分结果 |
| 超时 | > executionTimeout | 终止并返回部分结果 |
| 预算超限 | > singleTurnBudget | 简化输出 |
| 错误率 | > 50% | 进入安全模式 |

### 降级模式
```
正常模式
    ↓ (连续失败 ≥ 3)
降级模式
    ├── 禁用非核心技能
    ├── 禁用并行执行
    ├── 降低重试次数
    └── 简化输出格式
    ↓ (连续成功 ≥ 5)
恢复正常模式
```

### 安全模式
```
降级模式
    ↓ (错误率 > 50%)
安全模式
    ├── 仅允许核心技能
    ├── 仅允许低风险操作
    ├── 强制用户确认
    └── 记录详细日志
    ↓ (用户确认)
恢复正常模式
```

## 重试策略

### 重试规则
| 错误类型 | 重试次数 | 延迟 | 指数退避 |
|----------|----------|------|----------|
| 网络错误 | 3 | 1s | 是 |
| 超时错误 | 2 | 2s | 是 |
| 限流错误 | 3 | 5s | 否 |
| 业务错误 | 1 | 0s | 否 |
| 权限错误 | 0 | - | - |

### 重试条件
- 错误类型在可重试列表
- 重试次数未超限
- 非权限类错误
- 非业务逻辑错误

## 超时处理

### 超时分级
| 级别 | 超时时间 | 适用场景 |
|------|----------|----------|
| 快速 | 10s | 简单查询 |
| 标准 | 30s | 常规操作 |
| 长时 | 60s | 文件处理 |
| 超长 | 180s | 复杂任务 |

### 超时动作
1. 记录超时日志
2. 返回部分结果
3. 标记任务为部分完成
4. 提供继续执行选项

## 失败处理

### 失败分类
| 分类 | 处理 |
|------|------|
| 可恢复 | 重试 |
| 可降级 | 降级执行 |
| 不可恢复 | 返回错误 |

### 失败记录
```json
{
  "failureId": "fail_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "taskId": "task_001",
  "step": 5,
  "errorType": "timeout",
  "errorMessage": "Tool call exceeded 30s",
  "recoveryAction": "partial_result"
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 步骤超限 | 终止 + 部分结果 |
| 连续失败 | 降级模式 |
| 资源耗尽 | 安全模式 |
| 未知错误 | 记录 + 用户确认 |

## 维护方式
- 调整限制值: 修改对应配置项
- 新增降级条件: 更新触发条件表
- 调整重试策略: 更新重试规则表
