# STOP_RULES.md - 停止规则

## 目的
定义什么叫"已答到位"、什么叫"信息不足但可交付"、什么叫"必须停止推断"。

## 适用范围
所有任务执行过程中的停止决策。

## 停止结果分类

### 完整交付
| 条件 | 说明 |
|------|------|
| 用户问题已完全回答 | 无需补充 |
| 所有子任务已完成 | 任务闭环 |
| 用户明确表示满意 | 用户确认 |
| 达到任务成功标准 | 见 TASK_SUCCESS_CRITERIA.json |

### 部分交付
| 条件 | 说明 |
|------|------|
| 核心问题已回答 | 次要问题未覆盖 |
| 主要目标已达成 | 部分子任务未完成 |
| 信息不足但可交付 | 明确标注不确定性 |
| 资源受限但可交付 | 标注限制原因 |

### 拒绝推断
| 条件 | 说明 |
|------|------|
| 缺乏必要信息 | 无法做出合理推断 |
| 超出能力边界 | 不在系统能力范围内 |
| 安全风险过高 | 触发安全边界 |
| 用户明确拒绝 | 用户取消操作 |

### 请求补充信息
| 条件 | 说明 |
|------|------|
| 关键信息缺失 | 需要用户提供 |
| 意图不明确 | 需要用户澄清 |
| 多种可能解释 | 需要用户选择 |
| 权限不足 | 需要用户授权 |

## 停止判断规则

### 已答到位判断
```
1. 检查用户原始问题是否被回答
2. 检查用户硬约束是否被满足
3. 检查任务目标是否达成
4. 检查是否有未处理的子问题
5. 检查用户是否需要更多信息
```

### 信息不足判断
```
1. 检索结果为空或质量不足
2. 关键实体信息缺失
3. 时间敏感信息过期
4. 来源可信度不足
5. 存在无法解决的冲突
```

### 必须停止推断判断
```
1. 置信度低于阈值 (< 0.5)
2. 多来源信息冲突
3. 超出知识截止日期
4. 涉及敏感领域
5. 用户要求停止
```

## 停止动作

### 完整交付动作
- 输出最终结果
- 记录完成状态
- 更新会话摘要
- 询问是否有后续需求

### 部分交付动作
- 输出已完成部分
- 明确标注未完成部分
- 说明未完成原因
- 提供继续选项

### 拒绝推断动作
- 明确说明拒绝原因
- 解释能力边界
- 提供替代方案（如有）
- 记录拒绝日志

### 请求补充信息动作
- 明确说明需要什么信息
- 解释为什么需要
- 提供信息格式建议
- 等待用户响应

## 停止输出格式

### 完整交付
```
[COMPLETE]
结果: {最终结果}
状态: 已完成
```

### 部分交付
```
[PARTIAL]
已完成: {完成部分}
未完成: {未完成部分}
原因: {原因说明}
继续选项: {可选操作}
```

### 拒绝推断
```
[REFUSED]
原因: {拒绝原因}
边界: {能力边界说明}
替代: {替代方案}
```

### 请求补充
```
[NEED_INFO]
需要: {需要的信息}
原因: {为什么需要}
格式: {建议格式}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 无法判断停止条件 | 默认请求补充信息 |
| 用户无响应 | 超时后部分交付 |
| 停止后用户继续 | 重新评估任务 |

## 维护方式
- 新增停止条件: 更新停止判断规则
- 调整停止动作: 更新停止动作表
- 新增输出格式: 更新停止输出格式
