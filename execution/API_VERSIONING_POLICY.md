# API_VERSIONING_POLICY.md - API 版本治理规则

## 目的
定义 API 版本管理规则，确保 API 变更不会破坏外部集成。

## 适用范围
所有对外公开的 API 接口。

---

## 一、版本号规则

### 1.1 语义化版本
```
MAJOR.MINOR.PATCH

MAJOR: 不兼容的 API 变更
MINOR: 向后兼容的功能新增
PATCH: 向后兼容的问题修复
```

### 1.2 版本示例
| 版本 | 变更类型 | 说明 |
|------|----------|------|
| 1.0.0 → 1.0.1 | PATCH | 修复 bug，无接口变化 |
| 1.0.1 → 1.1.0 | MINOR | 新增可选参数，旧调用仍可用 |
| 1.1.0 → 2.0.0 | MAJOR | 删除字段，旧调用会失败 |

### 1.3 URL 版本策略
```
/v1/chat/completions  → 版本 1.x
/v2/chat/completions  → 版本 2.x
```

---

## 二、兼容性要求

### 2.1 向后兼容变更（无需升级主版本）
| 变更类型 | 示例 | 允许 |
|----------|------|------|
| 新增可选参数 | 添加 `temperature` 参数 | ✅ |
| 新增响应字段 | 响应增加 `metadata` | ✅ |
| 新增端点 | 添加 `/v1/chat/stream` | ✅ |
| 放宽验证规则 | 原必填改为可选 | ✅ |

### 2.2 破坏性变更（必须升级主版本）
| 变更类型 | 示例 | 要求 |
|----------|------|------|
| 删除端点 | 移除 `/v1/old-endpoint` | MAJOR |
| 删除参数 | 移除 `legacy_param` | MAJOR |
| 修改参数类型 | `id: int` → `id: string` | MAJOR |
| 修改响应结构 | 字段重命名 | MAJOR |
| 收紧验证规则 | 可选改为必填 | MAJOR |

## 三、废弃窗口

### 3.1 废弃流程
```
宣布废弃 → 过渡期 → 下线
   ↓          ↓        ↓
 公告通知   双版本   关闭旧版
```

### 3.2 时间窗口
| 稳定性级别 | 最低过渡期 | 公告提前量 |
|------------|------------|------------|
| stable | 12 个月 | 18 个月 |
| beta | 6 个月 | 9 个月 |
| experimental | 1 个月 | 2 个月 |

### 3.3 废弃通知内容
- 废弃日期
- 替代接口
- 迁移指南
- 影响范围

---

## 四、版本切换通知

### 4.1 通知渠道
| 渠道 | 用途 |
|------|------|
| 开发者邮件 | 重要变更通知 |
| 文档更新 | 详细迁移指南 |
| API 响应头 | 实时废弃警告 |
| Changelog | 版本历史记录 |

### 4.2 响应头警告
```http
X-API-Deprecated: true
X-API-Sunset: 2027-04-07T00:00:00Z
X-API-Replacement: /v2/chat/completions
```

### 4.3 通知时间线
```
T-18月: 首次公告
T-12月: 发布替代版本
T-6月:  强制迁移提醒
T-3月:  最终警告
T-0:    旧版本下线
```

---

## 五、强制升级条件

### 5.1 触发条件
| 条件 | 说明 |
|------|------|
| 安全漏洞 | 发现严重安全问题时 |
| 合规要求 | 法规要求变更时 |
| 依赖失效 | 底层依赖不再支持时 |
| 性能问题 | 旧版本严重影响系统时 |

### 5.2 紧急升级流程
```
发现问题 → 评估影响 → 发布补丁 → 通知受影响方 → 监控迁移
   ↓           ↓           ↓            ↓              ↓
  24h内      48h内       72h内        7天内          30天内
```

### 5.3 紧急升级通知
- 问题说明
- 风险等级
- 升级步骤
- 截止时间

---

## 六、版本共存策略

### 6.1 多版本支持
| 策略 | 说明 |
|------|------|
| 双版本运行 | 新旧版本同时可用 |
| 流量切换 | 按租户逐步切换 |
| 灰度发布 | 新版本先开放给部分租户 |

### 6.2 版本路由
```
请求 → 版本识别 → 路由到对应版本处理
        ↓
   Header/URL
```

---

## 七、版本治理检查清单

### 7.1 发布前检查
- [ ] 版本号是否正确递增
- [ ] 是否有破坏性变更
- [ ] 废弃通知是否完整
- [ ] 迁移文档是否就绪
- [ ] 测试覆盖是否完整

### 7.2 运行时检查
- [ ] 废弃警告是否生效
- [ ] 使用统计是否监控
- [ ] 迁移进度是否跟踪
