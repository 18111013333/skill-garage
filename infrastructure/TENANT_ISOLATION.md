# TENANT_ISOLATION.md - 租户隔离规则

**版本: V27.0**

## 目的
定义多租户环境下的隔离规则，确保租户之间的数据、配置、资源完全隔离，防止跨租户访问和数据泄露。

## 适用范围
所有多租户场景，包括：
- 数据存储与访问
- 记忆系统
- 配置管理
- 日志与审计
- 缓存系统
- 事件系统
- 资源配额

---

## 一、隔离原则

| 原则 | 说明 |
|------|------|
| 默认隔离 | 所有租户默认完全隔离，无例外 |
| 最小权限 | 跨租户访问需明确授权，且最小化 |
| 显式白名单 | 任何跨租户操作必须在白名单中显式声明 |
| 审计全覆盖 | 所有跨租户访问必须审计 |
| 数据不混存 | 不同租户数据不存储在同一物理位置 |

---

## 二、数据隔离

### 2.1 存储隔离

| 存储类型 | 隔离方式 | 说明 |
|----------|----------|------|
| 结构化数据 | tenant_id 分区 | 所有表必须有 tenant_id 列 |
| 非结构化数据 | 路径隔离 | `/tenant/{tenant_id}/...` |
| 向量数据 | 集合隔离 | 每租户独立 collection |
| 备份数据 | 独立备份 | 每租户独立备份集 |

### 2.2 访问隔离

```sql
-- 所有查询必须带 tenant_id 过滤
SELECT * FROM table_name 
WHERE tenant_id = :current_tenant_id 
  AND ...;

-- 禁止跨租户查询
-- ❌ SELECT * FROM table_name WHERE tenant_id IN (:tenant_list);
-- ❌ SELECT * FROM table_name; -- 无 tenant_id 过滤
```

### 2.3 数据迁移隔离

| 场景 | 隔离要求 |
|------|----------|
| 租户迁移 | 独立迁移任务，不与其他租户并发 |
| 数据导入 | 目标租户验证，源数据租户标记 |
| 数据导出 | 租户范围限定，导出审计 |

---

## 三、记忆隔离

### 3.1 记忆存储隔离

| 层级 | 隔离方式 |
|------|----------|
| L1 会话记忆 | session_id 绑定 tenant_id |
| L2 场景记忆 | scenario_id 绑定 tenant_id |
| L3 长期记忆 | memory_id 绑定 tenant_id |

### 3.2 向量索引隔离

```
# Qdrant 集合命名
collection_name = f"memories_{tenant_id}"

# SQLite-vec 表命名
table_name = f"vectors_{tenant_id}"
```

### 3.3 记忆访问规则

| 操作 | 隔离检查 |
|------|----------|
| 写入 | 验证 tenant_id 匹配 |
| 读取 | 自动注入 tenant_id 过滤 |
| 搜索 | 限定 tenant_id 范围 |
| 删除 | 验证 tenant_id 所有权 |

---

## 四、配置隔离

### 4.1 配置层级

```
全局配置 (global)
  └── 租户配置 (tenant/{tenant_id})
        └── 环境配置 (tenant/{tenant_id}/{env})
              └── 用户配置 (tenant/{tenant_id}/user/{user_id})
```

### 4.2 配置继承规则

| 规则 | 说明 |
|------|------|
| 下级覆盖上级 | 租户配置覆盖全局配置 |
| 隔离边界 | 配置不可跨租户继承 |
| 敏感配置 | 敏感配置仅租户级可见 |

### 4.3 配置访问控制

| 角色 | 全局配置 | 租户配置 | 环境配置 |
|------|----------|----------|----------|
| 系统管理员 | 读写 | 只读 | 只读 |
| 租户管理员 | 只读 | 读写 | 读写 |
| 普通用户 | 无 | 只读 | 只读 |

---

## 五、日志隔离

### 5.1 日志存储

| 日志类型 | 隔离方式 |
|----------|----------|
| 审计日志 | tenant_id 分区，独立保留策略 |
| 操作日志 | tenant_id 标记，独立查询接口 |
| 错误日志 | tenant_id 标记，敏感信息脱敏 |
| 性能日志 | tenant_id 标记，聚合时隔离 |

### 5.2 日志访问规则

| 角色 | 访问范围 |
|------|----------|
| 系统管理员 | 全部日志（脱敏） |
| 租户管理员 | 本租户日志 |
| 普通用户 | 本人操作日志 |

### 5.3 日志保留

| 租户类型 | 审计日志 | 操作日志 | 错误日志 |
|----------|----------|----------|----------|
| 个人 | 30 天 | 7 天 | 7 天 |
| 团队 | 90 天 | 30 天 | 30 天 |
| 企业 | 365 天 | 90 天 | 90 天 |

## 六、缓存隔离

### 6.1 缓存键设计

```
# 缓存键格式
cache_key = f"tenant:{tenant_id}:{resource_type}:{resource_id}"

# 示例
cache_key = "tenant:tenant_abc123:memory:mem_001"
```

### 6.2 缓存隔离规则

| 规则 | 说明 |
|------|------|
| 键前缀强制 | 所有缓存键必须带 tenant 前缀 |
| 命名空间隔离 | 每租户独立命名空间 |
| 清理隔离 | 租户缓存清理不影响其他租户 |

### 6.3 缓存配额

| 租户类型 | 缓存配额 | 驱逐策略 |
|----------|----------|----------|
| 个人 | 100 MB | LRU |
| 团队 | 1 GB | LRU |
| 企业 | 10 GB | 可配置 |

---

## 七、事件隔离

### 7.1 事件队列隔离

| 隔离方式 | 说明 |
|----------|------|
| 主题隔离 | 每租户独立事件主题 |
| 消费者隔离 | 每租户独立消费者组 |
| 死信隔离 | 每租户独立死信队列 |

### 7.2 事件路由规则

```
# 事件主题命名
topic = f"events.{tenant_id}.{event_type}"

# 示例
topic = "events.tenant_abc123.task.completed"
```

### 7.3 事件访问控制

| 操作 | 隔离检查 |
|------|----------|
| 发布 | 验证 tenant_id 匹配 |
| 订阅 | 限定 tenant_id 主题 |
| 回放 | 限定 tenant_id 范围 |

---

## 八、跨租户访问控制

### 8.1 默认规则

```
默认：禁止所有跨租户访问
```

### 8.2 白名单机制

| 白名单类型 | 审批要求 | 有效期 |
|------------|----------|--------|
| 数据共享 | 双方租户管理员 + 系统管理员 | 最长 30 天 |
| 配置继承 | 系统管理员 | 永久 |
| 事件订阅 | 双方租户管理员 | 最长 90 天 |

### 8.3 跨租户访问审计

| 审计项 | 说明 |
|--------|------|
| 访问者 | 源租户 + 用户 |
| 被访问者 | 目标租户 + 资源 |
| 访问类型 | 读/写/删除 |
| 授权依据 | 白名单 ID + 审批记录 |
| 访问结果 | 成功/失败 + 详情 |

---

## 九、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 跨租户访问尝试 | 拒绝 + 审计 + 告警 |
| 租户 ID 缺失 | 拒绝操作 + 记录错误 |
| 隔离边界突破 | 立即阻断 + 安全告警 |
| 白名单过期 | 自动失效 + 通知双方 |

---

## 十、完成标准

| 标准 | 验证方式 |
|------|----------|
| 数据隔离 | 跨租户查询返回空集 |
| 记忆隔离 | 跨租户搜索无结果 |
| 配置隔离 | 跨租户配置不可见 |
| 日志隔离 | 跨租户日志不可查 |
| 缓存隔离 | 跨租户缓存键无效 |
| 事件隔离 | 跨租户事件不可达 |
| 审计覆盖 | 所有跨租户访问有记录 |
