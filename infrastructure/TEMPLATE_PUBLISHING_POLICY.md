# TEMPLATE_PUBLISHING_POLICY.md - 模板/工作流上架规则

## 目的
定义模板与工作流上架规则，确保市场资产质量。

## 适用范围
所有发布到资产市场的模板与工作流。

---

## 一、提交流程

### 1.1 提交步骤
```
准备资产 → 填写元数据 → 提交审核 → 审核处理 → 发布上架
    ↓           ↓           ↓           ↓           ↓
  开发者      开发者       开发者      平台审核    自动发布
```

### 1.2 提交材料
| 材料 | 必需 | 说明 |
|------|------|------|
| 资产文件 | ✅ | 模板/工作流定义 |
| 元数据 | ✅ | 名称、描述、版本等 |
| 截图 | ✅ | 至少 1 张预览图 |
| 文档 | ⚠️ | 使用说明（复杂资产必需） |
| 示例 | ⚠️ | 示例数据（可选） |

### 1.3 提交入口
- Web 控制台
- API 接口
- CLI 工具

---

## 二、安全审查

### 2.1 安全检查项
| 检查项 | 说明 | 标准 |
|--------|------|------|
| 代码扫描 | 恶意代码检测 | 无恶意代码 |
| 依赖扫描 | 第三方库漏洞 | 无高危漏洞 |
| 权限检查 | 权限申请合理性 | 权限最小化 |
| 数据检查 | 数据处理合规 | 符合数据策略 |

### 2.2 安全评级
| 评级 | 说明 | 处理 |
|------|------|------|
| 安全 | 无问题 | 通过 |
| 低风险 | 小问题 | 通过 + 警告 |
| 中风险 | 需修复 | 驳回 |
| 高风险 | 严重问题 | 驳回 + 封禁 |

### 2.3 安全审查时间
| 资产类型 | 审查时间 |
|----------|----------|
| 模板 | 1-2 天 |
| 工作流 | 2-3 天 |
| 连接器包 | 3-5 天 |

---

## 三、合规审查

### 3.1 合规检查项
| 检查项 | 说明 |
|--------|------|
| 隐私合规 | PII 处理合规 |
| 数据合规 | 数据分类合规 |
| 区域合规 | 区域限制合规 |
| 许可合规 | 开源许可合规 |

### 3.2 合规标签
| 标签 | 说明 |
|------|------|
| GDPR | 符合 GDPR |
| CCPA | 符合 CCPA |
| PIPL | 符合个人信息保护法 |
| HIPAA | 符合健康数据保护 |

### 3.3 合规审查结果
| 结果 | 说明 |
|------|------|
| 合规 | 符合所有要求 |
| 部分合规 | 需要限制使用范围 |
| 不合规 | 需要修改 |

## 四、质量门槛

### 4.1 质量指标
| 指标 | 门槛 | 说明 |
|------|------|------|
| 文档完整度 | > 80% | 文档覆盖功能 |
| 示例可用性 | 100% | 示例可运行 |
| 错误处理 | > 90% | 错误场景覆盖 |
| 测试覆盖 | > 70% | 测试用例覆盖 |

### 4.2 质量评分
```
质量分 = (文档分 × 0.3) + (示例分 × 0.2) + (错误处理分 × 0.3) + (测试分 × 0.2)
```

### 4.3 质量等级
| 等级 | 分数 | 说明 |
|------|------|------|
| 优秀 | > 90 | 推荐展示 |
| 良好 | 70-90 | 正常上架 |
| 一般 | 50-70 | 限制展示 |
| 差 | < 50 | 驳回 |

---

## 五、版本管理

### 5.1 版本规则
```
MAJOR.MINOR.PATCH

MAJOR: 重大变更
MINOR: 功能新增
PATCH: 问题修复
```

### 5.2 版本更新
| 更新类型 | 审核要求 |
|----------|----------|
| PATCH | 自动通过 |
| MINOR | 快速审核 |
| MAJOR | 完整审核 |

### 5.3 版本历史
```
保留所有历史版本
支持版本回滚
显示版本变更日志
```

## 六、回滚要求

### 6.1 回滚触发
| 触发条件 | 说明 |
|----------|------|
| 严重问题 | 发现严重 bug |
| 安全问题 | 发现安全漏洞 |
| 合规问题 | 发现合规违规 |
| 用户投诉 | 大量用户投诉 |

### 6.2 回滚流程
```
发现问题 → 评估影响 → 决定回滚 → 执行回滚 → 通知用户
    ↓           ↓           ↓           ↓           ↓
  24h内      12h内       6h内        1h内        即时
```

### 6.3 回滚后处理
- 通知所有安装用户
- 提供替代方案
- 修复后重新提交

---

## 七、上架审批

### 7.1 审批流程
```
提交 → 自动检查 → 人工审核 → 审批决定 → 发布
        ↓            ↓           ↓
      1-2天        1-3天       即时
```

### 7.2 审批结果
| 结果 | 说明 | 后续 |
|------|------|------|
| 通过 | 满足所有要求 | 发布上架 |
| 有条件通过 | 需要修改 | 修改后发布 |
| 驳回 | 不满足要求 | 修改后重新提交 |

### 7.3 审批通知
- 邮件通知审批结果
- 提供详细反馈
- 支持申诉

---

## 八、下架规则

### 8.1 下架触发
| 触发条件 | 说明 |
|----------|------|
| 安全问题 | 发现安全漏洞 |
| 合规问题 | 合规违规 |
| 质量问题 | 质量不达标 |
| 作者请求 | 作者主动下架 |
| 长期不维护 | 超过 1 年未更新 |

### 8.2 下架流程
```
触发下架 → 通知作者 → 下架资产 → 通知用户
    ↓           ↓           ↓           ↓
  即时        24h内       48h内       即时
```

### 8.3 下架后处理
- 已安装用户可继续使用
- 新用户无法安装
- 提供迁移建议
