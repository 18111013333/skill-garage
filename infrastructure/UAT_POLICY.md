# UAT_POLICY.md - 终极鸽子王用户验收测试规则

## 目的
定义用户验收测试(UAT)的标准流程和通过条件。

## 适用范围
所有客户上线前的用户验收测试。

---

## 一、UAT总览

```
UAT流程
    │
    ├── 准备阶段
    │   ├── UAT计划制定
    │   ├── 测试环境准备
    │   └── 测试用例准备
    │
    ├── 执行阶段
    │   ├── 测试用例执行
    │   ├── 缺陷记录
    │   └── 日常报告
    │
    └── 收尾阶段
        ├── 缺陷修复验证
        ├── UAT报告
        └── 验收签字
```

## 四、验收角色

### 4.1 角色定义
| 角色 | 职责 | 人数 |
|------|------|------|
| UAT负责人 | 组织协调UAT | 1 |

## 五、缺陷分级

### 5.1 缺陷等级
| 等级 | 名称 | 定义 | 处理时限 |
|------|------|------|----------|
| P0 | 致命 | 系统崩溃/数据丢失/核心功能不可用 | 24小时 |
| P1 | 严重 | 主要功能异常/严重影响业务 | 48小时 |
| P2 | 一般 | 次要功能异常/有替代方案 | 72小时 |
| P3 | 轻微 | 界面问题/优化建议 | 上线后处理 |

### 5.2 缺陷处理流程
```
缺陷处理流程
    ↓
┌─────────────────────────────────────┐
│ 1. 缺陷提交                          │
│    - 描述问题                        │
│    - 提供复现步骤                    │
│    - 附上截图/日志                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 缺陷确认                          │
│    - 确认问题存在                    │
│    - 评估缺陷等级                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 缺陷修复                          │
│    - 开发修复                        │
│    - 内部验证                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 缺陷验证                          │
│    - 客户验证修复                    │
│    - 确认关闭或重新打开              │
└─────────────────────────────────────┘
```

## 六、通过标准

### 6.1 必须满足条件
| 条件 | 要求 |
|------|------|
| P0缺陷 | 0个 |
| P1缺陷 | 0个 |
| P2缺陷 | ≤ 3个（且有解决方案） |
| 测试覆盖率 | ≥ 95% |
| 测试通过率 | ≥ 95% |

### 6.2 有条件通过
| 条件 | 要求 |
|------|------|
| P0缺陷 | 0个 |
| P1缺陷 | ≤ 1个（且有临时方案） |
| P2缺陷 | ≤ 5个（且有解决计划） |
| 测试覆盖率 | ≥ 90% |
| 测试通过率 | ≥ 90% |

### 6.3 不通过条件
- 存在任何P0缺陷
- P1缺陷 > 1个
- 测试覆盖率 < 90%
- 测试通过率 < 90%

## 七、回归要求

### 7.1 回归触发条件
| 触发条件 | 回归范围 |
|----------|----------|
| P0缺陷修复 | 全量回归 |
| P1缺陷修复 | 相关模块回归 |
| P2缺陷修复 | 相关功能回归 |
| 配置变更 | 相关功能回归 |

### 7.2 回归测试清单
| 回归项 | 测试内容 | 状态 |
|--------|----------|------|
| 核心流程 | 核心业务流程验证 | ⬜ |
| 相关模块 | 受影响模块验证 | ⬜ |
| 集成场景 | 集成功能验证 | ⬜ |
| 权限控制 | 权限边界验证 | ⬜ |

---

## 八、上线阻断条件

### 8.1 阻断条件
| 条件 | 说明 |
|------|------|
| UAT未通过 | 未达到通过标准 |
| P0/P1缺陷未解决 | 存在未解决的严重缺陷 |
| 回归未完成 | 修复后回归未完成 |
| 验收未签字 | 客户未签字确认 |

### 8.2 阻断解除流程
```
阻断解除流程
    ↓
┌─────────────────────────────────────┐
│ 1. 问题解决                          │
│    - 解决阻断问题                    │
│    - 完成相关验证                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 回归测试                          │
│    - 执行回归测试                    │
│    - 确认问题已解决                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 验收确认                          │
│    - 客户确认验收                    │
│    - 签字确认                        │
└─────────────────────────────────────┘
    ↓
解除阻断
```
