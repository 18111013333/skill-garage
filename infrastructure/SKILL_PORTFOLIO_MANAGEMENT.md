# SKILL_PORTFOLIO_MANAGEMENT.md - 技能组合管理规则

**版本: V27.0**

## 目的
定义技能组合管理规则，确保 1500+ 技能不再只是数量，而是被组合管理，实现"核心稳定、高频优化、实验可控、低价值淘汰"。

## 适用范围
所有技能管理场景，包括：
- 核心技能管理
- 高频技能管理
- 实验技能管理
- 低价值技能管理
- 待淘汰技能管理

---

## 一、技能分类

| 分类 | 代码 | 说明 | 占比目标 |
|------|------|------|----------|
| 核心技能 | CORE | 平台核心能力，必须稳定 | 20% |
| 高频技能 | HIGH_FREQ | 使用频率高，持续优化 | 30% |
| 标准技能 | STANDARD | 正常使用，常规维护 | 35% |
| 实验技能 | EXPERIMENTAL | 新技能，实验验证 | 10% |
| 低价值技能 | LOW_VALUE | 使用少，价值低 | 3% |
| 待淘汰技能 | DEPRECATE | 计划淘汰 | 2% |

---

## 二、核心技能管理

### 2.1 核心技能定义

| 标准 | 说明 |
|------|------|
| 平台依赖 | 平台核心功能依赖此技能 |
| 不可替代 | 无替代方案或替代成本高 |
| 稳定要求 | 稳定性要求极高 |
| 用户依赖 | 大量用户依赖此技能 |

### 2.2 核心技能清单

| 技能领域 | 核心技能 | 说明 |
|----------|----------|------|
| 记忆 | memory_search, memory_get | 记忆检索核心 |
| 文件 | read, write, edit | 文件操作核心 |
| 执行 | exec, process | 命令执行核心 |
| 网络 | web_fetch, browser | 网络访问核心 |
| 会话 | sessions_spawn, sessions_send | 会话管理核心 |
| 安全 | secret-guardian, skill-scope | 安全核心 |

### 2.3 核心技能管理规则

| 规则 | 说明 |
|------|------|
| 变更审批 | 任何变更需技术总监审批 |
| 测试要求 | 100% 测试覆盖 |
| 回归测试 | 每次发布全量回归 |
| 监控要求 | 实时监控 + 告警 |
| 文档要求 | 完整文档 + 示例 |
| SLA 要求 | 可用性 > 99.99% |

## 四、实验技能管理

### 4.1 实验技能定义

| 标准 | 说明 |
|------|------|
| 新技能 | 新开发的技能 |
| 验证阶段 | 正在验证价值 |
| 用户有限 | 有限用户使用 |
| 不稳定 | 可能变更或废弃 |

### 4.2 实验技能管理规则

| 规则 | 说明 |
|------|------|
| 标记要求 | 必须标记为实验性 |
| 用户告知 | 使用前告知用户 |
| 监控要求 | 使用情况监控 |
| 反馈收集 | 收集用户反馈 |
| 验证周期 | 最长 3 个月验证期 |
| 决策要求 | 验证后决定升级或废弃 |

### 4.3 实验技能生命周期

```
新技能开发
    ↓
实验标记 + 有限发布
    ↓
用户使用 + 反馈收集
    ↓
效果评估（3个月内）
    ↓
┌────────────┬────────────┐
│ 升级为标准 │ 废弃       │
│ 或高频技能 │ 或降级     │
└────────────┴────────────┘
```

## 五、低价值技能管理

### 5.1 低价值技能定义

| 标准 | 说明 |
|------|------|
| 使用频率 | < 10 次/周 |
| 用户覆盖 | < 1% 用户使用 |
| 业务价值 | 低业务价值 |
| 维护成本 | 维护成本高于价值 |

### 5.2 低价值技能处理

| 处理方式 | 说明 | 条件 |
|----------|------|------|
| 观察期 | 继续观察使用情况 | 连续 1 个月低使用 |
| 降级处理 | 降低维护优先级 | 连续 3 个月低使用 |
| 合并处理 | 合并到其他技能 | 功能可被其他技能覆盖 |
| 淘汰处理 | 进入淘汰流程 | 连续 6 个月低使用 |

### 5.3 低价值技能监控

| 监控项 | 说明 | 频率 |
|--------|------|------|
| 使用频率 | 监控使用次数 | 每周 |
| 用户反馈 | 收集用户反馈 | 持续 |
| 维护成本 | 评估维护成本 | 每月 |
| 价值评估 | 综合价值评估 | 每季度 |

## 六、待淘汰技能管理

### 6.1 淘汰判定标准

| 标准 | 说明 |
|------|------|
| 使用频率 | < 1 次/月 |
| 替代方案 | 有明确替代方案 |
| 维护成本 | 维护成本高 |
| 技术债务 | 技术债务严重 |
| 安全风险 | 存在安全风险 |

### 6.2 淘汰流程

```
淘汰判定
    ↓
┌─────────────────────────────────────┐
│ 1. 影响评估                          │
│    - 评估对用户的影响                │
│    - 评估对系统的影响                │
│    - 制定迁移方案                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 公告通知                          │
│    - 发布淘汰公告                    │
│    - 通知受影响用户                  │
│    - 提供迁移指南                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 过渡期                            │
│    - 维持基本功能                    │
│    - 提供迁移支持                    │
│    - 监控迁移进度                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 正式淘汰                          │
│    - 停止服务                        │
│    - 移除代码                        │
│    - 更新文档                        │
└─────────────────────────────────────┘
```

### 6.3 淘汰时间线

| 阶段 | 时长 | 说明 |
|------|------|------|
| 公告期 | 2 周 | 发布淘汰公告 |
| 过渡期 | 1-3 月 | 提供迁移支持 |
| 警告期 | 2 周 | 使用时显示警告 |
| 停止期 | - | 正式停止服务 |
