# DEVELOPER_SUPPORT_WORKFLOW.md - 开发者支持流程

## 目的
定义开发者支持流程，确保问题得到及时处理。

## 适用范围
所有开发者支持请求。

---

## 一、支持渠道

### 1.1 支持渠道
| 渠道 | 用途 | 响应时间 |
|------|------|----------|
| 文档 | 自助查询 | 即时 |
| 社区论坛 | 问题讨论 | 24-48h |
| GitHub Issues | Bug 报告 | 24-48h |
| 邮件支持 | 一般问题 | 24h |
| 工单系统 | 企业支持 | 4h |
| 专属支持 | 企业专属 | 1h |

### 1.2 渠道选择
```
问题类型 → 推荐渠道
─────────────────────
使用问题 → 文档/社区
Bug 报告 → GitHub Issues
功能请求 → GitHub Discussions
安全问题 → security@openclaw.ai
企业问题 → 工单系统
```

---

## 二、问题受理

### 2.1 问题分类
| 类型 | 说明 | 优先级 |
|------|------|--------|
| Bug | 功能缺陷 | 高 |
| 文档问题 | 文档错误/缺失 | 中 |
| 功能请求 | 新功能需求 | 低 |
| 使用问题 | 使用咨询 | 中 |
| 安全问题 | 安全漏洞 | 紧急 |

### 2.2 问题受理流程
```
提交问题 → 分类 → 分配 → 处理 → 回复 → 关闭
    ↓         ↓       ↓       ↓       ↓       ↓
  用户      支持    开发者   开发者   支持    支持
```

### 2.3 问题模板
```markdown
## 问题描述
简要描述问题

## 复现步骤
1. 步骤一
2. 步骤二
3. 步骤三

## 期望行为
描述期望的行为

## 实际行为
描述实际的行为

## 环境信息
- SDK 版本:
- 语言版本:
- 操作系统:

## 日志/截图
粘贴相关日志或截图
```

## 三、Bug 分类

### 3.1 Bug 严重程度
| 级别 | 说明 | 响应时间 | 修复时间 |
|------|------|----------|----------|
| P0 紧急 | 核心功能不可用 | 1h | 24h |
| P1 严重 | 主要功能受影响 | 4h | 72h |
| P2 一般 | 次要功能问题 | 24h | 1 周 |
| P3 轻微 | 小问题/优化 | 48h | 下版本 |

### 3.2 Bug 处理流程
```
确认 Bug → 评估影响 → 分配开发者 → 修复 → 测试 → 发布
    ↓           ↓           ↓           ↓       ↓       ↓
  支持        开发        开发        开发     QA      发布
```

### 3.3 Bug 状态
| 状态 | 说明 |
|------|------|
| New | 新提交 |
| Confirmed | 已确认 |
| In Progress | 处理中 |
| Fixed | 已修复 |
| Verified | 已验证 |
| Closed | 已关闭 |
