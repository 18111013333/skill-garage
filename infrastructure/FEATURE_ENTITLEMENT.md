# FEATURE_ENTITLEMENT.md - 功能权益规则

## 目的
定义功能权益规则，明确不同套餐的功能权限。

## 适用范围
所有平台功能、工具、服务的权益判定。

## 额外开通功能

### 可单独购买功能
| 功能 | 说明 | 计费方式 |
|------|------|----------|
| 高级模型包 | 特殊大模型访问 | 按 Token 计费 |
| 扩展存储 | 额外存储空间 | 按 GB/月 |
| API 扩展包 | 更高 API 限额 | 按调用次数 |
| 专属支持 | 专属客户经理 | 按月订阅 |

### 开通流程
```
需求确认 → 套餐检查 → 购买确认 → 权益开通 → 配置生效 → 通知用户
```

## 权益检查

### 检查时机
| 时机 | 说明 |
|------|------|
| 功能访问 | 访问功能时检查 |
| API 调用 | API 调用时检查 |
| 工具执行 | 工具执行时检查 |
| 配置修改 | 配置修改时检查 |

### 检查逻辑
```json
{
  "entitlement_check": {
    "tenant_id": "tenant-001",
    "feature_id": "ff_advanced_models",
    "check_result": {
      "entitled": true,
      "source": "plan_professional",
      "remaining_quota": 5000000,
      "restrictions": []
    }
  }
}
```

### 无权益处理
| 处理方式 | 说明 |
|----------|------|
| 提示升级 | 提示用户升级套餐 |
| 引导购买 | 引导购买额外功能 |
| 限制访问 | 限制功能访问 |
| 记录日志 | 记录无权益访问尝试 |

## 权益变更

### 升级变更
| 变更 | 处理 |
|------|------|
| 套餐升级 | 立即获得新功能 |
| 功能购买 | 购买后立即生效 |
| 配额增加 | 立即增加配额 |

### 降级变更
| 变更 | 处理 |
|------|------|
| 套餐降级 | 下周期生效 |
| 功能取消 | 下周期失效 |
| 配额减少 | 下周期生效 |

### 过渡期处理
| 情况 | 处理 |
|------|------|
| 数据超限 | 保留数据，限制访问 |
| 功能使用中 | 允许完成，禁止新任务 |
| 配置依赖 | 提示用户处理 |
