# 第六阶段交付报告 - 数据化与自优化升级 V2.8.0

## 一、指标体系说明

### 统计指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 任务完成率 | 成功完成的任务比例 | 完成数 / 总数 |
| 输出合格率 | 质量达标的输出比例 | 合格数 / 总数 |
| 产物可直接使用率 | 无需修改即可使用的比例 | 可用数 / 总数 |
| Fallback 触发率 | 触发回退的比例 | 触发数 / 总数 |
| 平均耗时 | 任务平均执行时间 | 总耗时 / 总数 |
| 人工二次修改率 | 需要人工修改的比例 | 修改数 / 总数 |
| 用户采纳率 | 用户直接采用的比例 | 采纳数 / 总数 |
| 失败类型分布 | 各类失败的数量分布 | 按类型统计 |

### 统计维度

- 按任务类型
- 按工作流
- 按项目
- 按时间周期（今日/本周/本月）

## 二、反馈学习规则

### 反馈类型

| 类型 | 说明 | 学习动作 |
|------|------|----------|
| adopted | 直接采用 | 更新模板偏好 |
| rewritten | 被重写 | 降低工作流优先级 |
| clarification | 需要补问 | 增加前置确认 |
| rework | 需要返工 | 标记问题工作流 |
| preferred | 用户偏好 | 记录偏好设置 |

### 学习闭环

```
反馈记录 → 统计分析 → 生成规则 → 应用到：
  - 路由策略
  - 输出模板
  - 工作流顺序
  - 校验规则
```

### 规则置信度

- 初始置信度: 0.5
- 每次强化: +0.1
- 最高置信度: 1.0

## 三、工作流优选机制说明

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 成功率 | 25% | 任务成功完成的比例 |
| 质量分 | 25% | 输出质量评分 |
| 稳定性 | 15% | 执行稳定性 |
| 耗时 | 10% | 执行效率（越短越好） |
| Fallback率 | 10% | 回退频率（越低越好） |
| 采纳率 | 15% | 用户采纳比例 |

### 选择流程

1. 获取候选工作流列表
2. 计算各工作流历史评分
3. 按综合分排序
4. 返回最佳工作流 + 解释

### 回退链

当首选工作流失败时，按排名顺序尝试备选工作流。

## 四、多项目调度规则

### 项目分类

| 维度 | 选项 |
|------|------|
| 优先级 | 主项目 / 次项目 |
| 紧急度 | 紧急 / 正常 / 低 |
| 执行状态 | 就绪 / 阻塞 / 等待 |
| 类型 | 长期推进 / 一次性任务 |

### 调度分数计算

```
分数 = 优先级权重 + 紧急度权重 + 执行状态权重 - 阻塞惩罚 + 类型加分
```

| 因素 | 分数 |
|------|------|
| 主项目 | +30 |
| 次项目 | +10 |
| 紧急 | +30 |
| 正常 | +15 |
| 低 | +5 |
| 就绪 | +20 |
| 等待 | +10 |
| 阻塞 | +0 |
| 每个阻塞项 | -5 |
| 长期项目 | +10 |

### 输出

- 当前最优先项目
- 阻塞项目列表
- 可顺手推进任务

## 五、半自动推进边界说明

### 自动化级别

| 级别 | 说明 |
|------|------|
| full_auto | 全自动，高风险动作也自动执行 |
| guided | 引导式，低风险自动，高风险需确认 |
| manual | 手动，所有动作需确认 |

### 可自动处理动作

| 动作 | 默认自动 | 说明 |
|------|----------|------|
| 下一步建议生成 | ✅ | 低风险 |
| 待办列表更新 | ✅ | 低风险 |
| 工作流自动选择 | ✅ | 低风险 |
| 产物自动归档 | ✅ | 低风险 |
| 阻塞项自动标记 | ❌ | 需确认 |
| 阶段总结自动生成 | ✅ | 低风险 |
| 高风险动作 | ❌ | 始终需确认 |

### 边界控制

- 高风险动作始终需确认
- 自动化边界可配置
- 每步可追踪、可解释、可中止
