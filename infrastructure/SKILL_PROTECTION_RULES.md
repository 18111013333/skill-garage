# SKILL_PROTECTION_RULES.md - 技能保护规则

**版本: V28.0**

## 目的
定义技能保护机制，确保关键技能和孵化期技能不会被误删或误降级。

## 适用范围
所有技能，特别是孵化期技能和关键低频技能。

---

## 一、保护类型

| 类型 | 代码 | 说明 | 保护级别 |
|------|------|------|----------|
| 孵化保护 | INCUBATION | 孵化期技能保护 | 高 |
| 关键保护 | CRITICAL | 关键技能保护 | 最高 |
| 黄金路径保护 | GOLDEN | 黄金路径绑定技能保护 | 高 |
| 合规保护 | COMPLIANCE | 合规相关技能保护 | 最高 |
| 灾备保护 | DISASTER | 灾备相关技能保护 | 最高 |

---

## 二、孵化期保护规则

### 2.1 保护期定义

| 阶段 | 保护期 | 保护内容 |
|------|--------|----------|
| incubating | 30天 | 禁止删除、禁止降级 |
| candidate | 30天 | 禁止删除、降级需评审 |
| beta | 45天 | 禁止删除、降级需评审 |

### 2.2 孵化期删除条件

孵化期技能只有在以下条件**全部满足**时才能删除：

| 条件 | 说明 |
|------|------|
| 完整周期 | 已完成至少一个完整孵化周期 |
| 受控评测 | 已做过至少一次受控评测 |
| 无增益 | 没有显示出任何明确增益 |
| 无Owner | 没有 Owner 愿意继续负责 |
| 无场景 | 没有明确接入场景 |
| 有替代 | 有替代能力可以覆盖其作用 |

### 2.3 孵化期保护配置

```json
{
  "skill_id": "SKILL-XXX",
  "protection": {
    "type": "INCUBATION",
    "start_date": "2026-04-08",
    "end_date": "2026-05-08",
    "stage": "incubating",
    "delete_allowed": false,
    "downgrade_allowed": false,
    "review_required": true
  }
}
```

## 三、关键低频技能保护

### 3.1 关键低频技能定义

| 类型 | 说明 | 示例 |
|------|------|------|
| 审计技能 | 审计相关技能 | audit-log-export |
| 合规技能 | 合规相关技能 | compliance-check |
| 灾备技能 | 灾备相关技能 | disaster-recovery |
| 重大事故技能 | 重大事故处理 | incident-commander |
| RFP技能 | RFP特殊场景 | rfp-response |

### 3.2 关键低频技能标记

```json
{
  "skill_id": "SKILL-XXX",
  "tags": ["critical_but_low_frequency"],
  "protection": {
    "type": "CRITICAL",
    "delete_allowed": false,
    "downgrade_allowed": false,
    "review_required": true,
    "review_level": "CTO"
  }
}
```

### 3.3 关键低频技能删除条件

| 条件 | 说明 |
|------|------|
| 功能确认 | 功能确实不再需要 |
| 合规确认 | 合规团队确认 |
| 替代确认 | 有替代方案 |
| CTO审批 | CTO级别审批 |

## 四、黄金路径保护

### 4.1 黄金路径绑定技能

| 黄金路径 | 核心技能 |
|----------|----------|
| 项目拆解 | planning, milestone-tracker |
| 会议跟进 | meeting-processor, action-extractor |
| 知识检索 | search, evidence-binder |
| 事故处理 | incident-responder, rollback-manager |
| 租户接入 | tenant-onboarding, permission-manager |

| 伙伴交付 | partner-delivery, uat-manager |

| 问题学习 | feedback-processor, playbook-updater |
| ROI报告 | cost-analyzer, risk-summarizer |

### 4.2 黄金路径保护规则

| 规则 | 说明 |
|------|------|
| 禁止删除 | 黄金路径绑定技能禁止删除 |
| 降级需审批 | 降级需要技术负责人审批 |
| 替代必须 | 删除前必须有替代技能 |
| 路由迁移 | 删除前必须完成路由迁移 |

## 五、保护解除条件

### 5.1 孵化保护解除

| 条件 | 说明 |
|------|------|
| 晋升为ga | 成功晋升为正式可用 |
| 孵化失败 | 确认孵化失败并废弃 |
| 保护期结束 | 保护期结束且未晋升 |

### 5.2 关键保护解除

| 条件 | 说明 |
|------|------|
| 功能确认 | 功能确实不再需要 |
| 合规确认 | 合规团队确认 |
| 替代确认 | 有替代方案 |
| 审批通过 | 相应级别审批通过 |

## 六、保护检查流程

```
删除/降级请求
    ↓
┌─────────────────────────────────────┐
│ 1. 保护状态检查                      │
│    - 检查是否有保护标记              │
│    - 检查保护类型                    │
│    - 检查保护期限                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 保护规则匹配                      │
│    - 匹配对应保护规则                │
│    - 检查是否允许操作                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 审批流程                          │
│    - 如需审批，启动审批流程          │
│    - 等待审批结果                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 执行/拒绝                         │
│    - 通过：执行操作                  │
│    - 拒绝：拒绝操作                  │
└─────────────────────────────────────┘
```
