## 三、密钥处理

### 3.1 密钥处理原则
```
示例代码中不包含真实密钥
使用环境变量或配置文件
提供密钥获取指南
```

### 3.2 密钥配置方式
```python
# 方式 1: 环境变量
import os
api_key = os.environ.get("OPENCLAW_API_KEY")

# 方式 2: 配置文件
# config.yaml (不提交到版本控制)
api_key: "your-api-key-here"

# 方式 3: 命令行参数
parser.add_argument("--api-key", help="OpenClaw API Key")
```

### 3.3 密钥安全提示
```markdown
⚠️ 安全提示:
- 不要在代码中硬编码 API Key
- 不要将 API Key 提交到版本控制
- 使用环境变量或配置文件
- 定期轮换 API Key
```

## 四、假数据处理

### 4.1 假数据原则
```
示例使用假数据，不使用真实数据
假数据格式与真实数据一致
假数据不包含敏感信息
```

### 4.2 假数据示例
```python
# 假用户数据
fake_users = [
    {"id": "user_001", "name": "张三", "email": "zhangsan@example.com"},
    {"id": "user_002", "name": "李四", "email": "lisi@example.com"},
]

# 假消息数据
fake_messages = [
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮助你的？"},
]
```

### 4.3 假数据生成工具
```python
# 使用 Faker 库生成假数据
from faker import Faker
fake = Faker('zh_CN')

user = {
    "name": fake.name(),
    "email": fake.email(),
    "phone": fake.phone_number(),
}
```

## 五、安全最佳实践

### 5.1 示例中禁止的做法
| 禁止做法 | 原因 |
|----------|------|
| 硬编码密钥 | 安全风险 |
| 禁用 SSL 验证 | 安全风险 |
| 明文存储密码 | 安全风险 |
| 过度权限申请 | 安全风险 |
| 忽略错误处理 | 不推荐做法 |

### 5.2 示例中推荐的做法
| 推荐做法 | 说明 |
|----------|------|
| 使用环境变量 | 安全存储密钥 |
| 启用 SSL 验证 | 安全传输 |
| 错误处理 | 健壮性 |
| 日志记录 | 可调试 |
| 最小权限 | 安全原则 |

### 5.3 安全检查清单
- [ ] 无硬编码密钥
- [ ] 无禁用 SSL
- [ ] 无明文密码
- [ ] 有错误处理
- [ ] 有安全提示

## 六、示例应用结构

### 6.1 目录结构
```
sample-app/
├── README.md           # 说明文档
├── requirements.txt    # 依赖列表
├── config.example.yaml # 配置示例
├── .env.example        # 环境变量示例
├── src/
│   ├── main.py         # 主程序
│   └── utils.py        # 工具函数

    └── test_main.py    # 测试文件
```

### 6.2 README 内容
```markdown
# 示例应用名称

## 功能说明
简要说明示例功能

- OpenClaw API Key

pip install -r requirements.txt

cp .env.example .env
```
