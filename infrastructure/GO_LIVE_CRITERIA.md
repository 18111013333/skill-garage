# GO_LIVE_CRITERIA.md - 正式上线门槛

## 目的
定义系统正式上线的必要条件，确保上线受控。

---

## 上线门槛清单

### 1. 测试必须通过

| 测试类型 | 通过标准 | 验证方式 |
|----------|----------|----------|
| 功能测试 | 100%用例通过 | 测试报告 |
| 性能测试 | P99 < 500ms | 性能报告 |
| 安全测试 | 无高危漏洞 | 安全报告 |
| 集成测试 | 所有接口正常 | 集成报告 |
| UAT测试 | 客户签字确认 | UAT报告 |

### 2. 高风险能力默认关闭

| 能力 | 默认状态 | 开启条件 |
|------|----------|----------|
| 自动执行 | 关闭 | 客户书面确认 |
| 批量操作 | 关闭 | 审批流程就绪 |
| 外部集成 | 关闭 | 安全评估通过 |
| 数据导出 | 关闭 | 合规审批通过 |
| API开放 | 关闭 | 安全审计通过 |

### 3. 审批流必须配置

| 审批类型 | 必须配置 | 验证方式 |
|----------|----------|----------|
| 高风险操作 | ✅ | 配置检查 |
| 数据变更 | ✅ | 配置检查 |
| 权限变更 | ✅ | 配置检查 |
| 外部访问 | ✅ | 配置检查 |
| 批量操作 | ✅ | 配置检查 |

### 4. 知识库必须完成

| 知识类型 | 完成标准 | 验证方式 |
|----------|----------|----------|
| 产品知识 | 覆盖核心功能 | 知识检查 |
| 流程知识 | 覆盖关键流程 | 知识检查 |
| FAQ知识 | 覆盖常见问题 | 知识检查 |
| 集成知识 | 覆盖已配置集成 | 知识检查 |

### 5. 连接器必须就绪

| 连接器类型 | 就绪标准 | 验证方式 |
|----------|----------|----------|
| 核心连接器 | 连接测试通过 | 连接测试 |
| 数据连接器 | 数据同步正常 | 同步验证 |
| 认证连接器 | 认证流程正常 | 认证测试 |
| 通知连接器 | 通知发送正常 | 通知测试 |

### 6. 角色矩阵必须完成

| 角色类型 | 完成标准 | 验证方式 |
|----------|----------|----------|
| 管理员角色 | 权限配置完成 | 权限检查 |
| 业务角色 | 权限配置完成 | 权限检查 |
| 审批角色 | 审批流配置完成 | 流程检查 |
| 集成角色 | 接口权限配置 | 权限检查 |

### 7. 告警必须就绪

| 告警类型 | 就绪标准 | 验证方式 |
|----------|----------|----------|
| 系统告警 | 告警规则配置 | 规则检查 |
| 业务告警 | 告警规则配置 | 规则检查 |
| 安全告警 | 告警规则配置 | 规则检查 |
| 通知渠道 | 通知配置完成 | 通知测试 |

## 上线审批流程

```
上线申请
    ↓
┌─────────────────────────────────────┐
│ 技术评审                             │
│ - 测试报告审核                       │
│ - 安全报告审核                       │
│ - 性能报告审核                       │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 业务评审                             │
│ - UAT确认审核                        │
│ - 培训完成确认                       │
│ - 文档完成确认                       │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 客户确认                             │
│ - 上线时间确认                       │
│ - 应急预案确认                       │
│ - 联系人确认                         │
└─────────────────────────────────────┘
    ↓
上线执行
```

## 上线阻断条件

| 条件 | 阻断级别 |
|------|----------|
| 测试未通过 | 🔴 强制阻断 |
| 安全高危漏洞 | 🔴 强制阻断 |
| 审批流未配置 | 🔴 强制阻断 |
| 知识库未完成 | 🟡 条件阻断 |
| 告警未就绪 | 🟡 条件阻断 |
| 客户未确认 | 🔴 强制阻断 |
