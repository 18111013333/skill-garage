# PRODUCTION_PARITY_POLICY.md - 终极鸽子王预演环境一致性规则

## 目的
确保预演环境与生产环境的一致性，保证测试结果可靠。

## 适用范围
所有客户沙箱环境、UAT环境、预演环境。

---

## 一、一致性要求

### 1.1 必须一致项
| 项目 | 说明 | 验证方式 |
|------|------|----------|
| 系统版本 | 与生产版本一致 | 版本检查 |
| 功能配置 | 核心功能配置一致 | 配置对比 |
| 角色权限 | 权限模型一致 | 权限对比 |
| 业务规则 | 业务规则一致 | 规则对比 |
| 集成配置 | 集成方式一致 | 配置对比 |

### 1.2 可有差异项
| 项目 | 差异说明 | 限制 |
|------|----------|------|
| 数据量 | 可使用较少数据 | 需覆盖关键场景 |
| 用户数 | 可使用较少用户 | 需覆盖关键角色 |
| 性能配置 | 可降低配置 | 需满足测试需求 |
| 外部连接 | 可使用Mock | 需模拟真实行为 |

---

## 二、必须受限项

### 2.1 功能限制
| 功能 | 限制说明 | 原因 |
|------|----------|------|
| 批量删除 | 禁用或限制数量 | 防止误操作 |
| 批量导出 | 禁用或限制数量 | 数据安全 |
| 外部通知 | 禁用或Mock | 防止误发 |
| 支付功能 | 禁用或Mock | 防止真实交易 |
| 第三方集成 | Mock模式 | 防止影响生产 |

### 2.2 数据限制
| 数据类型 | 限制说明 |
|----------|----------|
| 生产数据 | 禁止直接使用 |
| PII数据 | 必须脱敏 |
| 敏感数据 | 禁止导入 |

---

## 三、高风险能力模拟

### 3.1 模拟执行
| 能力 | 模拟方式 | 说明 |
|------|----------|------|
| 批量操作 | 记录但不执行 | 验证流程正确性 |
| 外部调用 | 返回模拟响应 | 验证集成逻辑 |
| 通知发送 | 记录但不发送 | 验证通知逻辑 |
| 定时任务 | 手动触发 | 验证任务逻辑 |

### 3.2 模拟配置
```json
{
  "simulation_config": {
    "batch_operations": {
      "mode": "dry_run",
      "log_only": true,
      "max_simulation_count": 100
    },
    "external_calls": {
      "mode": "mock",
      "response_delay_ms": 100,
      "mock_responses": {}
    },
    "notifications": {
      "mode": "capture",
      "capture_to_log": true,
      "real_send": false
    }
  }
}
```

## 四、配置漂移检查

### 4.1 检查频率
| 检查项 | 频率 | 告警条件 |
|--------|------|----------|
| 版本一致性 | 每次部署 | 版本不一致 |
| 配置一致性 | 每天 | 关键配置不一致 |
| 权限一致性 | 每周 | 权限模型不一致 |

### 4.2 漂移处理
```
漂移检测流程
    ↓
┌─────────────────────────────────────┐
│ 1. 漂移检测                          │
│    - 自动对比配置                    │
│    - 标记差异项                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 漂移评估                          │
│    - 评估影响程度                    │
│    - 确定处理优先级                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 漂移修复                          │
│    - 同步生产配置                    │
│    - 或说明差异原因                  │
└─────────────────────────────────────┘
```
