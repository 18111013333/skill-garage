# CANARY_POLICY.md - 灰度发布策略

## 目的
定义灰度发布策略，确保大改动不能直接全量上线。

## 适用范围
所有需要灰度发布的系统变更。

## 必须走Canary的改动

### 核心规则变更
| 改动类型 | 说明 | 灰度要求 |
|----------|------|----------|
| ORCHESTRATOR.md | 总调度规则 | 必须 |
| TASK_CLASSIFIER.json | 任务分类规则 | 必须 |
| EXECUTION_POLICY.md | 执行策略 | 必须 |
| STOP_RULES.md | 停止规则 | 必须 |
| FAILOVER.md | 故障恢复策略 | 必须 |

### 路由变更
| 改动类型 | 说明 | 灰度要求 |
|----------|------|----------|
| SKILL_ROUTER.json | 技能路由 | 必须 |
| WORKFLOW_REGISTRY.json | 工作流注册 | 必须 |

### 记忆Schema变更
| 改动类型 | 说明 | 灰度要求 |
|----------|------|----------|
| MEMORY_SCHEMA.json | 记忆数据结构 | 必须 |
| USER_PROFILE_SCHEMA.json | 用户画像结构 | 必须 |
| SESSION_SUMMARY_SCHEMA.json | 会话摘要结构 | 必须 |

### 工具护栏变更
| 改动类型 | 说明 | 灰度要求 |
|----------|------|----------|
| TOOL_GUARDRAILS.json | 工具护栏配置 | 必须 |
| BOUNDARY.json | 安全边界配置 | 必须 |
| RISK_POLICY.md | 风险策略 | 必须 |

### 评测门禁变更
| 改动类型 | 说明 | 灰度要求 |
|----------|------|----------|
| REGRESSION_GATE.json | 回归门禁 | 必须 |
| core_benchmark.json | 核心测试集 | 必须 |
| safety_benchmark.json | 安全测试集 | 必须 |

## Canary观察指标

### 核心指标
| 指标 | 说明 | 回滚阈值 |
|------|------|----------|
| 错误率 | 请求错误率 | > 5% |
| 响应时间 | 平均响应时间 | > 200%基线 |
| 成功率 | 任务成功率 | < 90% |
| 用户投诉 | 用户投诉数 | > 5次/小时 |

### 质量指标
| 指标 | 说明 | 回滚阈值 |
|------|------|----------|
| 幻觉率 | 幻觉发生率 | > 10% |
| 约束违反 | 硬约束违反数 | > 0 |
| 安全违规 | 安全边界违规 | > 0 |

## Canary发布流程

```
变更准备
    ↓
┌─────────────────────────────────────┐
│ 1. 通过回归门禁                      │
│    - 运行所有测试集                  │
│    - 确保通过门禁阈值                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 配置灰度                          │
│    - 设置灰度比例                    │
│    - 选择灰度用户群                  │
│    - 配置回滚阈值                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行灰度发布                      │
│    - 阶段1: 5% 流量                  │
│    - 阶段2: 20% 流量                 │
│    - 阶段3: 50% 流量                 │
│    - 阶段4: 100% 流量                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 监控与回滚                        │
│    - 实时监控指标                    │
│    - 触发阈值自动回滚                │
└─────────────────────────────────────┘
    ↓
全量发布 / 回滚
```

## 灰度阶段配置

### 阶段配置
| 阶段 | 流量比例 | 持续时间 | 通过条件 |
|------|----------|----------|----------|
| 阶段1 | 5% | 2天 | 错误率 < 5% |
| 阶段2 | 20% | 2天 | 错误率 < 3% |
| 阶段3 | 50% | 2天 | 错误率 < 2% |
| 阶段4 | 100% | - | 正式上线 |

### 回滚条件
| 条件 | 阈值 | 动作 |
|------|------|------|
| 错误率 | > 5% | 立即回滚 |
| 响应时间 | > 200%基线 | 立即回滚 |
| 安全违规 | > 0 | 立即回滚 |
| 用户投诉 | > 5次/小时 | 评估回滚 |

## Canary配置格式

```json
{
  "canaryId": "canary_001",
  "change": {
    "type": "skill_router",
    "files": ["runtime/SKILL_ROUTER.json"],
    "version": "1.1.0"
  },
  "stages": [
    {
      "stage": 1,
      "percentage": 5,
      "duration": "2d",
      "thresholds": {
        "errorRate": 0.05,
        "responseTime": 2.0
      }
    }
  ],
  "rollbackThresholds": {
    "errorRate": 0.05,
    "responseTimeMultiplier": 2.0,
    "securityViolation": 0
  },
  "startDate": "2026-04-06",
  "endDate": "2026-04-12"
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 灰度失败 | 自动回滚 |
| 指标异常 | 暂停灰度，评估 |
| 用户反馈 | 评估后决定 |

## 维护方式
- 新增必须灰度类型: 添加到必须走Canary表
- 调整灰度阶段: 更新阶段配置
- 调整回滚阈值: 更新回滚条件表
