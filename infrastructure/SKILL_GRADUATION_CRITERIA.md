# SKILL_GRADUATION_CRITERIA.md - 技能毕业标准

**版本: V28.0**

## 目的
定义技能从孵化到正式可用的毕业标准，确保只有合格的技能才能进入生产环境。

## 适用范围
所有从 beta 晋升到 ga 的技能。

---

## 一、毕业条件总览

| 维度 | 最低要求 | 推荐标准 | 权重 |
|------|----------|----------|------|
| 功能完整性 | 90% | 95% | 25% |
| 稳定性 | 90% | 95% | 20% |
| 使用验证 | 10次真实调用 | 50次真实调用 | 15% |
| 用户反馈 | 3条正面反馈 | 10条正面反馈 | 15% |
| 文档完整性 | 80% | 95% | 10% |
| 测试覆盖 | 60% | 80% | 10% |
| 监控指标 | 基础监控 | 完整监控 | 5% |

---

## 三、稳定性标准

### 3.1 稳定性指标

| 指标 | 最低要求 | 推荐标准 |
|------|----------|----------|
| 成功率 | > 90% | > 95% |
| 错误率 | < 5% | < 2% |
| 超时率 | < 3% | < 1% |
| 平均响应时间 | < 5s | < 2s |
| P99响应时间 | < 10s | < 5s |

### 3.2 稳定性测试

| 测试类型 | 要求 | 通过标准 |
|----------|------|----------|
| 压力测试 | 10x正常负载 | 无崩溃 |
| 并发测试 | 100并发 | 无死锁 |
| 长时间测试 | 24小时运行 | 无内存泄漏 |
| 异常输入测试 | 100种异常输入 | 正确处理 |

---

## 四、使用验证标准

### 4.1 真实调用要求

| 阶段 | 最少调用次数 | 最少用户数 | 最少场景数 |
|------|--------------|------------|------------|
| beta | 10 | 3 | 2 |
| ga候选 | 50 | 10 | 5 |
| core候选 | 100 | 20 | 10 |

### 4.2 使用场景覆盖

| 场景类型 | 要求 |
|----------|------|
| 黄金路径场景 | 至少覆盖1条黄金路径 |
| 边界场景 | 至少覆盖3种边界场景 |
| 异常场景 | 至少覆盖5种异常场景 |

---

## 五、用户反馈标准

### 5.1 反馈收集

| 渠道 | 说明 |
|------|------|
| 显式反馈 | 用户主动评价 |
| 隐式反馈 | 使用行为分析 |
| 问题报告 | 问题工单统计 |

### 5.2 反馈指标

| 指标 | 最低要求 | 推荐标准 |
|------|----------|----------|
| 满意度 | > 3.5/5.0 | > 4.0/5.0 |
| 推荐率 | > 60% | > 80% |
| 问题率 | < 10% | < 5% |
| 复用率 | > 30% | > 50% |

## 六、文档完整性标准

### 6.1 必需文档

| 文档 | 要求 | 检查项 |
|------|------|--------|
| README.md | 必需 | 功能说明、使用方法、示例 |
| API文档 | 必需 | 参数说明、返回格式、错误码 |
| 配置文档 | 可选 | 配置项说明 |
| 最佳实践 | 推荐 | 使用建议、注意事项 |
| 故障排查 | 推荐 | 常见问题、解决方案 |

## 八、毕业评审流程

```
毕业申请
    ↓
┌─────────────────────────────────────┐
│ 1. 材料准备                          │
│    - 功能测试报告                    │
│    - 稳定性测试报告                  │
│    - 使用数据报告                    │
│    - 用户反馈报告                    │
│    - 文档检查报告                    │
│    - 测试覆盖报告                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 评审会议                          │
│    - 技能Owner陈述                   │
│    - 评审人提问                      │
│    - 投票表决                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 决策                              │
│    - 通过：晋升为ga                  │
│    - 有条件通过：修复后晋升          │
│    - 不通过：继续beta                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 状态更新                          │
│    - 更新成熟度状态                  │
│    - 开放常规路由                    │
│    - 通知相关方                      │
└─────────────────────────────────────┘
```
