# SKILL_DEPRECATION_POLICY.md - 技能淘汰规则

**版本: V27.0**

## 目的
定义技能淘汰规则，确保技能膨胀得到治理，实现"淘汰有标准、过程有通知、影响有评估"。

## 适用范围
所有技能淘汰场景，包括：
- 技能下线
- 技能合并
- 技能替换
- 技能废弃

---

## 一、淘汰原则

| 原则 | 说明 |
|------|------|
| 价值导向 | 无价值或低价值技能优先淘汰 |
| 用户优先 | 淘汰前确保用户有替代方案 |
| 渐进淘汰 | 分阶段淘汰，给用户适应时间 |
| 可回滚 | 淘汰后可恢复（一定时间内） |
| 文档完整 | 淘汰过程文档完整 |

---

## 二、淘汰判定标准

### 2.1 使用量标准

| 指标 | 淘汰阈值 | 说明 |
|------|----------|------|
| 日调用量 | < 1 次/天 | 极低使用 |
| 周调用量 | < 10 次/周 | 低使用 |
| 月调用量 | < 50 次/月 | 持续低使用 |
| 使用趋势 | 连续下降 > 50% | 快速下降 |

### 2.2 质量标准

| 指标 | 淘汰阈值 | 说明 |
|------|----------|------|
| 成功率 | < 80% | 质量问题严重 |
| 错误率 | > 20% | 错误率过高 |
| 用户满意度 | < 60% | 用户不满意 |
| Bug 数量 | > 10 个未修复 | 维护困难 |

### 2.3 价值标准

| 指标 | 淘汰阈值 | 说明 |
|------|----------|------|
| 业务价值 | 低或无 | 无业务价值 |
| 战略重要性 | 低 | 非战略技能 |
| 用户覆盖 | < 1% 用户 | 极少用户使用 |
| 替代方案 | 有更好替代 | 有替代方案 |

### 2.4 技术标准

| 指标 | 淘汰阈值 | 说明 |
|------|----------|------|
| 技术债务 | 严重 | 技术债务严重 |
| 安全风险 | 高 | 存在安全风险 |
| 依赖风险 | 高 | 依赖不可用 |
| 维护成本 | 高于价值 | 维护成本过高 |

---

## 三、淘汰类型

### 3.1 下线淘汰

| 项目 | 说明 |
|------|------|
| 定义 | 完全停止服务 |
| 条件 | 无用户使用或可完全替代 |
| 流程 | 公告 → 过渡 → 下线 |
| 恢复 | 30天内可恢复 |

### 3.2 合并淘汰

| 项目 | 说明 |
|------|------|
| 定义 | 合并到其他技能 |
| 条件 | 功能可被其他技能覆盖 |
| 流程 | 公告 → 迁移 → 合并 |
| 恢复 | 合并前可回滚 |

### 3.3 替换淘汰

| 项目 | 说明 |
|------|------|
| 定义 | 被新技能替换 |
| 条件 | 有更好的替代方案 |
| 流程 | 发布替代 → 迁移 → 淘汰 |
| 恢复 | 替换前可回滚 |

### 3.4 废弃淘汰

| 项目 | 说明 |
|------|------|
| 定义 | 标记为废弃但保留 |
| 条件 | 有历史价值或特殊需求 |
| 流程 | 标记废弃 → 限制访问 → 归档 |
| 恢复 | 可随时恢复 |

---

## 四、何时下线

### 4.1 下线时机

| 条件 | 说明 |
|------|------|
| 零使用 | 连续 30 天零调用 |
| 完全替代 | 替代方案完全覆盖功能 |
| 安全问题 | 存在无法修复的安全问题 |
| 依赖失效 | 核心依赖永久失效 |

### 4.2 下线时间线

| 阶段 | 时长 | 说明 |
|------|------|------|
| 公告期 | 2 周 | 发布下线公告 |
| 警告期 | 2 周 | 使用时显示警告 |
| 只读期 | 1 周 | 只读模式 |
| 下线期 | - | 正式下线 |
| 恢复期 | 30 天 | 可申请恢复 |
| 归档期 | - | 归档保存 |

---

## 五、何时合并

### 5.1 合并条件

| 条件 | 说明 |
|------|------|
| 功能重叠 | 与其他技能功能重叠 > 80% |
| 使用量低 | 使用量低于目标技能 |
| 维护成本高 | 单独维护成本高 |
| 用户体验 | 合并后用户体验更好 |

### 5.2 合并流程

```
合并评估
    ↓
┌─────────────────────────────────────┐
│ 1. 功能对比                          │
│    - 对比两个技能功能                │
│    - 识别差异点                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 合并方案                          │
│    - 制定合并方案                    │
│    - 评估影响                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 公告通知                          │
│    - 发布合并公告                    │
│    - 通知受影响用户                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 迁移执行                          │
│    - 迁移用户到目标技能              │
│    - 更新文档和路由                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 合并完成                          │
│    - 下线源技能                      │
│    - 更新技能目录                    │
└─────────────────────────────────────┘
```

## 六、何时替换

### 6.1 替换条件

| 条件 | 说明 |
|------|------|
| 更好方案 | 有明显更好的替代方案 |
| 技术升级 | 技术架构升级需要 |
| 性能提升 | 替代方案性能显著提升 |
| 功能增强 | 替代方案功能更完善 |

### 6.2 替换流程

```
替换评估
    ↓
┌─────────────────────────────────────┐
│ 1. 替代方案验证                      │
│    - 验证替代方案功能                │
│    - 验证替代方案性能                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 兼容性评估                        │
│    - 评估API兼容性                   │
│    - 制定迁移方案                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 发布替代方案                      │
│    - 发布新技能                      │
│    - 提供迁移指南                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 迁移过渡                          │
│    - 用户逐步迁移                    │
│    - 监控迁移进度                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 淘汰原技能                        │
│    - 原技能进入淘汰流程              │
│    - 完成迁移后下线                  │
└─────────────────────────────────────┘
```

## 七、回滚条件

### 7.1 可回滚场景

| 场景 | 回滚窗口 | 说明 |
|------|----------|------|
| 下线后 | 30 天 | 可申请恢复 |
| 合并后 | 14 天 | 可拆分恢复 |
| 替换后 | 30 天 | 可回退到原技能 |
| 废弃后 | 无限制 | 随时可恢复 |

### 7.2 回滚流程

```
回滚申请
    ↓
┌─────────────────────────────────────┐
│ 1. 申请评估                          │
│    - 评估回滚原因                    │
│    - 评估回滚影响                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 回滚审批                          │
│    - 技术负责人审批                  │
│    - 记录审批结果                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 回滚执行                          │
│    - 恢复技能服务                    │
│    - 更新路由配置                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 验证确认                          │
│    - 验证功能正常                    │
│    - 通知申请方                      │
└─────────────────────────────────────┘
```

## 八、路由影响评估

### 8.1 评估内容

| 评估项 | 说明 |
|--------|------|
| 直接调用 | 直接调用该技能的模块 |
| 间接调用 | 通过其他技能间接调用 |
| 配置引用 | 配置中引用该技能 |
| 文档引用 | 文档中引用该技能 |

### 8.2 评估流程

```
路由影响评估
    ↓
┌─────────────────────────────────────┐
│ 1. 调用链分析                        │
│    - 分析所有调用链                  │
│    - 识别受影响模块                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 影响量化                          │
│    - 量化影响范围                    │
│    - 评估影响程度                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 风险评估                          │
│    - 评估淘汰风险                    │
│    - 制定缓解措施                    │
└─────────────────────────────────────┘
    ↓
生成影响评估报告
```

### 8.3 影响报告

| 报告项 | 说明 |
|--------|------|
| 受影响模块 | 受影响的模块列表 |
| 受影响用户 | 受影响的用户数量 |
| 风险等级 | 风险等级评估 |
| 缓解措施 | 建议的缓解措施 |

## 九、淘汰通知

### 9.1 通知对象

| 对象 | 通知方式 | 说明 |
|------|----------|------|
| 受影响用户 | 邮件 + 站内信 | 直接通知 |
| 技术团队 | 邮件 + IM | 内部通知 |
| 管理层 | 邮件 | 管理层通知 |
| 合作伙伴 | 邮件 + 电话 | 重要伙伴 |

### 9.2 通知内容

```
【技能淘汰通知】
技能名称: {skill_name}
淘汰类型: {deprecation_type}
淘汰原因: {reason}
影响范围: {impact}
替代方案: {alternative}
时间安排: {timeline}
迁移指南: {migration_guide}
联系方式: {contact}
```

### 9.3 通知时间线

| 时间点 | 通知内容 |
|--------|----------|
| T-30天 | 淘汰计划通知 |
| T-14天 | 淘汰公告 |
| T-7天 | 淘汰提醒 |
| T-1天 | 最后提醒 |
| T-0 | 淘汰执行 |
| T+1天 | 淘汰确认 |

## 十、淘汰记录

### 10.1 记录内容

| 记录项 | 说明 |
|--------|------|
| 淘汰ID | 淘汰唯一标识 |
| 技能ID | 被淘汰技能ID |
| 淘汰类型 | 下线/合并/替换/废弃 |
| 淘汰原因 | 淘汰原因说明 |
| 淘汰时间 | 淘汰执行时间 |
| 影响评估 | 影响评估报告 |
| 替代方案 | 替代方案说明 |
| 审批记录 | 审批记录 |
| 通知记录 | 通知记录 |

### 10.2 记录保留

| 记录类型 | 保留期限 |
|----------|----------|
| 淘汰记录 | 永久 |
| 影响评估 | 3年 |
| 审批记录 | 3年 |
| 通知记录 | 1年 |

---

## 十一、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 用户强烈反对 | 暂停淘汰 + 重新评估 |
| 迁移失败 | 延长过渡期 + 技术支持 |
| 回滚请求 | 按回滚流程处理 |
| 影响超预期 | 暂停淘汰 + 重新评估 |

---

## 十二、完成标准

| 标准 | 验证方式 |
|------|----------|
| 判定正确 | 淘汰判定符合标准 |
| 影响评估 | 影响评估完整 |
| 通知到位 | 所有受影响方已通知 |
| 迁移完成 | 用户迁移完成 |
| 记录完整 | 淘汰记录完整 |
