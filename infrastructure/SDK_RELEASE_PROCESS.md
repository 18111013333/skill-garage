# SDK_RELEASE_PROCESS.md - SDK 发布流程

## 目的
定义 SDK 发布流程，确保发布标准化。

## 适用范围
所有官方 SDK 的发布。

---

## 一、发布流程

```
代码完成 → 构建 → 测试 → 签名 → 发布 → 验证 → 回滚准备
    ↓        ↓       ↓       ↓       ↓       ↓         ↓
  开发者   CI/CD   QA团队   安全团队  发布团队  QA团队   运维团队
```

---

## 二、构建阶段

### 2.1 构建环境
| 环境 | 说明 |
|------|------|
| Python | Python 3.8+ |
| JavaScript | Node.js 16+ |
| Java | JDK 11+ |
| Go | Go 1.18+ |

### 2.2 构建步骤
```
1. 拉取代码
2. 安装依赖
3. 编译代码
4. 打包产物
5. 生成文档
```

### 2.3 构建产物
| SDK | 产物格式 |
|-----|----------|
| Python | wheel, tar.gz |
| JavaScript | tgz |
| Java | jar |
| Go | 源码 |

---

## 三、测试阶段

### 3.1 测试类型
| 测试类型 | 说明 | 覆盖率要求 |
|----------|------|------------|
| 单元测试 | 函数级测试 | > 80% |
| 集成测试 | 接口集成测试 | > 70% |
| 端到端测试 | 完整流程测试 | 关键路径 |
| 性能测试 | 性能基准测试 | 基准对比 |
| 兼容测试 | 版本兼容测试 | 多版本 |

### 3.2 测试环境
| 环境 | 说明 |
|------|------|
| 开发环境 | 开发者本地 |
| 测试环境 | CI/CD 自动化 |
| 预发布环境 | 接近生产 |
| 生产环境 | 真实环境 |

### 3.3 测试通过标准
- [ ] 所有单元测试通过
- [ ] 所有集成测试通过
- [ ] 覆盖率达标
- [ ] 无高危漏洞
- [ ] 性能无退化

---

## 四、签名阶段

### 4.1 签名目的
```
确保 SDK 产物未被篡改
验证发布者身份
```

### 4.2 签名方式
| SDK | 签名方式 |
|-----|----------|
| Python | GPG 签名 |
| JavaScript | npm 签名 |
| Java | JAR 签名 |
| Go | 校验和 |

### 4.3 签名验证
```bash
# Python
gpg --verify package.whl.asc package.whl

# JavaScript
npm verify @openclaw/sdk

# Java
jarsigner -verify package.jar
```

---

## 五、发布阶段

### 5.1 发布渠道
| 渠道 | 说明 | 用途 |
|------|------|------|
| stable | 稳定版 | 生产使用 |
| beta | 测试版 | 提前体验 |
| alpha | 早期版 | 内部测试 |

### 5.2 发布平台
| SDK | 平台 |
|-----|------|
| Python | PyPI |
| JavaScript | npm |
| Java | Maven Central |
| Go | pkg.go.dev |

### 5.3 发布步骤
```
1. 上传产物
2. 发布公告
3. 更新文档
4. 通知用户
```

---

## 六、回滚准备

### 6.1 回滚条件
| 条件 | 说明 |
|------|------|
| 严重 Bug | 影响核心功能 |
| 安全漏洞 | 安全问题 |
| 兼容问题 | 严重兼容问题 |
| 性能问题 | 严重性能退化 |

### 6.2 回滚步骤
```
1. 决定回滚
2. 标记版本为 deprecated
3. 发布旧版本为 latest
4. 通知用户
5. 修复问题
6. 重新发布
```

### 6.3 回滚时间
| 问题级别 | 回滚时间 |
|----------|----------|
| 紧急 | 1 小时内 |
| 严重 | 4 小时内 |
| 一般 | 24 小时内 |

---

## 七、兼容性检查

### 7.1 兼容性测试
| 测试项 | 说明 |
|--------|------|
| API 兼容 | 与 API 版本兼容 |
| 语言版本 | 与语言版本兼容 |
| 依赖兼容 | 与依赖库兼容 |
| 平台兼容 | 与平台兼容 |

### 7.2 兼容性矩阵
```
SDK 版本 × API 版本 × 语言版本 × 平台
```

### 7.3 兼容性声明
```json
{
  "sdk_version": "1.0.0",
  "compatible_api_versions": ["v1"],
  "compatible_language_versions": ["3.8", "3.9", "3.10", "3.11"],
  "compatible_platforms": ["linux", "macos", "windows"]
}
```

---

## 八、发布说明

### 8.1 发布说明内容
| 内容 | 说明 |
|------|------|
| 版本号 | 新版本号 |
| 发布日期 | 发布时间 |
| 变更类型 | major/minor/patch |
| 新功能 | 新增功能列表 |
| 修复 | Bug 修复列表 |
| 变更 | 不兼容变更 |
| 废弃 | 废弃功能列表 |
| 升级指南 | 升级步骤 |

### 8.2 发布说明格式
```markdown
# Release v1.2.0

## 新功能
- 添加流式响应支持
- 添加自动重试机制

## 修复
- 修复 Token 刷新问题
- 修复超时处理问题

## 变更
- `client.chat()` → `client.chat.create()`

## 升级指南
1. 更新依赖版本
2. 修改方法调用
3. 测试验证
```

### 9.2 发布后检查
- [ ] 产物可下载
- [ ] 文档可访问
