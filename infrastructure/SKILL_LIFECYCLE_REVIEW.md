# SKILL_LIFECYCLE_REVIEW.md - 技能生命周期评审

**版本: V28.0**

## 目的
定义技能生命周期评审机制，确保技能从创建到废弃的完整生命周期都有规范的评审流程。

## 适用范围
所有技能的生命周期管理。

---

## 一、生命周期阶段

```
创建 → 孵化 → 候选 → 试用 → 正式 → 核心 → 废弃 → 删除
  │       │       │       │       │       │       │       │
  └───────┴───────┴───────┴───────┴───────┴───────┴───────┘
                          评审节点
```

---

## 二、评审类型

| 评审类型 | 代码 | 触发时机 | 评审人 |
|----------|------|----------|--------|
| 创建评审 | CREATE | 技能创建时 | 技能管理员 |
| 晋升评审 | PROMOTE | 阶段晋升时 | 技能委员会 |
| 降级评审 | DEMOTE | 阶段降级时 | 技能Owner |
| 废弃评审 | DEPRECATE | 标记废弃时 | 技能委员会 |
| 删除评审 | DELETE | 正式删除时 | 技能委员会 |
| 定期评审 | PERIODIC | 定期执行 | 自动+人工 |

---

## 三、创建评审

### 3.1 评审内容

| 检查项 | 说明 | 通过标准 |
|--------|------|----------|
| 技能名称 | 名称规范 | 符合命名规范 |
| 技能分类 | 分类正确 | 分类准确 |
| Owner | 有明确Owner | Owner已确认 |
| 目标问题 | 问题明确 | 问题描述清晰 |
| 黄金路径 | 有候选路径 | 至少1条候选 |
| 基础代码 | 代码可用 | 基础功能可用 |
| 基础文档 | 文档存在 | README存在 |

### 3.2 创建评审流程

```
创建申请
    ↓
┌─────────────────────────────────────┐
│ 1. 自动检查                          │
│    - 名称规范检查                    │
│    - 分类检查                        │
│    - 文档检查                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 人工评审                          │
│    - 功能合理性评审                  │
│    - 与现有技能重复检查              │
│    - Owner确认                       │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 决策                              │
│    - 通过：进入incubating            │
│    - 修改：修改后重新评审            │
│    - 拒绝：拒绝创建                  │
└─────────────────────────────────────┘
```

## 四、晋升评审

### 4.1 晋升评审标准

| 晋升路径 | 评审重点 | 通过标准 |
|----------|----------|----------|
| incubating → candidate | 功能验证 | 功能可用、有验证数据 |
| candidate → beta | 定向试用 | 试用成功、有正面反馈 |
| beta → ga | 受控验证 | 稳定可靠、文档完整 |
| ga → core | 价值验证 | 高频使用、绑定黄金路径 |

### 4.2 晋升评审材料

| 材料 | 说明 | 必需 |
|------|------|------|
| 晋升申请表 | 申请晋升的表单 | 是 |
| 验证报告 | 验证数据报告 | 是 |
| 测试报告 | 测试覆盖报告 | 是 |
| 用户反馈 | 用户反馈汇总 | 是 |
| 文档检查 | 文档完整性检查 | 是 |
| Owner确认 | Owner确认函 | 是 |

### 4.3 晋升评审流程

```
晋升申请
    ↓
┌─────────────────────────────────────┐
│ 1. 材料审核                          │
│    - 检查材料完整性                  │
│    - 检查数据真实性                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 评审会议                          │
│    - Owner陈述                       │
│    - 评审人提问                      │
│    - 投票表决                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 决策                              │
│    - 通过：晋升                      │
│    - 有条件通过：修复后晋升          │
│    - 不通过：继续当前阶段            │
└─────────────────────────────────────┘
```

## 五、降级评审

### 5.1 降级触发条件

| 条件 | 说明 | 自动/人工 |
|------|------|-----------|
| 成功率下降 | 成功率 < 阈值 | 自动 |
| 错误率上升 | 错误率 > 阈值 | 自动 |
| 无调用 | 长期无调用 | 自动 |
| 无Owner | 长期无Owner | 自动 |
| 严重Bug | 严重Bug未修复 | 人工 |
| 安全问题 | 安全漏洞 | 人工 |

### 5.2 降级评审流程

```
降级触发
    ↓
┌─────────────────────────────────────┐
│ 1. 原因分析                          │
│    - 分析降级原因                    │
│    - 评估影响范围                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. Owner确认                         │
│    - 通知Owner                       │
│    - Owner确认或申诉                 │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 决策                              │
│    - 确认降级：执行降级              │
│    - 申诉成功：保持当前级别          │
│    - 需修复：修复后重新评估          │
└─────────────────────────────────────┘
```

## 六、废弃评审

### 6.1 废弃触发条件

| 条件 | 说明 |
|------|------|
| 有替代技能 | 新技能已上线可替代 |
| 功能过时 | 功能不再需要 |
| 长期无人维护 | 超过90天无维护 |
| 严重问题 | 严重问题未修复 |
| 低使用率 | 长期低使用且无保护 |

### 6.2 废弃评审流程

```
废弃申请
    ↓
┌─────────────────────────────────────┐
│ 1. 替代验证                          │
│    - 检查是否有替代技能              │
│    - 验证替代技能可用性              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 影响评估                          │
│    - 评估对黄金路径的影响            │
│    - 评估对用户的影响                │
│    - 评估对其他技能的影响            │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 迁移计划                          │
│    - 制定迁移计划                    │
│    - 确定迁移时间表                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 评审会议                          │
│    - 技能委员会评审                  │
│    - 投票表决                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 执行废弃                          │
│    - 标记为deprecated                │
│    - 通知相关方                      │
│    - 执行迁移                        │
└─────────────────────────────────────┘
```

## 七、删除评审

### 7.1 删除四步法

| 步骤 | 说明 | 时长 |
|------|------|------|
| 识别 | 识别删除候选 | - |
| 冻结 | 冻结技能，观察调用 | 7-21天 |
| 观察 | 观察是否有真实调用 | 冻结期内 |
| 删除 | 确认无调用后删除 | - |

### 7.2 删除评审条件

| 条件 | 说明 |
|------|------|
| 冻结期结束 | 冻结期已结束 |
| 无真实调用 | 冻结期内无真实调用 |
| 有替代项 | 有替代技能上线 |
| 路由迁移完成 | 路由已迁移到替代项 |
| 无blocker | 无阻塞问题 |
| 评审通过 | 删除评审通过 |
| 回滚方案 | 有回滚方案 |

## 八、定期评审

### 8.1 评审频率

| 技能类型 | 评审频率 | 评审内容 |
|----------|----------|----------|
| core | 每月 | 使用情况、问题、改进 |
| ga | 每月 | 使用情况、稳定性 |
| beta | 每两周 | 验证进展、问题 |
| candidate | 每周 | 试用情况、反馈 |
| incubating | 每周 | 开发进展、问题 |
| deprecated | 每周 | 迁移进展、删除准备 |

### 8.2 定期评审报告
