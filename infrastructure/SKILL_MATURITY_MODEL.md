# SKILL_MATURITY_MODEL.md - 技能成熟度模型

**版本: V28.0**

## 目的
定义技能成熟度分级标准，确保技能从孵化到核心的完整生命周期管理。

## 适用范围
所有技能，包括真技能、适配器、处理器、人格、元工具。

---

## 一、成熟度阶段定义

| 阶段 | 代码 | 定义 | 保护期 | 路由权限 |
|------|------|------|--------|----------|
| 孵化中 | incubating | 孵化中，先做小样本与离线验证 | 30天 | 禁止自动路由 |
| 候选 | candidate | 候选可用，进入定向试用 | 30天 | 仅指定场景 |
| 受控试用 | beta | 受控试用，只能在低风险或指定场景上场 | 45天 | 低风险路由 |
| 正式可用 | ga | 正式可用，可进入常规路由 | 无 | 常规路由 |
| 核心技能 | core | 核心技能，进入主路由与黄金路径 | 无 | 主路由+黄金路径 |
| 待退役 | deprecated | 待退役，已标记为即将删除 | 14天 | 禁止新调用 |

---

## 二、成熟度评估维度

| 维度 | 权重 | 说明 | 评估方式 |
|------|------|------|----------|
| 功能完整性 | 25% | 功能是否完整可用 | 功能测试 |
| 稳定性 | 20% | 是否稳定可靠 | 错误率监控 |
| 使用频率 | 15% | 实际使用频率 | 调用统计 |
| 用户反馈 | 15% | 用户评价反馈 | 反馈收集 |
| 文档完整性 | 10% | 文档是否完整 | 文档检查 |
| 测试覆盖 | 10% | 测试是否充分 | 测试报告 |
| 维护活跃度 | 5% | 是否持续维护 | 提交记录 |

## 三、成熟度晋升标准

### 3.1 incubating → candidate

| 条件 | 要求 |
|------|------|
| 功能验证 | 完成基础功能验证 |
| 离线测试 | 通过离线测试 |
| 文档 | 有基础使用文档 |
| Owner | 有明确Owner |
| 黄金路径 | 有候选黄金路径 |

### 3.2 candidate → beta

| 条件 | 要求 |
|------|------|
| 定向试用 | 完成至少3次定向试用 |
| 成功率 | 成功率 > 80% |
| 反馈 | 有正面用户反馈 |
| 测试 | 有基础测试用例 |
| 评估报告 | 有评估报告 |

### 3.3 beta → ga

| 条件 | 要求 |
|------|------|
| 受控试用 | 完成至少10次受控试用 |
| 成功率 | 成功率 > 90% |
| 错误率 | 错误率 < 5% |
| 文档 | 有完整使用文档 |
| 测试 | 测试覆盖率 > 60% |
| 监控 | 有监控指标 |

### 3.4 ga → core

| 条件 | 要求 |
|------|------|
| 使用频率 | 月调用 > 100次 |
| 黄金路径 | 绑定到黄金路径 |
| 成功率 | 成功率 > 95% |
| 用户满意度 | 满意度 > 4.0/5.0 |
| 文档 | 有完整文档+示例 |
| 测试 | 测试覆盖率 > 80% |
| Owner | 有长期维护Owner |

---

## 四、成熟度降级标准

### 4.1 降级触发条件

| 条件 | 降级 |
|------|------|
| 连续30天无调用 | 降一级 |
| 成功率 < 80% | 降一级 |
| 错误率 > 10% | 降一级 |
| 无Owner超过60天 | 降一级 |
| 严重Bug未修复 | 降级至beta |
| 安全漏洞 | 降级至deprecated |

### 4.2 deprecated 触发条件

| 条件 | 说明 |
|------|------|
| 有替代技能 | 新技能已上线 |
| 功能过时 | 功能不再需要 |
| 长期无人维护 | 超过90天无维护 |
| 严重问题未修复 | 超过30天未修复 |

## 五、成熟度评估流程

```
评估触发
    ↓
┌─────────────────────────────────────┐
│ 1. 数据收集                          │
│    - 收集使用数据                    │
│    - 收集反馈数据                    │
│    - 收集测试数据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 维度评分                          │
│    - 各维度独立评分                  │
│    - 计算加权总分                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 成熟度判定                        │
│    - 根据分数判定成熟度              │
│    - 检查晋升/降级条件               │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 状态更新                          │
│    - 更新成熟度状态                  │
│    - 通知相关方                      │
│    - 调整路由权限                    │
└─────────────────────────────────────┘
```

## 六、成熟度报告

### 6.1 报告频率

| 报告类型 | 频率 | 内容 |
|----------|------|------|
| 孵化报告 | 每周 | 孵化中技能进展 |
| 成熟度报告 | 每月 | 所有技能成熟度状态 |
| 晋降报告 | 实时 | 晋升/降级事件 |
| 废弃报告 | 每月 | 待废弃技能清单 |

### 6.2 报告内容

```json
{
  "skill_id": "SKILL-001",

  "current_maturity": "ga",
  "scores": {

    "stability": 0.85,
    "usage": 0.7,
    "feedback": 0.8,
    "documentation": 0.9,
    "testing": 0.75,
    "maintenance": 0.8
  },
  "weighted_score": 0.83,
  "next_review": "2026-05-08",
  "recommendation": "consider_core"
}
```
