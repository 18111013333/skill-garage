# THIRD_PARTY_RISK_POLICY.md - 第三方风险治理规则

## 目的
定义第三方接入和使用的风险治理规则，确保外部依赖受控、可审计、可退出。

## 适用范围
所有第三方服务、SDK、模型、数据源、云服务等外部依赖。

---

## 一、第三方风险治理原则

| 原则 | 说明 | 执行要求 |
|------|------|----------|
| 最小暴露 | 数据暴露最小化 | 仅提供必要数据 |
| 最小权限 | 权限最小化 | 仅授予必要权限 |
| 可审计 | 所有操作可追溯 | 完整日志记录 |
| 可退出 | 随时可终止依赖 | 有替代方案 |
| 持续监控 | 持续风险评估 | 定期审查 |

---

## 二、接入前审查

### 2.1 审查流程
```
需求提出
    ↓
┌─────────────────────────────────────┐
│ 1. 必要性评估                        │
│    - 是否必须使用第三方               │
│    - 是否有内部替代方案               │
│    - 成本效益分析                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 供应商评估                        │
│    - 公司背景调查                    │
│    - 安全资质审查                    │
│    - 合规状态验证                    │
│    - 历史事件调查                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 技术评估                          │
│    - 接口安全评估                    │
│    - 数据流分析                      │
│    - 性能影响评估                    │
│    - 集成复杂度评估                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 风险评估                          │
│    - 数据暴露风险                    │
│    - 服务依赖风险                    │
│    - 合规风险                        │
│    - 供应链风险                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 审批决策                          │
│    - 风险是否可接受                  │
│    - 缓解措施是否充分                │
│    - 是否需要附加条件                │
└─────────────────────────────────────┘
    ↓
接入实施
```

### 2.2 审查清单

#### 必要性评估
| 评估项 | 问题 | 通过标准 |
|--------|------|----------|
| 业务必要性 | 是否业务必需？ | 有明确业务需求 |
| 替代方案 | 是否有内部替代？ | 无或内部方案不可行 |
| 成本效益 | 成本是否合理？ | ROI 可接受 |
| 战略一致性 | 是否符合技术战略？ | 符合架构规划 |

#### 供应商评估
| 评估项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| 公司资质 | 注册信息、经营状态 | 合法合规经营 |
| 安全认证 | SOC2、ISO27001 等 | 具备相关认证 |
| 隐私合规 | GDPR、数据保护 | 符合要求 |
| 财务稳定 | 财务状况、经营历史 | 财务健康 |
| 声誉调查 | 客户评价、历史事件 | 无重大负面 |
| 服务能力 | SLA、支持能力 | 满足需求 |

#### 技术评估
| 评估项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| 接口安全 | 认证、加密、签名 | 符合安全标准 |
| 数据处理 | 数据存储、传输、销毁 | 符合数据政策 |
| 可用性 | SLA、灾备能力 | 满足业务需求 |
| 性能 | 响应时间、吞吐量 | 满足性能要求 |
| 兼容性 | 与现有系统集成 | 兼容无冲突 |
| 可维护性 | 文档、支持、更新 | 可维护 |

#### 风险评估
| 风险类型 | 评估内容 | 风险等级判定 |
|----------|----------|--------------|
| 数据风险 | 数据暴露范围、敏感度 | 暴露越大风险越高 |
| 依赖风险 | 服务重要性、替代性 | 越核心风险越高 |
| 合规风险 | 数据跨境、隐私保护 | 违规风险高 |
| 供应链风险 | 供应商上游依赖 | 多层依赖风险高 |

### 2.3 审批权限
| 风险等级 | 审批人 | 审批流程 |
|----------|--------|----------|
| LOW | 技术负责人 | 单人审批 |
| MEDIUM | 技术负责人 + 安全负责人 | 双人审批 |
| HIGH | 技术负责人 + 安全负责人 + 合规负责人 | 三人审批 |
| CRITICAL | CTO + CISO + 法务 | 委员会审批 |

## 三、数据最小暴露

### 3.1 数据暴露分级
| 级别 | 数据类型 | 暴露要求 | 审批要求 |
|------|----------|----------|----------|
| L0 | 无数据暴露 | 无限制 | 无需审批 |
| L1 | 非敏感数据 | 最小必要 | 技术负责人 |
| L2 | 内部数据 | 必要 + 脱敏 | 安全负责人 |
| L3 | 敏感数据 | 必要 + 加密 + 审计 | 高管审批 |
| L4 | 核心机密 | 禁止暴露 | 不允许 |

### 3.2 数据暴露控制措施
| 措施 | 适用场景 | 实施方式 |
|------|----------|----------|
| 数据脱敏 | 敏感数据 | 传输前脱敏处理 |
| 数据加密 | 所有传输数据 | TLS + 字段加密 |
| 数据最小化 | 所有场景 | 仅传输必要字段 |
| 访问控制 | 所有接口 | API Key + IP 白名单 |
| 数据隔离 | 多租户场景 | 租户数据隔离 |
| 日志脱敏 | 日志记录 | 日志中脱敏敏感数据 |

### 3.3 数据暴露审计
| 审计项 | 频率 | 审计内容 |
|--------|------|----------|
| 数据流审计 | 每月 | 数据传输路径和内容 |
| 权限审计 | 每季度 | 第三方访问权限 |
| 数据使用审计 | 每月 | 第三方数据使用情况 |
| 异常审计 | 实时 | 异常数据访问告警 |

---

## 四、最小权限

### 4.1 权限分级
| 权限级别 | 权限范围 | 适用场景 | 审批要求 |
|----------|----------|----------|----------|
| P0 | 只读公开数据 | 公开 API | 无需审批 |
| P1 | 只读内部数据 | 数据查询 | 技术负责人 |
| P2 | 读写指定数据 | 业务操作 | 安全负责人 |
| P3 | 系统操作权限 | 系统集成 | 高管审批 |
| P4 | 管理权限 | 禁止授予 | 不允许 |

### 4.2 权限控制措施
| 措施 | 说明 | 实施要求 |
|------|------|----------|
| 细粒度授权 | 按功能/数据授权 | 最小权限集 |
| 时间限制 | 权限有效期 | 定期续期 |
| 条件限制 | 条件触发权限 | 条件满足时生效 |
| 审计记录 | 所有权限操作 | 完整审计日志 |
| 定期审查 | 权限有效性 | 每季度审查 |

### 4.3 权限生命周期
```
权限申请
    ↓
权限审批
    ↓
权限授予（记录有效期）
    ↓
权限使用（审计记录）
    ↓
权限审查（定期）
    ↓
权限续期 或 权限回收
```

## 五、日志要求

### 5.1 日志记录要求
| 日志类型 | 记录内容 | 保留期限 | 存储要求 |
|----------|----------|----------|----------|
| 调用日志 | 请求/响应摘要 | 90 天 | 加密存储 |
| 错误日志 | 错误详情 | 180 天 | 加密存储 |
| 审计日志 | 权限/配置变更 | 3 年 | 不可篡改 |
| 数据日志 | 数据传输记录 | 1 年 | 加密存储 |

### 5.2 日志内容要求
```json
{
  "log_id": "unique-id",
  "timestamp": "2026-04-07T10:00:00Z",
  "vendor_id": "VENDOR-000001",
  "service_name": "LLM API",
  "operation": "inference",
  "request_hash": "sha256-hash",
  "response_hash": "sha256-hash",
  "data_size_bytes": 1024,
  "latency_ms": 500,
  "status": "success",
  "error_code": null,
  "user_id": "user-xxx",
  "session_id": "session-xxx",
  "ip_address": "x.x.x.x",
  "data_classification": "internal",
  "consent_id": "consent-xxx"
}
```

### 5.3 日志审计要求
| 审计项 | 频率 | 审计内容 |
|--------|------|----------|
| 日志完整性 | 每日 | 日志是否完整 |
| 日志异常 | 实时 | 异常调用检测 |
| 日志合规 | 每月 | 日志是否符合要求 |
| 日志保留 | 每季度 | 保留期限合规 |

## 六、退出机制

### 6.1 退出触发条件
| 触发条件 | 退出类型 | 通知期限 | 处理时限 |
|----------|----------|----------|----------|
| 合同到期 | 正常退出 | 按合同约定 | 30 天 |
| 服务不达标 | 违约退出 | 立即通知 | 7 天 |
| 安全事件 | 紧急退出 | 立即通知 | 24 小时 |
| 合规问题 | 强制退出 | 立即通知 | 48 小时 |
| 战略调整 | 计划退出 | 90 天通知 | 90 天 |

### 6.2 退出流程
```
退出决策
    ↓
┌─────────────────────────────────────┐
│ 1. 影响评估                          │
│    - 业务影响范围                    │
│    - 数据影响范围                    │
│    - 系统影响范围                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 替代方案准备                      │
│    - 替代供应商确认                  │
│    - 数据迁移方案                    │
│    - 系统切换方案                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 通知与沟通                        │
│    - 通知供应商                      │
│    - 通知内部相关方                  │
│    - 通知受影响用户                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 数据处理                          │
│    - 数据导出                        │
│    - 数据删除确认                    │
│    - 数据销毁证明                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 系统切换                          │
│    - 切换到替代方案                  │
│    - 验证功能正常                    │
│    - 关闭原服务访问                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 6. 权限回收                          │
│    - 回收 API 密钥                   │
│    - 回收访问权限                    │
│    - 关闭网络访问                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 7. 退出确认                          │
│    - 确认数据已删除                  │
│    - 确认权限已回收                  │
│    - 更新供应商台账                  │
└─────────────────────────────────────┘
```

### 6.3 数据处理要求
| 处理项 | 要求 | 验证方式 |
|--------|------|----------|
| 数据导出 | 导出所有我方数据 | 导出记录 |
| 数据删除 | 删除供应商存储的我方数据 | 删除确认函 |
| 数据销毁 | 安全销毁数据 | 销毁证明 |
| 凭证回收 | 回收所有访问凭证 | 凭证失效确认 |
| 日志保留 | 保留审计日志 | 日志归档 |

## 七、替代方案要求

### 7.1 替代方案类型
| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 热备替代 | 实时可切换 | 核心服务 |
| 冷备替代 | 可快速启用 | 重要服务 |
| 内部替代 | 内部实现 | 关键能力 |
| 手工替代 | 人工处理 | 非关键服务 |

### 7.2 替代方案要求
| 服务重要性 | 替代要求 | 切换时间要求 |
|------------|----------|--------------|
| CRITICAL | 必须有热备 | < 5 分钟 |
| HIGH | 必须有冷备 | < 1 小时 |
| MEDIUM | 建议有替代 | < 24 小时 |
| LOW | 可无替代 | 无要求 |

### 7.3 替代方案验证
| 验证项 | 频率 | 验证内容 |
|--------|------|----------|
| 功能验证 | 每季度 | 替代方案功能完整 |
| 性能验证 | 每季度 | 替代方案性能达标 |
| 切换演练 | 每半年 | 切换流程可行 |
| 数据同步 | 持续 | 数据可迁移 |

## 八、持续监控

### 8.1 监控指标
| 指标类型 | 监控项 | 告警阈值 |
|----------|--------|----------|
| 可用性 | 服务可用率 | < 99.9% |
| 性能 | 响应时间 | > SLA |
| 错误率 | 错误比例 | > 1% |
| 数据量 | 数据传输量 | 异常增长 |
| 安全 | 安全事件 | 任何事件 |

### 8.2 定期审查
| 审查类型 | 频率 | 审查内容 |
|----------|------|----------|
| 日常监控 | 实时 | 服务状态监控 |
| 月度审查 | 每月 | SLA 合规、事件回顾 |
| 季度审查 | 每季度 | 风险评估、权限审查 |
| 年度审查 | 每年 | 全面评估、合同续签 |

### 8.3 风险再评估触发
| 触发条件 | 动作 |
|----------|------|
| 供应商安全事件 | 立即风险评估 |
| 服务变更通知 | 评估变更影响 |
| SLA 不达标 | 评估是否继续使用 |
| 合规要求变化 | 评估合规影响 |
| 负面新闻 | 调查并评估风险 |

## 九、第三方风险等级

### 9.1 风险等级判定
| 因素 | 权重 | 评分标准 |
|------|------|----------|
| 数据暴露 | 30% | L0=1, L1=2, L2=3, L3=4 |
| 服务重要性 | 25% | LOW=1, MEDIUM=2, HIGH=3, CRITICAL=4 |
| 替代性 | 20% | 有热备=1, 有冷备=2, 有替代=3, 无替代=4 |
| 合规风险 | 15% | 无风险=1, 低风险=2, 中风险=3, 高风险=4 |
| 历史事件 | 10% | 无事件=1, 轻微=2, 中等=3, 严重=4 |

### 9.2 风险等级
| 总分 | 风险等级 | 管理要求 |
|------|----------|----------|
| 1.0-1.5 | LOW | 标准监控 |
| 1.5-2.5 | MEDIUM | 加强监控 |
| 2.5-3.5 | HIGH | 严格管控 + 定期审查 |
| 3.5-4.0 | CRITICAL | 最高管控 + 持续审查 |
