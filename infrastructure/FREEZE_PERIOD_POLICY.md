# FREEZE_PERIOD_POLICY.md - 冻结期规则

**版本: V28.0**
**生成时间**: 2026-04-09

---

## 一、冻结期定义

冻结期不是等待删除，而是**删除前的受控观察阶段**。

### 1.1 冻结期目标

| 目标 | 说明 |
|------|------|
| 验证真实价值 | 该对象是否还有真实价值 |
| 验证替代项 | 替代项是否已经接住 |
| 发现隐藏依赖 | 是否存在隐藏依赖 |
| 识别关键能力 | 是否属于低频但关键能力 |

### 1.2 冻结期本质

冻结期是删除前的：
- 受控观察阶段
- 迁移验证阶段
- 风险缓冲阶段
- 证据留存阶段

---

## 二、冻结期为什么不能新增依赖

### 2.1 四大原因

| 原因 | 说明 |
|------|------|
| 污染调用数据 | 会污染真实调用数据，无法判断是否低价值 |
| 重新绑定风险 | 会让低价值对象被重新绑定，无法判断是否该删 |
| 干扰替代迁移 | 会干扰替代迁移，造成一边迁出一边迁回 |
| 放大删除风险 | 会放大删除风险和影响范围 |

### 2.2 详细说明

**原因一：无法判断真实价值**

冻结期的目的是隔离观察，看清对象真实的生存价值。如果允许新增依赖：
- 调用量会被后来硬接出来的新依赖污染
- 无法判断它到底是不是低价值
- 删除决策会失去依据

**原因二：删除风险放大**

- 原本只是一个低价值旧项
- 冻结期里一旦又被接入新路由
- 就会重新变成"不能删"的对象
- 删除风险被人为放大

**原因三：替代迁移失效**

- 本来是要把流量迁到替代项
- 结果一边迁出、一边迁回
- 替代方案验证失败
- 迁移工作白做

**原因四：观察数据失真**

以下数据都会变得不可信：
- 调用频率
- 影响范围
- Owner判断
- 删除后果

## 三、冻结期允许做什么

| 允许操作 | 说明 |
|----------|------|
| 从主路由移除 | 停止自动推荐和路由 |
| 保留只读访问 | 允许查询和诊断 |
| 观察真实调用 | 记录实际调用情况 |
| 验证替代项 | 确认替代方案有效 |
| 记录影响范围 | 记录受影响的路由和技能 |
| 收集Owner反馈 | 给Owner申诉机会 |
| 准备回滚信息 | 记录回滚方案 |

---

## 四、冻结期不允许做什么

| 禁止操作 | 说明 |
|----------|------|
| 新增正式业务依赖 | 禁止接入新的业务调用 |
| 新增主路由绑定 | 禁止加入主路由 |
| 新增黄金路径绑定 | 禁止绑定到黄金路径 |
| 新增外部承诺引用 | 禁止对外承诺使用 |
| 当成推荐默认项 | 禁止作为默认推荐 |
| 隐式恢复调用 | 禁止通过其他方式恢复调用 |
| 边冻结边扩建 | 禁止在冻结期继续建设 |

---

## 五、冻结期的两个例外

### 5.1 迁移用临时依赖

| 条件 | 说明 |
|------|------|
| 用途 | 只为数据迁移或兼容映射 |
| 路由 | 不进入正式主路由 |
| 时效 | 必须有失效时间 |
| 留痕 | 必须留痕记录 |

### 5.2 安全/回滚保底依赖

| 条件 | 说明 |
|------|------|
| 用途 | 仅限 fallback only |
| 范围 | 必须明确范围 |
| 时效 | 必须明确时效 |
| 审批 | 必须经过审批 |
| 性质 | 不算新增正式业务依赖 |

**总结**：冻结期不是绝对不能碰，而是不能新增"正式业务依赖"。

---

## 六、正式删除前必须满足的条件

| 条件 | 说明 |
|------|------|
| 冻结期结束 | 冻结期时间已满 |
| 观察报告完成 | 观察报告已生成 |
| 替代项验证通过 | 替代方案已验证有效 |
| 没有新的blocker | 无阻塞问题 |
| 删除清单已生成 | 删除清单与回滚方案已生成 |

---

## 七、冻结期时长

| 对象类型 | 冻结期 | 说明 |
|----------|--------|------|
| 技能 | 7天 | 观察真实调用 |
| 模块 | 14天 | 验证依赖关系 |
| 核心模块 | 21天 | 高风险对象 |
| 目录 | 14天 | 验证影响范围 |

## 八、冻结期观察报告

### 8.1 报告内容

```json
{
  "freeze_id": "FREEZE-2026-001",
  "object_type": "skill",
  "object_name": "skill-name",
  "freeze_start": "2026-04-08",
  "freeze_end": "2026-04-15",
  "observations": {
    "total_calls": 0,
    "unique_callers": 0,
    "has_real_calls": false,
    "has_blockers": false,
    "replacement_verified": true
  },
  "recommendation": "delete",
  "approver": "tech-lead@example.com"
}
```

### 8.2 报告结论

| 结论 | 说明 |
|------|------|
| delete | 可以删除 |
| extend | 延长冻结期 |
| restore | 恢复正常状态 |
| critical | 标记为关键低频 |

---

## 九、最容易记住的两句话

1. **冻结期不是为了拖延删除，而是为了防止把"应该观察的东西"直接误删。**

2. **未成熟技能最需要冻结期保护，老旧重复技能最需要冻结期验证。**

|------|------|
