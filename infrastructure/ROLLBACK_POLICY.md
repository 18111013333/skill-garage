# ROLLBACK_POLICY.md - 回滚策略

## 目的
明确什么情况触发回滚，如何回滚，回滚后如何处理一致性。

## 适用范围
所有版本升级、配置变更、规则修改的回滚操作。

## 回滚触发条件

### 自动回滚触发
| 条件 | 阈值 | 说明 |
|------|------|------|
| 错误率激增 | > 10% (原 < 2%) | 5分钟内 |
| 响应时间激增 | > 200% 基线 | 5分钟内 |
| 用户投诉激增 | > 10次/小时 | 1小时内 |
| 安全事件 | 任何安全事件 | 立即触发 |
| 评测不通过 | 回归门禁失败 | 阻止发布 |

### 人工回滚触发
| 条件 | 说明 |
|------|------|
| 发现严重Bug | 影响核心功能 |
| 用户反馈问题 | 多用户反馈 |
| 性能问题 | 影响用户体验 |
| 安全漏洞 | 发现安全风险 |

## 回滚粒度

### 文件级回滚
| 适用场景 | 说明 |
|----------|------|
| 单个配置文件修改 | 恢复单个文件 |
| 单个规则文件修改 | 恢复单个文件 |
| 单个技能文件修改 | 恢复单个文件 |

### 模块级回滚
| 适用场景 | 说明 |
|----------|------|
| runtime模块变更 | 恢复整个runtime目录 |
| safety模块变更 | 恢复整个safety目录 |
| governance模块变更 | 恢复整个governance目录 |

### 版本级回滚
| 适用场景 | 说明 |
|----------|------|
| 大版本升级失败 | 恢复到上一版本 |
| 多模块变更失败 | 恢复整个版本 |
| 数据结构变更失败 | 恢复整个版本 |

## 回滚流程

### 自动回滚流程
```
触发条件检测
    ↓
┌─────────────────────────────────────┐
│ 1. 确认回滚必要性                    │
│    - 验证触发条件                    │
│    - 评估影响范围                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行回滚                          │
│    - 选择回滚粒度                    │
│    - 恢复文件/模块/版本              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 处理一致性                        │
│    - 记忆一致性                      │
│    - 配置一致性                      │
│    - 技能版本一致性                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 验证回滚结果                      │
│    - 检查系统状态                    │
│    - 运行基础测试                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 通知相关人员                      │
│    - 发送回滚通知                    │
│    - 记录回滚日志                    │
└─────────────────────────────────────┘
```

### 人工回滚流程
```
人工触发回滚
    ↓
┌─────────────────────────────────────┐
│ 1. 确认回滚范围                      │
│    - 确定回滚目标版本                │
│    - 确定回滚粒度                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 备份当前状态                      │
│    - 备份当前文件                    │
│    - 备份当前配置                    │
└─────────────────────────────────────┘
    ↓
执行回滚 → 处理一致性 → 验证结果
```

## 一致性处理

### 记忆一致性
| 场景 | 处理方式 |
|------|----------|
| 记忆Schema变更回滚 | 迁移数据到旧Schema |
| 记忆写入失败回滚 | 恢复备份记忆 |
| 记忆冲突回滚 | 标记冲突，等待处理 |

### 配置一致性
| 场景 | 处理方式 |
|------|----------|
| 配置格式变更回滚 | 转换配置格式 |
| 配置值变更回滚 | 恢复旧配置值 |
| 配置缺失回滚 | 使用默认配置 |

### 技能版本一致性
| 场景 | 处理方式 |
|------|----------|
| 技能接口变更回滚 | 恢复旧技能版本 |
| 技能依赖变更回滚 | 恢复旧依赖 |
| 技能配置变更回滚 | 恢复旧配置 |

## 回滚后验证

### 基础验证
| 验证项 | 说明 |
|--------|------|
| 系统启动 | 系统正常启动 |
| 基础功能 | 核心功能可用 |
| 配置加载 | 配置正确加载 |
| 技能加载 | 技能正确加载 |

### 完整验证
| 验证项 | 说明 |
|--------|------|
| 回归测试 | 运行回归测试集 |
| 性能测试 | 验证性能指标 |
| 安全测试 | 验证安全边界 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 回滚失败 | 尝试更小粒度回滚 |
| 一致性处理失败 | 标记问题，人工处理 |
| 验证失败 | 进入安全模式 |

## 维护方式
- 新增触发条件: 添加到触发条件表
- 新增回滚粒度: 添加到粒度表
- 新增一致性处理: 更新一致性处理表
