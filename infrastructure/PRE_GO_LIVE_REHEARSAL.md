## 二、预演流程

```
预演流程
    │
    ├── 准备阶段
    │   ├── 预演计划制定
    │   ├── 预演环境准备
    │   └── 预演人员就位
    │
    ├── 执行阶段
    │   ├── 关键路径彩排
    │   ├── 审批链彩排
    │   ├── 回滚彩排
    │   └── 故障演练
    │
    └── 总结阶段
        ├── 预演结果评估
        ├── 问题记录
        └── 改进计划
```

## 三、通过标准

### 3.1 必须通过项
| 检查项 | 通过标准 |
|--------|----------|
| 核心流程 | 100%成功 |
| 审批流程 | 100%正确 |
| 回滚能力 | 回滚成功 |
| 故障恢复 | 恢复成功 |

### 3.2 可接受问题
| 问题类型 | 可接受数量 |
|----------|------------|
| P3问题 | ≤ 3个 |
| 优化建议 | 不限 |

### 3.3 阻断问题
| 问题类型 | 说明 |
|----------|------|
| P0/P1问题 | 必须解决后才能上线 |
| 回滚失败 | 必须解决后才能上线 |
| 核心流程失败 | 必须解决后才能上线 |
