# SHADOW_EVALUATION_POLICY.md - 影子评估策略

**版本: V28.0**

## 目的
定义影子评估机制，确保新技能可以在不影响生产环境的情况下进行验证。

## 适用范围
所有处于 incubating 和 candidate 阶段的技能。

---

## 一、影子评估定义

| 项目 | 说明 |
|------|------|
| 模式 | Shadow Mode（影子模式） |
| 原理 | 并行运行但不返回结果 |
| 目的 | 验证技能行为而不影响用户 |
| 阶段 | incubating / candidate |

---

## 二、影子评估原理

```
用户请求
    ↓
┌─────────────────────────────────────┐
│ 主路由                               │
│ ┌─────────────────────────────────┐ │
│ │ 生产技能 → 返回结果给用户        │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
    ↓ (并行)
┌─────────────────────────────────────┐
│ 影子路由                             │
│ ┌─────────────────────────────────┐ │
│ │ 影子技能 → 记录结果（不返回）    │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 对比分析                             │
│ - 结果差异分析                       │
│ - 性能对比                           │
│ - 错误对比                           │
└─────────────────────────────────────┘
```

---

## 三、影子评估配置

### 3.1 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| enabled | 是否启用 | true |
| sample_rate | 采样率 | 10% |
| timeout | 超时时间 | 30s |
| compare_results | 是否对比结果 | true |
| log_all | 是否记录所有 | false |

### 3.2 配置示例

```json
{
  "skill_id": "SKILL-XXX",
  "shadow_config": {
    "enabled": true,
    "sample_rate": 0.1,
    "timeout": 30,
    "compare_results": true,
    "log_all": false,
    "target_skill": "production-skill-id",
    "comparison_metrics": ["accuracy", "latency", "error_rate"]
  }
}
```

## 四、影子评估流程

### 4.1 请求处理流程

```
请求到达
    ↓
┌─────────────────────────────────────┐
│ 1. 采样判断                          │
│    - 根据采样率判断是否执行影子      │
│    - 记录采样决策                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 并行执行                          │
│    - 主技能：正常执行并返回          │
│    - 影子技能：并行执行但不返回      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 结果记录                          │
│    - 记录主技能结果                  │
│    - 记录影子技能结果                │
│    - 记录执行时间                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 对比分析                          │
│    - 结果差异分析                    │
│    - 性能对比                        │
│    - 生成评估报告                    │
└─────────────────────────────────────┘
```

## 五、评估指标

### 5.1 结果一致性

| 指标 | 说明 | 目标 |
|------|------|------|
| 完全一致 | 结果完全相同 | > 80% |
| 部分一致 | 结果部分相同 | > 90% |
| 完全不同 | 结果完全不同 | < 5% |

### 5.2 性能对比

| 指标 | 说明 | 目标 |
|------|------|------|
| 响应时间 | 影子vs主技能 | 不超过主技能2倍 |
| 错误率 | 影子vs主技能 | 不超过主技能1.5倍 |
| 资源使用 | 影子技能资源 | 在预算范围内 |

### 5.3 功能验证

| 指标 | 说明 | 目标 |
|------|------|------|
| 功能覆盖 | 覆盖的功能点 | 100% |
| 边界处理 | 边界场景处理 | 正确 |
| 错误处理 | 错误场景处理 | 正确 |

## 七、影子评估通过标准

### 7.1 incubating → candidate

| 条件 | 要求 |
|------|------|
| 影子执行次数 | > 50次 |
| 成功率 | > 90% |
| 结果一致率 | > 80% |
| 错误率 | < 10% |
| 无严重问题 | 无严重Bug |

### 7.2 candidate → beta

| 条件 | 要求 |
|------|------|
| 影子执行次数 | > 100次 |
| 成功率 | > 95% |
| 结果一致率 | > 90% |
| 错误率 | < 5% |
| 性能可接受 | 响应时间在可接受范围 |

---

## 八、影子评估终止条件

| 条件 | 说明 |
|------|------|
| 严重错误 | 发现严重Bug |
| 性能问题 | 性能严重不达标 |
| 安全问题 | 发现安全风险 |
| Owner决定 | Owner决定终止 |
| 晋升成功 | 成功晋升到下一阶段 |

---

## 九、完成标准

| 标准 | 验证方式 |
|------|----------|
| 影子配置正确 | 配置检查 |
| 采样执行正确 | 采样记录 |
| 对比分析正确 | 分析报告 |
| 评估报告完整 | 报告检查 |
