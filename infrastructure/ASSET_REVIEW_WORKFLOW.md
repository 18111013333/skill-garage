# ASSET_REVIEW_WORKFLOW.md - 资产审核工作流

## 目的
定义资产审核工作流，确保市场资产质量与安全。

## 适用范围
所有资产市场的资产审核。

---

## 一、审核流程

```
提交资产 → 机器审核 → 人工审核 → 红队抽检 → 区域审查 → 审核结论
    ↓           ↓           ↓           ↓           ↓           ↓
  作者        系统        审核        安全        合规        系统
```

---

## 二、机器审核

### 2.1 自动检查项
| 检查项 | 说明 | 标准 |
|--------|------|------|
| 格式检查 | 文件格式正确 | 符合规范 |
| 元数据检查 | 元数据完整 | 必填字段完整 |
| 安全扫描 | 恶意代码检测 | 无恶意代码 |
| 依赖扫描 | 依赖漏洞检测 | 无高危漏洞 |
| 兼容检查 | 兼容性检查 | 兼容性声明完整 |

### 2.2 自动检查时间
| 资产类型 | 检查时间 |
|----------|----------|
| 模板 | < 5 分钟 |
| 工作流 | < 10 分钟 |
| 连接器包 | < 30 分钟 |

### 2.3 自动检查结果
| 结果 | 说明 |
|------|------|
| 通过 | 进入人工审核 |
| 警告 | 进入人工审核 + 标记 |
| 失败 | 驳回 + 说明原因 |

---

## 三、人工审核

### 3.1 审核内容
| 审核项 | 审核方 | 说明 |
|--------|--------|------|
| 功能审核 | 功能团队 | 功能是否正常 |
| 文档审核 | 文档团队 | 文档是否完整 |
| 代码审核 | 开发团队 | 代码质量 |
| 安全审核 | 安全团队 | 安全合规 |

### 3.2 审核标准
| 标准 | 说明 |
|------|------|
| 功能正确 | 功能实现正确 |
| 文档完整 | 使用文档完整 |
| 代码规范 | 代码风格规范 |
| 安全合规 | 符合安全要求 |

### 3.3 审核时间
| 资产类型 | 审核时间 |
|----------|----------|
| 模板 | 1-2 天 |
| 工作流 | 2-3 天 |
| 连接器包 | 3-5 天 |

---

## 四、红队抽检

### 4.1 抽检范围
| 范围 | 说明 |
|------|------|
| 高风险资产 | 必须抽检 |
| 新发布资产 | 随机抽检 20% |
| 热门资产 | 定期抽检 |
| 用户举报 | 必须抽检 |

### 4.2 红队测试
| 测试类型 | 说明 |
|----------|------|
| 安全测试 | 安全漏洞测试 |
| 渗透测试 | 攻击模拟测试 |
| 滥用测试 | 滥用场景测试 |
| 合规测试 | 合规性测试 |

### 4.3 红队结果
| 结果 | 处理 |
|------|------|
| 通过 | 正常发布 |
| 低风险 | 发布 + 警告 |
| 中风险 | 驳回修复 |
| 高风险 | 驳回 + 封禁 |

---

## 五、区域审查

### 5.1 区域审查内容
| 审查项 | 说明 |
|--------|------|
| 数据驻留 | 数据驻留要求 |
| 隐私合规 | 隐私保护要求 |
| 内容合规 | 内容合规要求 |
| 功能合规 | 功能合规要求 |

### 5.2 区域审查规则
| 区域 | 审查要求 |
|------|----------|
| 中国 | PIPL 合规 |
| 欧洲 | GDPR 合规 |
| 美国 | CCPA 合规 |
| 其他 | 按当地法规 |

### 5.3 区域审查结果
| 结果 | 说明 |
|------|------|
| 全区域通过 | 所有区域可用 |
| 部分区域通过 | 仅通过区域可用 |
| 不通过 | 需要修改 |

---

## 六、合规标签

### 6.1 合规标签类型
| 标签 | 说明 |
|------|------|
| GDPR | 符合 GDPR 要求 |
| CCPA | 符合 CCPA 要求 |
| PIPL | 符合个人信息保护法 |
| SOC2 | 符合 SOC2 要求 |
| HIPAA | 符合健康数据保护 |
| PCI | 符合支付数据保护 |

### 6.2 标签授予
```
审核通过 → 合规验证 → 授予标签 → 展示标签
```

### 6.3 标签撤销
```
发现问题 → 验证问题 → 撤销标签 → 通知作者
```

---

## 七、审核结论

### 7.1 审核结果
| 结果 | 说明 | 后续 |
|------|------|------|
| 通过 | 满足所有要求 | 发布上架 |
| 有条件通过 | 需要修改 | 修改后发布 |
| 驳回 | 不满足要求 | 修改后重新提交 |
| 封禁 | 严重违规 | 禁止发布 |

### 7.2 审核通知
| 通知内容 | 说明 |
|----------|------|
| 审核结果 | 通过/驳回 |
| 审核意见 | 详细意见 |
| 修改建议 | 修改建议 |
| 申诉方式 | 申诉渠道 |

### 7.3 审核时效
| 阶段 | 时效 |
|------|------|
| 机器审核 | < 30 分钟 |
| 人工审核 | 1-5 天 |
| 红队抽检 | 1-3 天 |
| 区域审查 | 1-3 天 |
| 总计 | 3-10 天 |

---

## 八、复议流程

### 8.1 复议条件
| 条件 | 说明 |
|------|------|
| 审核驳回 | 对驳回结果有异议 |
| 标签撤销 | 对标签撤销有异议 |
| 下架决定 | 对下架决定有异议 |

### 8.2 复议流程
```
提交复议 → 复议审核 → 复议结论 → 执行
    ↓           ↓           ↓           ↓
  作者        复议团队    复议团队    系统
```

### 8.3 复议时效
| 阶段 | 时效 |
|------|------|
| 提交复议 | 驳回后 7 天内 |
| 复议审核 | 3-5 天 |
| 复议结论 | 审核后 1 天 |
