# WHITE_LABEL_POLICY.md - 白标输出规则

## 目的
定义白标输出的边界与规则，确保品牌可控、合规可追溯。

---

## 可替换品牌元素

| 元素 | 可替换 | 限制条件 |
|------|--------|----------|
| Logo | ✅ | 需审核备案 |
| 配色方案 | ✅ | 无限制 |
| 控制台标题 | ✅ | 需审核备案 |
| 域名 | ✅ | 需SSL证书 |
| 邮件模板 | ✅ | 需保留合规声明 |
| 报表抬头 | ✅ | 需保留数据来源 |
| 帮助文档入口 | ✅ | 可指向自有文档 |
| 沙箱提示 | ✅ | 可自定义文案 |

---

## 不可替换元素

| 元素 | 原因 |
|------|------|
| 法律声明 | 合规要求 |
| 隐私政策链接 | 法定要求 |
| 数据处理声明 | 合规要求 |
| 安全审计入口 | 安全要求 |
| 版本信息 | 可追溯要求 |
| 许可证信息 | 法律要求 |

## 白标边界

### 品牌边界
- 白标客户可使用自有品牌对外服务
- 但不得声称拥有平台核心技术
- 不得误导终端客户关于数据归属

### 合规边界
- 所有合规声明必须保留
- 数据处理责任不可转移
- 审计追踪必须完整

### 支持边界
- 白标客户负责终端用户支持
- 平台方负责技术问题支持
- 故障升级路径必须明确
