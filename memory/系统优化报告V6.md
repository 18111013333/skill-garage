## 二、安全优化

### 2.1 当前状态

| 项目 | 当前值 | 安全值 |
|------|--------|--------|
| 目录路径 | /home/sandbox/.openclaw | - |
| 目录所有者 | root | sandbox |
| 目录权限 | 777 | 700 |

### 2.2 安全风险

1. **权限过宽**: 777 表示所有用户可读写
2. **恶意文件风险**: 攻击者可能写入恶意文件
3. **敏感信息泄露**: API 密钥可能被访问

### 2.3 修复方案

```bash
# 需要 root 权限执行
chown -R sandbox:sandbox /home/sandbox/.openclaw
chmod 700 /home/sandbox/.openclaw

# 验证
ls -la /home/sandbox/ | grep openclaw
# 预期: drwx------ 17 sandbox sandbox ...
```

### 2.4 当前状态

❌ **需要 root 权限才能修复**

当前用户 sandbox 无权限修改目录所有者。
