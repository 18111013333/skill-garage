## 一、记忆层级结构

```
┌─────────────────────────────────────────────────────────┐
│                    L1 长期记忆                           │
│                    MEMORY.md                             │
│              （精选记忆，永久保存）                        │
└─────────────────────────────────────────────────────────┘
                          ▲
                          │ 提炼升级
                          │
┌─────────────────────────────────────────────────────────┐
│                    L2 中期记忆                           │
│              memory/YYYY-MM-DD.md                        │
│              （日常笔记，30天周期）                        │
└─────────────────────────────────────────────────────────┘
                          ▲
                          │ 会话结束
                          │
┌─────────────────────────────────────────────────────────┐
│                    L3 短期记忆                           │
│                  会话上下文                              │
│              （当前对话，会话结束即消失）                   │
└─────────────────────────────────────────────────────────┘
```

## 三、记忆升级规则

### 短期 → 中期
**触发条件**：
- 会话结束前有价值的信息
- 用户明确要求记录
- 完成重要任务

**执行方式**：
- 写入 `memory/YYYY-MM-DD.md`
- 添加标签

### 中期 → 长期
**触发条件**：
- 信息被检索 ≥3 次
- 用户说"记住这个"、"以后都这样"
- 决策被反复引用
- 错误教训需要长期避免

**执行方式**：
- 提炼到 `MEMORY.md`
- 在原文件标记 `[已升级]`

---

## 四、记忆清理规则

### 中期记忆清理
| 条件 | 操作 |
|------|------|
| 超过30天且无重要标记 | 删除 |
| 超过30天但有重要标记 | 归档到 archive/ |
| 已升级到长期 | 可删除 |

### 长期记忆审查
每月执行一次：
1. 检查 MEMORY.md 中的过时信息
2. 标记为 `[已过期]` 或删除
3. 更新冲突信息

---

## 五、记忆检索优先级

```
1. 当前上下文 → 会话中已有的信息
2. MEMORY.md → 长期精选记忆
3. memory/YYYY-MM-DD.md → 中期日常记忆
4. 专题文件 → shiyuantang_*.md
```

## 七、记忆质量标准

### 应该记录
- ✅ 用户明确表达的偏好
- ✅ 重要决策及其理由
- ✅ 踩过的坑和解决方案
- ✅ 反复使用的知识
- ✅ 待办事项和承诺

### 不应该记录
- ❌ 闲聊内容
- ❌ 临时性、一次性信息
- ❌ 用户明确要求不记录
- ❌ 敏感信息（密码、密钥）
