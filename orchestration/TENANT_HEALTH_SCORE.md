# TENANT_HEALTH_SCORE.md - 租户健康分模型

## 目的
定义租户健康分模型，快速判断租户运行状态。

## 适用范围
所有租户健康评估、风险预警、支持优先级判定。

## 健康分模型

### 分数范围
| 分数范围 | 状态 | 说明 |
|----------|------|------|
| 90-100 | 健康 | 运行良好 |
| 70-89 | 良好 | 轻微问题 |
| 50-69 | 关注 | 需要关注 |
| 30-49 | 警告 | 需要干预 |
| 0-29 | 危险 | 紧急处理 |

### 健康分维度
| 维度 | 权重 | 说明 |
|------|------|------|
| 使用活跃度 | 25% | 租户使用频率和深度 |
| 任务成功率 | 25% | 任务执行成功率 |
| 系统稳定性 | 20% | 服务可用性和性能 |
| 合规健康度 | 15% | 合规审查通过率 |
| 资源健康度 | 15% | 配额使用和成本控制 |

## 维度详解

### 使用活跃度 (25%)
| 指标 | 权重 | 计算方式 |
|------|------|----------|
| 日活跃用户比例 | 40% | DAU / 总用户数 |
| 会话活跃度 | 30% | 活跃会话 / 总会话 |
| 功能使用广度 | 20% | 使用功能数 / 总功能数 |
| 使用深度 | 10% | 高级功能使用比例 |

**评分标准**:
| 指标值 | 得分 |
|--------|------|
| > 80% | 100 |
| 60-80% | 80 |
| 40-60% | 60 |
| 20-40% | 40 |
| < 20% | 20 |

### 任务成功率 (25%)
| 指标 | 权重 | 计算方式 |
|------|------|----------|
| 总体成功率 | 50% | 成功任务 / 总任务 |
| 核心任务成功率 | 30% | 核心任务成功 / 核心任务总数 |
| 错误恢复率 | 20% | 恢复的错误 / 总错误 |

**评分标准**:
| 指标值 | 得分 |
|--------|------|
| > 98% | 100 |
| 95-98% | 80 |
| 90-95% | 60 |
| 80-90% | 40 |
| < 80% | 20 |

### 系统稳定性 (20%)
| 指标 | 权重 | 计算方式 |
|------|------|----------|
| SLO 达成率 | 40% | 达成 SLO 数 / 总 SLO 数 |
| 平均响应时间 | 30% | 与基准对比 |
| 错误率 | 30% | 1 - 错误率 |

**评分标准**:
| 指标值 | 得分 |
|--------|------|
| > 99% | 100 |
| 95-99% | 80 |
| 90-95% | 60 |
| 80-90% | 40 |
| < 80% | 20 |

### 合规健康度 (15%)
| 指标 | 权重 | 计算方式 |
|------|------|----------|
| 审查通过率 | 50% | 一次通过 / 总审查 |
| 例外比例 | 30% | 1 - 例外数 / 总操作 |
| 审计问题数 | 20% | 1 - 问题数 / 基准 |

**评分标准**:
| 指标值 | 得分 |
|--------|------|
| > 95% | 100 |
| 90-95% | 80 |
| 80-90% | 60 |
| 70-80% | 40 |
| < 70% | 20 |

### 资源健康度 (15%)
| 指标 | 权重 | 计算方式 |
|------|------|----------|
| 配额使用率 | 40% | 合理使用比例 |
| 成本效率 | 30% | 产出 / 成本 |
| 资源浪费率 | 30% | 1 - 浪费资源 / 总资源 |

**评分标准**:
| 指标值 | 得分 |
|--------|------|
| 60-80% | 100 |
| 40-60% 或 80-90% | 80 |
| 20-40% 或 90-95% | 60 |
| < 20% 或 > 95% | 40 |
| > 100% | 20 |

## 扣分项

### 直接扣分
| 扣分项 | 扣分值 | 说明 |
|--------|--------|------|
| Sev1 事故 | -20 | 发生 Sev1 事故 |
| Sev2 事故 | -10 | 发生 Sev2 事故 |
| 合规违规 | -15 | 合规审查违规 |
| 安全事件 | -20 | 发生安全事件 |
| 数据问题 | -15 | 数据丢失或损坏 |
| 配额超限 | -5 | 配额超限 |
| 长期不活跃 | -10 | 30 天无活跃 |

### 累计扣分上限
- 单次扣分上限: 20 分
- 月度累计扣分上限: 50 分

## 恢复项

### 恢复加分
| 恢复项 | 加分值 | 条件 |
|--------|--------|------|
| 问题解决 | +5 | 问题完全解决 |
| 改进措施落实 | +10 | 改进措施全部落实 |
| 连续无事故 | +5 | 连续 30 天无事故 |
| 合规提升 | +5 | 合规评分提升 10% |
| 使用增长 | +5 | 使用量增长 20% |

### 恢复限制
- 单次恢复上限: 10 分
- 月度累计恢复上限: 30 分
- 恢复后分数不超过 100

## 高风险租户判定

### 判定条件
| 条件 | 说明 |
|------|------|
| 健康分 < 50 | 连续 7 天健康分低于 50 |
| 连续下降 | 健康分连续 14 天下降 |
| 多维度异常 | 3 个以上维度同时异常 |
| 重大事故 | 发生 Sev1 事故 |
| 合规严重违规 | 合规审查严重违规 |

### 高风险租户处理
| 处理 | 说明 |
|------|------|
| 标记高风险 | 标记为高风险租户 |
| 通知租户 | 通知租户 Owner |
| 专项跟进 | 指定专人跟进 |
| 限制自治 | 限制高风险自治能力 |
| 增加监控 | 增加监控频率 |

## 健康分应用

### 支持策略
| 健康分 | 支持策略 |
|--------|----------|
| 90-100 | 标准支持 |
| 70-89 | 定期关怀 |
| 50-69 | 主动介入 |
| 30-49 | 专项支持 |
| 0-29 | 紧急干预 |

### 运营优先级
| 健康分 | 优先级 | 说明 |
|--------|--------|------|
| 0-29 | P0 | 最高优先级处理 |
| 30-49 | P1 | 高优先级处理 |
| 50-69 | P2 | 中优先级处理 |
| 70-89 | P3 | 低优先级处理 |
| 90-100 | P4 | 标准处理 |
