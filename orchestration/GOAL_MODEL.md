# GOAL_MODEL.md - 用户目标模型

## 目的
定义用户目标模型，确保系统围绕目标交付，而不是只围绕句子作答。

## 适用范围
所有任务执行中的目标识别和追踪。

## 目标层级

### 当前会话目标
| 特征 | 说明 |
|------|------|
| 生命周期 | 当前会话有效 |
| 来源 | 用户当前请求 |
| 优先级 | 最高 |
| 可变性 | 可随会话变化 |

### 阶段性目标
| 特征 | 说明 |
|------|------|
| 生命周期 | 数天到数周 |
| 来源 | 项目或任务 |
| 优先级 | 中等 |
| 可变性 | 相对稳定 |

### 长期目标
| 特征 | 说明 |
|------|------|
| 生命周期 | 数月到数年 |
| 来源 | 用户画像 |
| 优先级 | 基础参考 |
| 可变性 | 相对稳定 |

## 目标识别

### 表面问题 vs 真正目标
| 表面问题 | 真正目标 |
|----------|----------|
| "怎么写Python代码" | 学习Python编程 |
| "这个报错怎么解决" | 完成项目开发 |
| "帮我写个报告" | 完成工作汇报 |
| "搜索XX信息" | 做出决策 |

### 目标识别方法
| 方法 | 说明 |
|------|------|
| 直接陈述 | 用户直接说"我要..." |
| 上下文推断 | 从上下文推断目标 |
| 历史关联 | 关联历史目标 |
| 任务模式 | 识别常见任务模式 |

## 目标模型结构

```json
{
  "goalId": "goal_001",
  "level": "session",
  "description": "完成Python项目开发",
  "success_criteria": [
    "代码能运行",
    "通过测试",
    "符合需求"
  ],
  "constraints": [
    "使用Python 3.8+",
    "代码规范PEP8"
  ],
  "sub_goals": [
    {
      "goalId": "sub_001",
      "description": "实现核心功能",
      "status": "in_progress"
    }
  ],
  "priority": "high",
  "deadline": null,
  "created_at": "2026-04-06T10:00:00+08:00",
  "updated_at": "2026-04-06T10:32:00+08:00"
}
```

## 目标追踪

### 目标状态
| 状态 | 说明 |
|------|------|
| identified | 已识别 |
| in_progress | 进行中 |
| blocked | 阻塞 |
| completed | 已完成 |
| abandoned | 已放弃 |

### 目标进度
```
目标进度 = 已完成子目标数 / 总子目标数
```

### 目标检查点
| 检查点 | 说明 |
|--------|------|
| 目标识别 | 确认目标已识别 |
| 子目标完成 | 子目标完成检查 |
| 目标完成 | 目标完成检查 |
| 目标验证 | 验证目标达成 |

## 目标与任务关联

### 目标分解
```
长期目标
    ↓
阶段性目标
    ↓
会话目标
    ↓
具体任务
```

### 任务对齐
| 检查项 | 说明 |
|--------|------|
| 任务是否服务于目标 | 任务应服务于目标 |
| 任务优先级是否正确 | 高优先级任务优先 |
| 任务是否偏离目标 | 防止任务偏离 |

## 目标变更处理

### 目标修正
| 情况 | 处理 |
|------|------|
| 用户明确变更 | 更新目标，调整任务 |
| 发现目标错误 | 确认后修正目标 |
| 目标已完成 | 标记完成，识别新目标 |

### 目标冲突
| 情况 | 处理 |
|------|------|
| 新旧目标冲突 | 请求用户确认 |
| 多目标冲突 | 按优先级处理 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 目标无法识别 | 请求用户澄清 |
| 目标过于模糊 | 分解为具体子目标 |
| 目标无法达成 | 建议调整目标 |

## 维护方式
- 新增目标类型: 添加到目标层级表
- 新增识别方法: 更新识别方法表
- 新增检查点: 更新检查点表
