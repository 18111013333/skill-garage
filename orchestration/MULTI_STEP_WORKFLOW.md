# MULTI_STEP_WORKFLOW.md - 多步任务工作流规范

## 目的
专门定义多步任务工作流规范，确保复杂任务执行更稳。

## 适用范围
所有多步任务执行。

## 中间结果保存

### 保存时机
| 时机 | 说明 |
|------|------|
| 步骤完成 | 每个步骤完成后保存 |
| 阶段完成 | 阶段性成果保存 |
| 用户请求 | 用户请求时保存 |
| 异常中断 | 异常中断时保存 |

### 保存内容
| 内容 | 说明 |
|------|------|
| 步骤结果 | 当前步骤执行结果 |
| 进度状态 | 整体进度状态 |
| 上下文 | 当前上下文信息 |
| 时间戳 | 保存时间 |

### 保存格式
```json
{
  "checkpointId": "cp_001",
  "workflowId": "wf_001",
  "step": 5,
  "totalSteps": 10,
  "result": {
    "data": "...",
    "status": "success"
  },
  "context": {
    "goal": "...",
    "constraints": []
  },
  "timestamp": "2026-04-06T10:32:00+08:00"
}
```

## 阶段结果回传

### 回传时机
| 条件 | 说明 |
|------|------|
| 重要阶段完成 | 关键阶段完成时 |
| 用户请求 | 用户请求查看进度 |
| 长时间执行 | 执行超过阈值时间 |
| 需要用户决策 | 需要用户决策时 |

### 回传内容
| 内容 | 说明 |
|------|------|
| 当前进度 | 完成百分比 |
| 阶段成果 | 当前阶段成果 |
| 下一步计划 | 接下来的步骤 |
| 预计时间 | 预计完成时间 |

### 回传格式
```
[进度更新]
已完成: {已完成步骤}/{总步骤} ({百分比}%)
当前阶段: {阶段名称}
阶段成果: {成果摘要}
下一步: {下一步描述}
预计完成: {预计时间}
```

## 终止条件

### 正常终止
| 条件 | 说明 |
|------|------|
| 目标达成 | 任务目标已完成 |
| 用户确认完成 | 用户确认任务完成 |
| 所有步骤完成 | 所有步骤执行完成 |

### 异常终止
| 条件 | 说明 |
|------|------|
| 用户取消 | 用户主动取消 |
| 错误无法恢复 | 错误无法恢复 |
| 资源超限 | 资源超过限制 |
| 超时 | 执行超时 |

### 部分终止
| 条件 | 说明 |
|------|------|
| 部分目标达成 | 部分目标已达成 |
| 用户要求停止 | 用户要求停止 |
| 继续成本过高 | 继续执行成本过高 |

## 用户改需求处理

### 需求变更检测
| 情况 | 检测方式 |
|------|----------|
| 明确变更 | 用户明确说"改成..." |
| 隐含变更 | 新请求与当前目标冲突 |
| 目标偏移 | 检测到目标偏移 |

### 变更处理策略
| 策略 | 适用场景 |
|------|----------|
| 立即调整 | 变更明确且影响大 |
| 完成后调整 | 变更可延后处理 |
| 分支执行 | 可并行处理 |
| 重新规划 | 变更根本性改变 |

### 变更处理流程
```
检测到需求变更
    ↓
┌─────────────────────────────────────┐
│ 1. 评估变更影响                      │
│    - 影响范围                        │
│    - 已完成工作                      │
│    - 需要调整的步骤                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 选择处理策略                      │
│    - 立即调整                        │
│    - 完成后调整                      │
│    - 重新规划                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行变更                          │
│    - 保存当前状态                    │
│    - 调整执行计划                    │
│    - 继续执行                        │
└─────────────────────────────────────┘
    ↓
继续执行
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 步骤失败 | 重试或跳过 |
| 依赖失败 | 等待或调整 |
| 资源不足 | 降级或等待 |
| 用户无响应 | 超时后继续或终止 |
