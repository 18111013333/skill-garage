# SERVICE_OWNERSHIP.md - 服务归属与责任

**版本: V27.0**

## 目的
定义服务归属与责任，确保每个服务都有人负责，不再只有模块没有责任归属。

## 适用范围
所有平台正式服务，包括：
- 核心服务
- 关键服务
- 标准服务
- 辅助服务
- 实验服务

---

## 一、服务角色定义

| 角色 | 代码 | 说明 | 主要职责 |
|------|------|------|----------|
| 服务所有者 | SERVICE_OWNER | 服务最终负责人 | 服务战略、重大决策、资源协调 |
| 备份所有者 | BACKUP_OWNER | 服务备用负责人 | Owner 缺席时代理 |
| 运营所有者 | OPS_OWNER | 服务运营负责人 | 日常运营、监控、维护 |
| 事故所有者 | INCIDENT_OWNER | 事故处理负责人 | 事故响应、根因分析、复盘 |
| 变更所有者 | CHANGE_OWNER | 变更管理负责人 | 变更审批、发布管理、回滚 |

---

## 二、服务所有者 (SERVICE_OWNER)

### 2.1 职责

| 职责 | 说明 |
|------|------|
| 服务战略 | 制定服务发展方向和路线图 |
| 重大决策 | 服务重大变更决策 |
| 资源协调 | 协调服务所需资源 |
| 团队建设 | 建设服务团队 |
| 对外沟通 | 服务相关对外沟通 |
| 最终责任 | 对服务负最终责任 |

### 2.2 权限

| 权限 | 说明 |
|------|------|
| 服务决策权 | 服务相关决策权 |
| 资源调配权 | 服务资源调配权 |
| 人员指派权 | 服务人员指派权 |
| 变更审批权 | 服务变更审批权 |
| 事故升级权 | 事故升级决策权 |

### 2.3 要求

| 要求 | 说明 |
|------|------|
| 响应时间 | 紧急情况 30 分钟内响应 |
| 可用性 | 工作时间可用，非工作时间有代理 |
| 授权 | 可授权给备份所有者 |
| 培训 | 需完成服务管理培训 |

## 三、备份所有者 (BACKUP_OWNER)

### 3.1 职责

| 职责 | 说明 |
|------|------|
| 代理职责 | Owner 缺席时代理职责 |
| 协助决策 | 协助 Owner 做决策 |
| 知识备份 | 保持服务知识同步 |
| 临时负责 | Owner 离职时临时负责 |

### 3.2 代理场景

| 场景 | 说明 |
|------|------|
| Owner 休假 | Owner 休假期间代理 |
| Owner 离职 | Owner 离职后临时负责 |
| Owner 忙碌 | Owner 无法及时响应时协助 |
| Owner 缺席 | Owner 无法出席时代理 |

### 3.3 要求

| 要求 | 说明 |
|------|------|
| 知识同步 | 与 Owner 保持知识同步 |
| 权限相同 | 具有与 Owner 相同权限 |
| 响应时间 | 紧急情况 1 小时内响应 |

## 七、升级路径

### 7.1 升级矩阵

| 场景 | 第一级 | 第二级 | 第三级 | 第四级 |
|------|--------|--------|--------|--------|
| 服务问题 | Ops Owner | Service Owner | 技术总监 | CTO |
| 事故升级 | Incident Owner | Service Owner | 技术总监 | CTO |
| 变更争议 | Change Owner | Service Owner | 技术总监 | CTO |
| 资源问题 | Service Owner | 技术总监 | VP | CEO |
| 战略问题 | Service Owner | 技术总监 | VP | CEO |

### 7.2 升级触发条件

| 条件 | 说明 |
|------|------|
| 响应超时 | 超过响应时限未响应 |
| 处理超时 | 超过处理时限未解决 |
| 权限不足 | 需要更高级别权限 |
| 影响扩大 | 影响范围扩大 |
| 争议升级 | 存在争议无法解决 |

### 7.3 升级流程

```
升级触发
    ↓
┌─────────────────────────────────────┐
│ 1. 升级判断                          │
│    - 判断升级条件                    │
│    - 确定升级目标                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 信息传递                          │
│    - 准备升级信息                    │
│    - 通知升级目标                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 接手确认                          │
│    - 升级目标确认接手                │
│    - 继续处理                        │
└─────────────────────────────────────┘
```

|------|---------------|--------------|-----------|----------------|--------------|
| 服务战略 | A/R | C | C | I | C |
| 日常运营 | A | I | R | C | C |
| 监控告警 | A | I | R | C | I |
| 事故处理 | A | I | C | R | C |
| 变更审批 | A | C | C | I | R |
| 发布管理 | A | I | C | I | R |
| 容量规划 | A | C | R | I | C |
| 成本管理 | A | I | R | I | C |
| 文档维护 | A | C | R | C | C |

**图例：** R=负责执行，A=最终审批，C=需要咨询，I=需要通知

## 九、Owner 变更

### 9.1 变更场景

| 场景 | 说明 | 处理方式 |
|------|------|----------|
| 主动变更 | Owner 主动申请变更 | 审批后变更 |
| 离职 | Owner 离职 | 备份 Owner 临时负责 |
| 调岗 | Owner 调岗 | 重新指派 |
