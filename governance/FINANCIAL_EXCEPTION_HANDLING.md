# FINANCIAL_EXCEPTION_HANDLING.md - 终极鸽子王生态财务异常处理

## 目的
定义生态财务异常的标准处理流程。

## 适用范围
所有渠道伙伴、市场开发者的财务异常处理。

---

## 一、异常类型

### 1.1 结算失败
| 失败原因 | 处理方式 |
|----------|----------|
| 账户信息错误 | 联系更新账户信息 |
| 银行拒绝 | 核实原因后重试 |
| 金额超限 | 分批结算 |
| 系统故障 | 故障修复后重试 |

### 1.2 渠道退款
| 退款场景 | 处理方式 |
|----------|----------|
| 客户退款 | 扣除已结算分成 |
| 服务退款 | 按比例扣除 |
| 争议退款 | 争议解决后处理 |

### 1.3 资产下架后收益
| 场景 | 处理方式 |
|------|----------|
| 已销售未结算 | 正常结算 |
| 已销售已结算 | 不追回 |
| 待处理订单 | 按情况处理 |

### 1.4 违规收益冻结
| 违规类型 | 冻结比例 |
|----------|----------|
| 轻微违规 | 20% |
| 一般违规 | 50% |
| 严重违规 | 100% |

### 1.5 对账争议升级
| 升级条件 | 升级目标 |
|----------|----------|
| 差异无法协商 | 财务总监 |
| 涉及金额 > 10万 | 高管审批 |
| 法律风险 | 法务介入 |

## 二、处理流程

### 2.1 结算失败处理
```
结算失败处理
    ↓
┌─────────────────────────────────────┐
│ 1. 失败识别                          │
│    - 接收失败通知                    │
│    - 分析失败原因                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 原因处理                          │
│    - 账户错误 → 更新信息             │
│    - 银行拒绝 → 核实重试             │
│    - 系统故障 → 等待修复             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 重试结算                          │
│    - 问题解决后重试                  │
│    - 记录处理过程                    │
└─────────────────────────────────────┘
```

### 2.2 违规收益冻结处理
```
违规收益冻结
    ↓
┌─────────────────────────────────────┐
│ 1. 违规认定                          │
│    - 确认违规事实                    │
│    - 评估违规等级                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 冻结执行                          │
│    - 按比例冻结收益                  │
│    - 通知伙伴                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 后续处理                          │
│    - 违规解决 → 解冻                 │
│    - 违规确认 → 扣除                 │
└─────────────────────────────────────┘
```

### 2.3 对账争议升级处理
```
对账争议升级
    ↓
┌─────────────────────────────────────┐
│ 1. 升级申请                          │
│    - 说明争议内容                    │
│    - 提供证据材料                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 升级审核                          │
│    - 上级审核争议                    │
│    - 必要时调查核实                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 升级决定                          │
│    - 出具处理决定                    │
│    - 执行处理决定                    │
└─────────────────────────────────────┘
```

## 三、异常记录

### 3.1 记录内容
| 记录项 | 说明 |
|--------|------|
| 异常ID | 唯一标识 |
| 异常类型 | 异常分类 |
| 发生时间 | 发现时间 |
| 涉及金额 | 相关金额 |
| 处理过程 | 处理步骤 |
| 处理结果 | 最终结果 |

### 3.2 记录保留
| 记录类型 | 保留期限 |
|----------|----------|
| 异常记录 | 3年 |
| 处理记录 | 3年 |
| 证据材料 | 争议解决后1年 |
