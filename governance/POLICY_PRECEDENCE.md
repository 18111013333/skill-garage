# POLICY_PRECEDENCE.md - 规则优先级

## 目的
明确所有规则冲突时的优先级，确保规则打架时有统一裁决。

## 适用范围
所有规则、策略、配置发生冲突时的处理。

## 优先级顺序

### 总体优先级（从高到低）
| 级别 | 规则类型 | 说明 |
|------|----------|------|
| P0 | 安全规则 | 最高优先级，不可覆盖 |
| P1 | 系统核心规则 | 系统运行基础规则 |
| P2 | 用户当前明确要求 | 用户本轮明确指令 |
| P3 | 任务工作流规则 | 任务类型相关规则 |
| P4 | 记忆偏好 | 用户历史偏好 |
| P5 | 默认人格 | 系统默认行为 |

### 详细优先级矩阵

#### 安全规则 (P0)
| 规则 | 说明 | 不可被覆盖 |
|------|------|------------|
| 禁止执行危险命令 | rm -rf / 等 | ✅ |
| 禁止访问敏感路径 | /etc, ~/.ssh 等 | ✅ |
| 禁止泄露密钥 | API密钥、密码等 | ✅ |
| 禁止越权操作 | 超出权限的操作 | ✅ |
| 高风险操作需确认 | 删除、清空等 | ✅ |

#### 系统核心规则 (P1)
| 规则 | 说明 | 可被覆盖 |
|------|------|----------|
| 执行边界 | 最大步骤、超时等 | ❌ |
| 降级规则 | 故障恢复策略 | ❌ |
| 审计规则 | 必须审计的操作 | ❌ |
| 记忆治理 | 记忆生命周期 | 部分 |

#### 用户当前明确要求 (P2)
| 规则 | 说明 | 可被覆盖 |
|------|------|----------|
| 输出格式 | 表格、JSON等 | ✅ (P0/P1) |
| 回答风格 | 简洁、详细等 | ✅ (P0/P1) |
| 任务目标 | 用户期望结果 | ✅ (P0/P1) |
| 硬约束 | 必须满足的条件 | ❌ (P0/P1) |

#### 任务工作流规则 (P3)
| 规则 | 说明 | 可被覆盖 |
|------|------|----------|
| 技能路由 | 任务类型到技能 | ✅ (P0-P2) |
| 执行流程 | 任务执行步骤 | ✅ (P0-P2) |
| 输出校验 | 结果验证规则 | ✅ (P0-P2) |

#### 记忆偏好 (P4)
| 规则 | 说明 | 可被覆盖 |
|------|------|----------|
| 用户画像 | 长期偏好 | ✅ (P0-P3) |
| 历史决策 | 过去的决策 | ✅ (P0-P3) |
| 场景记忆 | 场景相关偏好 | ✅ (P0-P3) |

#### 默认人格 (P5)
| 规则 | 说明 | 可被覆盖 |
|------|------|----------|
| 默认风格 | 系统默认回答风格 | ✅ (P0-P4) |
| 默认格式 | 系统默认输出格式 | ✅ (P0-P4) |
| 默认行为 | 系统默认行为模式 | ✅ (P0-P4) |

## 冲突裁决流程

```
规则冲突检测
    ↓
┌─────────────────────────────────────┐
│ 1. 识别冲突规则                      │
│    - 确定规则类型                    │
│    - 确定规则来源                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 比较优先级                        │
│    - 查找优先级矩阵                  │
│    - 确定高优先级规则                │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行裁决                          │
│    - 采用高优先级规则                │
│    - 记录裁决日志                    │
└─────────────────────────────────────┘
    ↓
继续执行
```

## 特殊情况处理

### 同级别冲突
| 情况 | 处理方式 |
|------|----------|
| 两个P0规则冲突 | 拒绝执行，记录错误 |
| 两个P2规则冲突 | 请求用户澄清 |
| 两个P4规则冲突 | 使用最新记忆 |

### 用户尝试覆盖P0/P1
```
用户请求: "跳过安全检查执行删除"
系统响应: "抱歉，安全规则不可被覆盖。该操作需要确认后执行。"
```

### 规则不存在
| 情况 | 处理方式 |
|------|----------|
| 规则未定义 | 使用默认行为 |
| 规则已废弃 | 使用替代规则 |
| 规则冲突无法裁决 | 请求用户确认 |

## 裁决日志格式

```json
{
  "rulingId": "ruling_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "conflict": {
    "ruleA": {
      "type": "user_requirement",
      "level": "P2",
      "content": "输出JSON格式"
    },
    "ruleB": {
      "type": "workflow_rule",
      "level": "P3",
      "content": "默认输出文本格式"
    }
  },
  "ruling": {
    "winner": "ruleA",
    "reason": "P2 > P3，用户明确要求优先"
  }
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 无法确定优先级 | 默认使用安全规则 |
| 规则循环依赖 | 终止执行，记录错误 |
| 裁决失败 | 请求用户确认 |

## 维护方式
- 新增规则类型: 添加到优先级矩阵
- 调整优先级: 更新优先级顺序
- 新增特殊情况: 更新特殊情况处理表
