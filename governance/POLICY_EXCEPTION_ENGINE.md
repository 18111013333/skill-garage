# POLICY_EXCEPTION_ENGINE.md - 策略例外执行机制

**版本: V27.0**

## 目的
定义策略例外执行机制，确保策略例外不再靠人工记忆处理，实现"例外可申请、范围可限定、自动可失效"。

## 适用范围
所有策略例外场景，包括：
- 例外申请
- 限定范围
- 时间限制
- 审计记录
- 自动失效
- 补偿控制

---

## 一、例外类型

| 类型 | 代码 | 说明 | 最长有效期 |
|------|------|------|------------|
| 临时例外 | TEMPORARY | 临时性例外 | 7天 |
| 项目例外 | PROJECT | 项目期间例外 | 项目周期 |
| 租户例外 | TENANT | 租户级别例外 | 30天 |
| 紧急例外 | EMERGENCY | 紧急情况例外 | 24小时 |

## 二、例外申请

### 2.1 申请内容

| 申请项 | 说明 | 必填 |
|--------|------|------|
| 策略ID | 申请例外的策略 | 是 |
| 申请原因 | 例外申请原因 | 是 |
| 申请范围 | 例外适用范围 | 是 |
| 申请时限 | 例外有效期 | 是 |
| 补偿措施 | 补偿控制措施 | 是 |

### 2.2 申请流程

```
例外申请
    ↓
┌─────────────────────────────────────┐
│ 1. 申请提交                          │
│    - 提交例外申请                    │
│    - 附带申请理由                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 影响评估                          │
│    - 评估例外影响                    │
│    - 评估风险等级                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 审批流程                          │
│    - 路由到审批人                    │
│    - 等待审批结果                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 例外生效                          │
│    - 审批通过后生效                  │
│    - 记录例外信息                    │
└─────────────────────────────────────┘
```
