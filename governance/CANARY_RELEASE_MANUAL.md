# CANARY_RELEASE_MANUAL.md - 金丝雀发布手册

## 目的
定义金丝雀发布的标准流程和操作规范。

## 适用范围
所有金丝雀发布、灰度发布、渐进式发布。

## 金丝雀发布原理

### 发布模式
```
┌─────────────────────────────────────────────────────────────┐
│                    金丝雀发布模式                            │
└─────────────────────────────────────────────────────────────┘

旧版本 (90%)          新版本 (10%)
┌──────────┐         ┌──────────┐
│          │         │          │
│  稳定版本 │         │ 金丝雀版本 │
│          │         │          │
└──────────┘         └──────────┘
     │                     │
     └─────────┬───────────┘
               │
               ▼
         监控对比
               │
        ┌──────┴──────┐
        ▼             ▼
    指标正常       指标异常
        │             │
        ▼             ▼
    继续放量       自动回滚
```

### 发布策略
| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 百分比放量 | 按流量比例放量 | 通用场景 |
| 用户分组 | 按用户分组放量 | 用户隔离场景 |
| 地域分组 | 按地域分组放量 | 地域隔离场景 |
| 特性开关 | 按特性开关控制 | 功能开关场景 |

## 金丝雀发布流程

### 标准流程
```
┌─────────────────────────────────────────────────────────────┐
│ 1. 发布准备                                                  │
│    - 确认发布内容                                            │
│    - 准备监控指标                                            │
│    - 配置回滚条件                                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 初始放量 (1-5%)                                           │
│    - 部署金丝雀版本                                          │
│    - 配置流量比例                                            │
│    - 开始监控                                                │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 指标验证                                                  │
│    - 对比新旧版本指标                                        │
│    - 检查错误率                                              │
│    - 检查性能指标                                            │
└─────────────────────────────────────────────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
              指标正常              指标异常
                    │                   │
                    ▼                   ▼
┌───────────────────────────┐  ┌───────────────────────────┐
│ 4a. 继续放量               │  │ 4b. 自动回滚               │
│    - 增加流量比例          │  │    - 触发回滚              │
│    - 持续监控              │  │    - 恢复旧版本            │
└───────────────────────────┘  └───────────────────────────┘
                    │                   │
                    ▼                   ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 全量发布/回滚完成                                         │
│    - 完成全量发布                                            │
│    - 或完成回滚                                              │
│    - 记录发布结果                                            │
└─────────────────────────────────────────────────────────────┘
```

## 放量策略

### 放量比例
| 阶段 | 流量比例 | 持续时间 | 验证内容 |
|------|----------|----------|----------|
| 阶段 1 | 1% | 10 分钟 | 基本功能 |
| 阶段 2 | 5% | 20 分钟 | 错误率 |
| 阶段 3 | 10% | 30 分钟 | 性能指标 |
| 阶段 4 | 25% | 30 分钟 | 业务指标 |
| 阶段 5 | 50% | 30 分钟 | 稳定性 |
| 阶段 6 | 100% | - | 全量发布 |

### 放量条件
| 条件 | 说明 |
|------|------|
| 错误率 | 新版本错误率 ≤ 旧版本 |
| 响应时间 | 新版本响应时间 ≤ 旧版本 × 1.1 |
| 业务指标 | 关键业务指标正常 |
| 无严重告警 | 无 P0/P1 告警 |

## 回滚触发

### 自动回滚条件
| 条件 | 阈值 | 说明 |
|------|------|------|
| 错误率上升 | > 2 倍 | 自动回滚 |
| 响应时间上升 | > 2 倍 | 自动回滚 |
| P0 告警 | 任何 P0 告警 | 自动回滚 |
| 业务指标下降 | > 20% | 自动回滚 |

### 手动回滚条件
| 条件 | 说明 |
|------|------|
| 发现严重问题 | 需要人工判断 |
| 用户反馈异常 | 用户报告问题 |
| 依赖服务问题 | 外部因素 |
| 决策需要 | 需要人工决策 |

## 监控指标

### 核心指标
| 指标 | 说明 | 对比方式 |
|------|------|----------|
| 错误率 | 请求错误比例 | 新旧对比 |
| 响应时间 | 请求响应时间 | 新旧对比 |
| 吞吐量 | 请求处理量 | 新旧对比 |
| 资源使用 | CPU/内存使用 | 新旧对比 |

### 业务指标
| 指标 | 说明 | 对比方式 |
|------|------|----------|
| 转化率 | 业务转化率 | 新旧对比 |
| 成功率 | 业务成功率 | 新旧对比 |
| 用户满意度 | 用户反馈 | 新旧对比 |
