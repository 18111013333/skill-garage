# POLICY_AUTHORING_GUIDELINES.md - 策略编写规范

**版本: V27.0**

## 目的
定义策略编写规范，确保策略不会越写越乱，而是有统一写法，实现"命名规范、作用域明确、测试完整"。

## 适用范围
所有策略编写场景，包括：
- 命名规范
- 作用域定义
- 默认拒绝原则
- 例外定义
- 依赖声明
- 风险说明
- 测试要求

---

## 一、命名规范

### 1.1 策略ID命名

| 规则 | 说明 | 示例 |
|------|------|------|
| 前缀 | pac_ | pac_access_001 |
| 领域 | 领域代码 | access, security, compliance |
| 序号 | 3位序号 | 001, 002 |

### 1.2 策略名称命名

| 规则 | 说明 | 示例 |
|------|------|------|
| 语言 | 中文或英文 | 访问控制策略 |
| 简洁 | 简洁明确 | 数据保留策略 |
| 一致 | 同类策略一致 | XX策略 |

---

## 二、作用域定义

### 2.1 作用域声明

| 作用域 | 说明 | 示例 |
|--------|------|------|
| global | 全局作用域 | 全局访问策略 |
| tenant | 租户作用域 | 租户数据策略 |
| project | 项目作用域 | 项目权限策略 |
| user | 用户作用域 | 用户配额策略 |

### 2.2 作用域限制

| 限制项 | 说明 |
|--------|------|
| 单一作用域 | 策略只能有一个作用域 |
| 明确声明 | 作用域必须明确声明 |
| 不越界 | 策略不能越界执行 |

---

## 三、默认拒绝原则

### 3.1 原则定义

| 原则 | 说明 |
|------|------|
| 默认拒绝 | 无明确允许则拒绝 |
| 显式允许 | 只有显式允许才允许 |
| 最小权限 | 只授予最小必要权限 |

### 3.2 实现方式

```json
{
  "default_decision": "deny",
  "rules": [
    {
      "condition": { ... },
      "action": { "type": "allow" }
    }
  ]
}
```

## 四、例外定义

### 4.1 例外声明

| 声明项 | 说明 |
|--------|------|
| 例外条件 | 例外触发条件 |
| 例外范围 | 例外适用范围 |
| 例外时限 | 例外有效期 |
| 例外审批 | 例外审批要求 |

### 4.2 例外格式

```json
{
  "exception_mode": {
    "allow_exceptions": true,
    "exception_approval": "manual",
    "exception_duration_max": "30d"
  }
}
```

## 五、依赖声明

### 5.1 依赖类型

| 类型 | 说明 |
|------|------|
| requires | 必须依赖 |
| conflicts | 冲突依赖 |
| extends | 扩展依赖 |
| overrides | 覆盖依赖 |

### 5.2 依赖格式

```json
{
  "dependencies": [
    {
      "policy_id": "pac_base_001",
      "dependency_type": "requires"
    }
  ]
}
```
