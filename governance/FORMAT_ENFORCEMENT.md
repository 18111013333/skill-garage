# FORMAT_ENFORCEMENT.md - 格式强制策略

## 目的
定义输出格式强制策略，确保输出样式稳定，不容易跑偏。

## 适用范围
所有需要特定格式输出的回答。

## 格式类型

### 表格格式
| 触发词 | 说明 |
|--------|------|
| "用表格" | 输出表格格式 |
| "列个表" | 输出表格格式 |
| "表格形式" | 输出表格格式 |

**格式要求**:
```
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

### 清单格式
| 触发词 | 说明 |
|--------|------|
| "列个清单" | 输出清单格式 |
| "列出要点" | 输出清单格式 |
| "逐条列出" | 输出清单格式 |

**格式要求**:
```
1. {要点1}
2. {要点2}
3. {要点3}
```

### JSON格式
| 触发词 | 说明 |
|--------|------|
| "JSON格式" | 输出JSON格式 |
| "返回JSON" | 输出JSON格式 |
| "JSON对象" | 输出JSON格式 |

**格式要求**:
```json
{
  "key1": "value1",
  "key2": "value2"
}
```

### 步骤格式
| 触发词 | 说明 |
|--------|------|
| "步骤" | 输出步骤格式 |
| "分步骤" | 输出步骤格式 |
| "一步步" | 输出步骤格式 |

**格式要求**:
```
步骤1: {描述}
步骤2: {描述}
步骤3: {描述}
```

### 邮件格式
| 触发词 | 说明 |
|--------|------|
| "写封邮件" | 输出邮件格式 |
| "邮件格式" | 输出邮件格式 |

**格式要求**:
```
收件人: {收件人}
主题: {主题}

{正文}

此致
敬礼
```

### 合同文本格式
| 触发词 | 说明 |
|--------|------|
| "合同格式" | 输出合同格式 |
| "协议格式" | 输出协议格式 |

**格式要求**:
```
{标题}

甲方: {甲方}
乙方: {乙方}

第一条 {条款标题}
{条款内容}

...
```

## 格式校正流程

```
格式要求识别
    ↓
┌─────────────────────────────────────┐
│ 1. 识别格式类型                      │
│    - 匹配触发词                      │
│    - 确定格式要求                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 生成内容                          │
│    - 按格式生成内容                  │
│    - 确保结构完整                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 格式校验                          │
│    - 检查格式正确性                  │
│    - 检查结构完整性                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 格式修正（如需要）                │
│    - 修正格式错误                    │
│    - 补充缺失结构                    │
└─────────────────────────────────────┘
    ↓
输出结果
```

## 格式与内容冲突处理

### 优先级规则
| 优先级 | 规则 | 说明 |
|--------|------|------|
| 1 | 内容正确性 | 内容必须正确 |
| 2 | 格式符合性 | 尽量符合格式 |
| 3 | 结构完整性 | 结构尽量完整 |

### 冲突处理
| 冲突情况 | 处理方式 |
|----------|----------|
| 内容过长影响格式 | 适当精简内容 |
| 格式限制内容表达 | 优先保证内容，格式次之 |
| 结构缺失 | 补充缺失结构 |

## 格式校验规则

### 表格校验
| 检查项 | 说明 |
|--------|------|
| 表头存在 | 必须有表头 |
| 列对齐 | 列必须对齐 |
| 分隔线 | 必须有分隔线 |

### JSON校验
| 检查项 | 说明 |
|--------|------|
| 语法正确 | JSON语法正确 |
| 键值完整 | 键值对完整 |
| 类型正确 | 数据类型正确 |

### 步骤校验
| 检查项 | 说明 |
|--------|------|
| 编号连续 | 步骤编号连续 |
| 描述完整 | 每步有描述 |
| 顺序合理 | 步骤顺序合理 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 格式无法满足 | 说明原因，提供替代格式 |
| 格式校验失败 | 尝试修正，修正失败则说明 |
| 格式与内容严重冲突 | 优先内容，简化格式 |
