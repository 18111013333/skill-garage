# ALERT_ESCALATION_RULES.md - 告警升级规则

## 目的
定义告警升级的触发条件和升级路径。

## 适用范围
所有告警的升级处理、跨团队升级、管理层升级。

## 升级触发条件

### 响应超时升级
| 告警级别 | 响应时限 | 超时后升级 |
|----------|----------|------------|
| P0 | 5 分钟 | 3 分钟未响应升级 |
| P1 | 15 分钟 | 10 分钟未响应升级 |
| P2 | 1 小时 | 30 分钟未响应升级 |
| P3 | 4 小时 | 2 小时未响应升级 |

### 处理超时升级
| 告警级别 | 处理时限 | 超时后升级 |
|----------|----------|------------|
| P0 | 30 分钟 | 20 分钟未恢复升级 |
| P1 | 2 小时 | 1 小时未恢复升级 |
| P2 | 8 小时 | 4 小时未恢复升级 |
| P3 | 24 小时 | 12 小时未恢复升级 |

### 影响扩大升级
| 条件 | 说明 | 升级动作 |
|------|------|----------|
| 影响用户数增加 | > 2 倍 | 自动升级 |
| 影响服务增加 | 新增服务受影响 | 自动升级 |
| 影响租户增加 | 新增租户受影响 | 自动升级 |
| 错误率上升 | > 2 倍 | 自动升级 |

## 升级路径

### 技术升级路径
```
一级值班 (主值守)
    ↓ (超时/困难)
二级值班 (备值守)
    ↓ (超时/困难)
团队技术负责人
    ↓ (超时/困难)
部门技术负责人
    ↓ (超时/困难)
CTO
```

### 管理升级路径
```
值班工程师
    ↓ (影响扩大)
团队负责人
    ↓ (影响扩大)
部门负责人
    ↓ (影响扩大)
VP 级别
    ↓ (影响扩大)
CTO/CEO
```

### 跨团队升级路径
```
发现团队
    ↓ (需要其他团队)
相关团队值班
    ↓ (需要协调)
双方团队负责人
    ↓ (需要决策)
部门负责人
```

## 升级配置

### 按告警类型升级
| 告警类型 | 初级处理 | 升级目标 |
|----------|----------|----------|
| 系统告警 | 运维值班 | 运维负责人 |
| 业务告警 | 业务值班 | 业务负责人 |
| 安全告警 | 安全值班 | 安全负责人 |
| 数据告警 | 数据值班 | 数据负责人 |

### 按服务类型升级
| 服务类型 | 初级处理 | 升级目标 |
|----------|----------|----------|
| 推理服务 | 平台值班 | 平台负责人 |
| 工具服务 | 工具值班 | 工具负责人 |
| 记忆服务 | 平台值班 | 平台负责人 |
| 外部集成 | 集成值班 | 集成负责人 |

## 升级通知

### 通知方式
| 升级级别 | 通知方式 | 说明 |
|----------|----------|------|
| 一级升级 | IM + 邮件 | 团队内升级 |
| 二级升级 | 短信 + IM | 跨团队升级 |
| 三级升级 | 电话 + 短信 | 管理层升级 |
| 四级升级 | 电话 + 短信 + 邮件 | 最高级别 |

### 通知内容
| 内容 | 说明 |
|------|------|
| 告警摘要 | 告警简要描述 |
| 当前状态 | 当前处理状态 |
| 升级原因 | 升级触发原因 |
| 需要支持 | 需要什么支持 |
| 联系方式 | 当前处理人联系方式 |

| 当前状态 | 当前处理状态 |
| 下一步计划 | 计划执行的动作 |
| 需要支持 | 需要的支持 |

| 要求 | 说明 |
|------|------|
