# RBAC_POLICY.md - 角色访问控制总规则

**版本: V27.0**

## 目的
定义基于角色的访问控制（RBAC）总规则，确保平台权限控制进入正式制度层，实现"谁能对什么对象做什么"的精确控制。

## 适用范围
所有用户访问、操作执行、资源访问、权限分配场景。

---

## 一、核心原则

| 原则 | 说明 | 优先级 |
|------|------|--------|
| 默认拒绝 | 无明确授权则拒绝访问 | P0 |
| 最小权限 | 只授予完成任务所需的最小权限 | P0 |
| 按作用域授权 | 权限限定在特定作用域内 | P1 |
| 按对象授权 | 权限限定在特定对象上 | P1 |
| 权限冲突从严 | 权限冲突时选择更严格的 | P0 |
| 显式授权 | 所有授权必须显式声明 | P1 |

---

## 二、权限模型

### 2.1 权限组成

```
权限 = 主体(Who) + 动作(What) + 对象(On What) + 条件(When/Where)
```

### 2.2 动作层级

| 层级 | 动作 | 说明 | 风险等级 |
|------|------|------|----------|
| L1 | read | 读取/查看 | 低 |
| L2 | comment | 评论/反馈 | 低 |
| L3 | propose | 提议/建议 | 中 |
| L4 | execute | 执行/操作 | 中 |
| L5 | export | 导出/下载 | 中 |
| L6 | approve | 审批/批准 | 高 |
| L7 | configure | 配置/设置 | 高 |
| L8 | delete | 删除/销毁 | 极高 |
| L9 | admin | 完全管理 | 极高 |

### 2.3 对象分类

| 分类 | 对象 | 敏感度 |
|------|------|--------|
| 数据类 | memory, knowledge | 高 |
| 项目类 | project, portfolio | 中 |
| 配置类 | config, integrations | 高 |
| 运维类 | logs, reports, events | 中 |
| 管理类 | users, roles, tenants | 极高 |
| 能力类 | skills | 中 |

---

## 三、作用域层级

### 3.1 作用域定义

| 作用域 | 说明 | 权限范围 |
|--------|------|----------|
| system | 系统级 | 全平台所有租户 |
| tenant | 租户级 | 单个租户内所有资源 |
| project | 项目级 | 单个项目内所有资源 |
| task | 任务级 | 单个任务相关资源 |
| resource | 资源级 | 单个特定资源 |

### 3.2 作用域继承

```
system > tenant > project > task > resource
```

- 上级权限可被下级继承
- 下级权限不可超越上级
- 同级权限取并集

## 四、授权规则

### 4.1 授权流程

```
授权请求
    ↓
┌─────────────────────────────────────┐
│ 1. 权限检查                          │
│    - 授权者是否有授权权限             │
│    - 被授权者是否可接受该角色         │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 冲突检测                          │
│    - 是否与现有权限冲突               │
│    - 是否违反隔离规则                 │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 审批流程（如需要）                 │
│    - 按角色配置触发审批               │
│    - 记录审批链                       │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 授权生效                          │
│    - 写入权限表                       │
│    - 通知相关方                       │
│    - 审计记录                         │
└─────────────────────────────────────┘
```

### 4.2 授权条件

| 条件类型 | 说明 | 示例 |
|----------|------|------|
| own_only | 仅限自己创建的 | 只能删除自己的记忆 |
| time_restricted | 时间限制 | 仅工作时间可执行 |
| ip_restricted | IP限制 | 仅内网可访问 |
| approval_required | 需要审批 | 执行前需审批 |
| mfa_required | 需要MFA | 敏感操作需二次认证 |

---

## 五、权限冲突处理

### 5.1 冲突类型

| 冲突类型 | 说明 | 处理方式 |
|----------|------|----------|
| 动作冲突 | 同一对象不同动作 | 取并集 |
| 对象冲突 | 不同对象同一动作 | 分别生效 |
| 作用域冲突 | 不同作用域权限 | 从严处理 |
| 条件冲突 | 不同条件限制 | 取交集 |

### 5.2 冲突解决规则

```
规则1: 拒绝优先于允许
规则2: 高风险优先于低风险
规则3: 窄作用域优先于宽作用域
规则4: 显式授权优先于继承
```

### 5.3 冲突示例

| 场景 | 权限A | 权限B | 结果 |
|------|-------|-------|------|
| 同对象不同动作 | read memory | delete memory | read + delete |
| 同动作不同对象 | read memory | read config | 分别生效 |
| 作用域冲突 | tenant read all | project deny memory | project 内 deny |
| 条件冲突 | own_only | all | own_only |

## 六、临时授权

### 6.1 临时授权类型

| 类型 | 最长有效期 | 审批要求 |
|------|------------|----------|
| 紧急授权 | 24 小时 | 事后补审批 |
| 项目授权 | 项目周期 | 项目管理员 |
| 任务授权 | 任务周期 | 任务负责人 |
| 代理授权 | 30 天 | 双方确认 |

### 6.2 临时授权流程

```
申请 → 审批 → 生效 → 监控 → 自动回收 → 审计
```

### 6.3 临时授权监控

| 监控项 | 说明 |
|--------|------|
| 使用频率 | 异常高频告警 |
| 使用范围 | 越界使用告警 |
| 有效期 | 即将过期提醒 |
| 回收状态 | 回收失败告警 |

---

## 七、权限回收

### 7.1 回收触发条件

| 触发条件 | 说明 |
|----------|------|
| 角色变更 | 用户角色变化时回收旧权限 |
| 项目结束 | 项目关闭时回收项目权限 |
| 租户退出 | 租户终止时回收所有权限 |
| 安全事件 | 安全事件时紧急回收 |
| 定期审计 | 审计发现异常时回收 |

### 7.2 回收流程

```
触发回收
    ↓
┌─────────────────────────────────────┐
│ 1. 确定回收范围                      │
│    - 直接权限                        │
│    - 继承权限                        │
│    - 临时权限                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 检查依赖                          │
│    - 是否有进行中任务依赖该权限       │
│    - 是否有审批链依赖该权限           │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行回收                          │
│    - 移除权限记录                    │
│    - 终止相关会话                    │
│    - 通知用户                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 审计记录                          │
│    - 回收原因                        │
│    - 回收时间                        │
│    - 影响范围                        │
└─────────────────────────────────────┘
```

## 八、权限审计

### 8.1 审计内容

| 审计项 | 说明 | 保留期限 |
|--------|------|----------|
| 授权记录 | 谁授权给谁什么权限 | 365 天 |
| 访问记录 | 谁在何时访问了什么 | 90 天 |
| 拒绝记录 | 谁被拒绝访问什么 | 90 天 |
| 回收记录 | 权限何时被回收 | 365 天 |

### 8.2 审计报告

| 报告类型 | 频率 | 内容 |
|----------|------|------|
| 权限清单 | 每周 | 所有用户权限列表 |
| 异常访问 | 每日 | 异常访问模式分析 |
| 权限变更 | 每周 | 权限变更历史 |
| 过期权限 | 每日 | 即将过期的权限 |

---

## 九、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 权限不足 | 拒绝 + 记录 + 提示申请 |
| 权限冲突 | 从严处理 + 告警 |
| 授权失败 | 回滚 + 通知 + 审计 |
| 回收失败 | 重试 + 告警 + 人工介入 |
| 审计异常 | 告警 + 安全团队通知 |

---

## 十、完成标准

| 标准 | 验证方式 |
|------|----------|
| 默认拒绝 | 无权限用户无法访问任何资源 |
| 最小权限 | 用户权限不超过其角色需求 |
| 作用域隔离 | 权限不越界 |
| 冲突处理 | 冲突权限正确解决 |
| 临时授权 | 临时权限自动回收 |
| 审计覆盖 | 所有权限操作有记录 |
