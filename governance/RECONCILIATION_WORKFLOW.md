# RECONCILIATION_WORKFLOW.md - 终极鸽子王生态对账流程

## 目的
定义生态伙伴的对账流程，确保账务准确。

## 适用范围
所有渠道伙伴、转售商、市场开发者的对账管理。

---

## 一、对账周期

### 1.1 周期设置
| 对账类型 | 周期 | 说明 |
|----------|------|------|
| 月度对账 | 每月 | 标准对账周期 |
| 季度对账 | 每季度 | 大额伙伴对账 |
| 年度对账 | 每年 | 年度总结对账 |

### 1.2 对账时间表
| 阶段 | 时间 | 说明 |
|------|------|------|
| 数据导出 | 周期结束后1日 | 导出账务数据 |
| 对账执行 | 周期结束后1-5日 | 执行对账 |
| 差异处理 | 周期结束后5-10日 | 处理差异 |
| 确认签字 | 周期结束后10-15日 | 签字确认 |

---

## 二、对账维度

### 2.1 对账项目
| 维度 | 说明 |
|------|------|
| 订单对账 | 订单数量和金额核对 |
| 退款对账 | 退款金额核对 |
| 分成对账 | 分成金额核对 |
| 费用对账 | 各类费用核对 |

### 2.2 对账数据
| 数据项 | 来源 | 说明 |
|--------|------|------|
| 销售数据 | 平台系统 | 订单记录 |
| 退款数据 | 平台系统 | 退款记录 |
| 分成数据 | 分成系统 | 分成计算 |
| 伙伴数据 | 伙伴系统 | 伙伴记录 |

---

## 三、异常订单处理

### 3.1 异常类型
| 异常类型 | 说明 | 处理方式 |
|----------|------|----------|
| 订单缺失 | 平台有伙伴无 | 核实后补充 |
| 订单多余 | 伙伴有平台无 | 核实后调整 |
| 金额差异 | 金额不一致 | 核实后调整 |
| 状态差异 | 状态不一致 | 核实后同步 |

### 3.2 处理流程
```
异常订单处理
    ↓
┌─────────────────────────────────────┐
│ 1. 异常识别                          │
│    - 自动对比识别                    │
│    - 标记异常订单                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 异常核实                          │
│    - 调查异常原因                    │
│    - 收集相关证据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 异常处理                          │
│    - 确定处理方案                    │
│    - 执行调整                        │
└─────────────────────────────────────┘
```

---

## 四、重复计费处理

### 4.1 检测规则
| 规则 | 说明 |
|------|------|
| 同订单重复 | 同一订单多次计费 |
| 同周期重复 | 同一周期重复结算 |
| 跨周期重复 | 跨周期重复计费 |

### 4.2 处理方式
| 处理方式 | 说明 |
|----------|------|
| 自动冲销 | 系统自动冲销重复项 |
| 手动调整 | 人工核实后调整 |
| 退款处理 | 需退款时执行退款 |

---

## 五、漏计费处理

### 5.1 检测规则
| 规则 | 说明 |
|------|------|
| 订单漏计 | 有订单无计费记录 |
| 分成漏计 | 有销售无分成记录 |
| 费用漏计 | 有费用无扣费记录 |

### 5.2 处理方式
| 处理方式 | 说明 |
|----------|------|
| 补充计费 | 补充漏计费用 |
| 调整账单 | 调整相关账单 |
| 说明原因 | 记录漏计原因 |

---

## 六、分成冲突处理

### 6.1 冲突类型
| 冲突类型 | 说明 |
|----------|------|
| 归属冲突 | 多方主张分成 |
| 比例冲突 | 分成比例争议 |
| 时间冲突 | 分成时间争议 |

### 6.2 处理流程
```
分成冲突处理
    ↓
┌─────────────────────────────────────┐
│ 1. 冲突识别                          │
│    - 识别冲突类型                    │
│    - 收集相关证据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 冲突评估                          │
│    - 评估各方主张                    │
│    - 确定处理原则                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 冲突解决                          │
│    - 按规则或协商解决                │
│    - 记录解决方案                    │
└─────────────────────────────────────┘
```

---

## 七、人工复核

### 7.1 复核触发
| 触发条件 | 说明 |
|----------|------|
| 差异金额 > 1万 | 大额差异需复核 |
| 差异比例 > 5% | 高比例差异需复核 |
| 伙伴申请 | 伙伴申请人工复核 |
| 系统异常 | 系统异常需复核 |

### 7.2 复核流程
```
人工复核流程
    ↓
┌─────────────────────────────────────┐
│ 1. 复核申请                          │
│    - 提交复核申请                    │
│    - 说明复核原因                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 复核执行                          │
│    - 财务团队复核                    │
│    - 必要时技术支持                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 复核结论                          │
│    - 出具复核报告                    │
│    - 执行复核结论                    │
└─────────────────────────────────────┘
```
