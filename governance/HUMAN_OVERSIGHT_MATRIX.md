# HUMAN_OVERSIGHT_MATRIX.md - 人类监督矩阵

## 目的
定义人类监督边界，明确自治与人工监管的分界。

## 适用范围
- 所有 AI 用例
- 所有 AI 决策
- 所有 AI 输出

## 监督模式

### 全自动执行
```yaml
fully_automatic:
  criteria:
    - 低风险决策
    - 可逆操作
    - 无敏感影响
    - 内部辅助用途
  
  oversight:
    - 系统监控
    - 异常告警
    - 事后审计
  
  examples:
    - 内部文档检索
    - 格式转换
    - 非敏感内容生成
```

### 人工确认后执行
```yaml
human_confirmation:
  criteria:
    - 中等风险决策
    - 部分可逆操作
    - 有限外部影响
  
  oversight:
    - AI 建议
    - 人工确认
    - 执行记录
  
  examples:
    - 客户服务回复建议
    - 内部报告草稿
    - 数据分析建议
```

### 人工复核后生效
```yaml
human_review:
  criteria:
    - 高风险决策
    - 不可逆操作
    - 影响客户权益
    - 涉及敏感数据
  
  oversight:
    - AI 初步判断
    - 人工详细复核
    - 复核意见记录
    - 最终人工决定
  
  examples:
    - 信用评估建议
    - 风险预警
    - 合规判断建议
```

### 人工主导 AI 辅助
```yaml
human_led:
  criteria:
    - 极高风险决策
    - 重大业务影响
    - 法律合规风险
    - 监管要求
  
  oversight:
    - 人工主导决策
    - AI 提供参考
    - 决策理由记录
    - 多级审批
  
  examples:
    - 大额交易审批
    - 合规判定
    - 重大客户处理
```

## 风险等级监督要求

| 风险等级 | 监督模式 | 复核比例 | 记录要求 |
|----------|----------|----------|----------|
| 低 | 全自动 | 事后抽样 1% | 简化记录 |
| 中 | 人工确认 | 100% 确认 | 标准记录 |
| 高 | 人工复核 | 100% 复核 | 详细记录 |
| 极高 | 人工主导 | 多级审批 | 完整记录 |

## 异常升级

```yaml
escalation:
  triggers:
    - AI 置信度低于阈值
    - 检测到异常模式
    - 触发敏感词/场景
    - 用户请求人工介入
  
  process:
    - 自动标记
    - 转人工处理
    - 记录升级原因
    - 反馈优化模型
```
