# INCIDENT_SEVERITY.md - 事故分级标准

## 目的
定义事故严重等级判定标准，确保所有异常有统一分级口径。

## 适用范围
所有生产事故、服务异常、安全事件、数据问题。

## 事故等级定义

### Sev1 - 严重事故
| 维度 | 标准 |
|------|------|
| 影响范围 | 全站或核心服务不可用 |
| 用户影响 | > 50% 用户受影响 |
| 业务影响 | 核心业务完全中断 |
| 数据影响 | 数据丢失或损坏 |
| 响应时限 | 5 分钟内响应 |
| 恢复目标 | 30 分钟内恢复 |

**典型案例**:
- 全站服务不可用
- 数据库主库故障
- 安全入侵导致数据泄露
- 支付系统完全不可用

### Sev2 - 重大事故
| 维度 | 标准 |
|------|------|
| 影响范围 | 核心功能不可用 |
| 用户影响 | 20% - 50% 用户受影响 |
| 业务影响 | 核心业务部分中断 |
| 数据影响 | 部分数据不一致 |
| 响应时限 | 15 分钟内响应 |
| 恢复目标 | 2 小时内恢复 |

**典型案例**:
- 推理服务大面积超时
- 工具执行成功率大幅下降
- 记忆服务不可用
- 单个重要租户完全不可用

### Sev3 - 一般事故
| 维度 | 标准 |
|------|------|
| 影响范围 | 部分功能受影响 |
| 用户影响 | 5% - 20% 用户受影响 |
| 业务影响 | 非核心功能受影响 |
| 数据影响 | 无数据影响 |
| 响应时限 | 30 分钟内响应 |
| 恢复目标 | 4 小时内恢复 |

**典型案例**:
- 单个工具不可用
- 性能下降但服务可用
- 部分租户受影响
- 非核心服务故障

### Sev4 - 轻微事故
| 维度 | 标准 |
|------|------|
| 影响范围 | 小范围问题 |
| 用户影响 | < 5% 用户受影响 |
| 业务影响 | 业务基本不受影响 |
| 数据影响 | 无数据影响 |
| 响应时限 | 2 小时内响应 |
| 恢复目标 | 24 小时内恢复 |

**典型案例**:
- 单个用户报告问题
- 非关键功能异常
- UI 显示问题
- 文档错误

## 分级判定矩阵

### 按影响范围判定
```
┌─────────────────────────────────────────────────────────────┐
│                    影响范围判定矩阵                          │
└─────────────────────────────────────────────────────────────┘
                    │
                    │  用户影响比例
                    │
         < 5%       │    Sev4
         5% - 20%   │    Sev3
         20% - 50%  │    Sev2
         > 50%      │    Sev1
                    │
```

### 按服务重要性判定
| 服务等级 | 不可用判定 | 降级判定 |
|----------|------------|----------|
| 核心服务 | Sev1 | Sev2 |
| 重要服务 | Sev2 | Sev3 |
| 一般服务 | Sev3 | Sev4 |
| 辅助服务 | Sev4 | Sev4 |

### 按数据影响判定
| 数据影响 | 等级提升 |
|----------|----------|
| 数据丢失 | 提升 2 级 |
| 数据损坏 | 提升 2 级 |
| 数据泄露 | 提升 2 级 |
| 数据不一致 | 提升 1 级 |
| 无数据影响 | 不提升 |

## 分级流程

### 初步分级
```
发现异常 → 快速评估 → 初步定级 → 通知响应
```

### 分级调整
```
持续评估 → 影响变化 → 调整等级 → 更新通知
```

### 最终定级
```
事故恢复 → 全面评估 → 最终定级 → 记录归档
```

## 处理时限

### 响应时限
| 等级 | 首响时限 | 升级时限 |
|------|----------|----------|
| Sev1 | 5 分钟 | 15 分钟 |
| Sev2 | 15 分钟 | 30 分钟 |
| Sev3 | 30 分钟 | 2 小时 |
| Sev4 | 2 小时 | 8 小时 |

### 恢复时限
| 等级 | 恢复目标 | 最大时限 |
|------|----------|----------|
| Sev1 | 30 分钟 | 2 小时 |
| Sev2 | 2 小时 | 4 小时 |
| Sev3 | 4 小时 | 24 小时 |
| Sev4 | 24 小时 | 72 小时 |

### 沟通时限
| 等级 | 首次沟通 | 更新频率 |
|------|----------|----------|
| Sev1 | 10 分钟 | 每 15 分钟 |
| Sev2 | 20 分钟 | 每 30 分钟 |
| Sev3 | 1 小时 | 每 2 小时 |
| Sev4 | 4 小时 | 每 8 小时 |

## 升级规则

### 自动升级
| 条件 | 升级动作 |
|------|----------|
| 响应超时 | 自动升级到下一级 |
| 恢复超时 | 自动升级到下一级 |
| 影响扩大 | 自动升级等级 |
| 新增数据影响 | 自动升级等级 |

### 人工升级
| 角色 | 升级权限 |
|------|----------|
| 值班工程师 | 可升级 1 级 |
| 团队负责人 | 可升级 2 级 |
| 部门负责人 | 可升级 3 级 |
| VP 级别 | 可升级到 Sev1 |

## 特殊场景分级

### 安全事件
| 事件类型 | 基础等级 |
|----------|----------|
| 数据泄露 | Sev1 |
| 系统入侵 | Sev1 |
| 权限绕过 | Sev2 |
| 漏洞利用 | Sev2 |
| 异常访问 | Sev3 |

### 数据事件
| 事件类型 | 基础等级 |
|----------|----------|
| 数据丢失 | Sev1 |
| 数据损坏 | Sev1 |
| 数据不一致 | Sev2 |
| 数据延迟 | Sev3 |

### 第三方事件
| 事件类型 | 基础等级 |
|----------|----------|
| 核心依赖不可用 | Sev2 |
| 重要依赖不可用 | Sev3 |
| 一般依赖不可用 | Sev4 |
