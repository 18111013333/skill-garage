# COMPLIANCE_REVIEW_POLICY.md - 合规审查总规则

## 目的
定义合规审查总规则，确保高风险输出和动作经过合规审批。

## 适用范围
所有输出内容、执行动作、数据操作、外部交互。

## 合规审查原则

### 核心原则
1. **风险导向**: 高风险必须审查，低风险可豁免
2. **分级审批**: 按风险等级匹配审批级别
3. **全程留痕**: 所有审查决策必须记录
4. **持续监控**: 审查后仍需监控执行

### 合规责任
| 角色 | 责任 |
|------|------|
| 系统自动审查 | 初筛、阻断、记录 |
| 合规专员 | 复核、审批、指导 |
| 业务负责人 | 业务风险评估 |
| 安全团队 | 安全风险评估 |

## 审查流程

### 标准流程
```
触发审查 → 自动初筛 → 人工复核 → 审批决策 → 执行/阻断 → 记录归档
```

### 快速通道
| 条件 | 通道 |
|------|------|
| 低风险 + 已有类似审批 | 自动通过 |
| 紧急 + 有预授权 | 快速审批 |
| 标准场景 + 模板匹配 | 模板审批 |

## 审查决策

### 决策类型
| 决策 | 说明 | 后续动作 |
|------|------|----------|
| 通过 | 符合合规要求 | 允许执行 |
| 有条件通过 | 需附加条件 | 条件满足后执行 |
| 驳回 | 不符合合规要求 | 阻断执行 |
| 需补充 | 信息不足 | 补充信息后重新审查 |

## 联动机制

### 与 Audit 联动
- 所有审查决策记录审计
- 审计可追溯审查历史
- 异常审查触发审计

### 与 Governance Center 联动
- 合规政策由治理中心发布
- 合规违规上报治理中心
- 政策变更通知合规审查

### 与 Alert 联动
- 合规违规触发告警
- 审查超时触发告警
- 批量阻断触发告警
