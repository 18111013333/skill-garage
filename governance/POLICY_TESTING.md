# POLICY_TESTING.md - 策略测试规则

**版本: V27.0**

## 目的
定义策略测试规则，确保策略不是写完就上线，而是先可验证，实现"策略可测试、变更可验证、上线有保障"。

## 适用范围
所有策略代码测试场景，包括：
- 单元测试
- 场景测试
- 回归测试
- 冲突测试
- 区域/租户差异测试

---

## 一、测试原则

| 原则 | 说明 |
|------|------|
| 测试先行 | 策略变更必须先测试 |
| 覆盖完整 | 测试覆盖所有规则分支 |
| 自动化 | 测试自动化执行 |
| 可重复 | 测试结果可重复 |
| 阻断上线 | 测试失败阻断上线 |

---

## 二、单元测试

### 2.1 测试范围

| 测试项 | 说明 |
|--------|------|
| 规则条件 | 测试每个规则条件逻辑 |
| 规则动作 | 测试每个规则动作执行 |
| 默认决策 | 测试默认决策逻辑 |
| 输入验证 | 测试输入参数验证 |
| 输出格式 | 测试输出格式正确性 |

### 2.2 测试用例结构

```json
{
  "case_id": "ut_001",
  "case_name": "规则条件测试 - 正常情况",
  "policy_id": "pac_access_001",
  "input": {
    "user_id": "user_001",
    "resource": "memory",
    "action": "read"
  },
  "expected_output": {
    "decision": "allow",
    "matched_rule": "rule_001"
  },
  "setup": {},
  "teardown": {}
}
```

### 2.3 测试覆盖率要求

| 策略类型 | 最低覆盖率 | 说明 |
|----------|------------|------|
| 访问控制 | 100% | 所有规则分支覆盖 |
| 安全策略 | 100% | 所有规则分支覆盖 |
| 合规策略 | 100% | 所有规则分支覆盖 |
| 限流策略 | 90% | 主要分支覆盖 |
| 成本控制 | 90% | 主要分支覆盖 |

## 三、场景测试

### 3.1 测试场景

| 场景 | 说明 | 测试重点 |
|------|------|----------|
| 正常场景 | 正常使用场景 | 策略正常执行 |
| 边界场景 | 边界条件场景 | 边界条件处理 |
| 异常场景 | 异常输入场景 | 异常处理 |
| 极端场景 | 极端条件场景 | 极端条件处理 |
| 例外场景 | 例外申请场景 | 例外处理 |

### 3.2 场景测试用例

```
场景测试用例结构:
{
  "scenario_id": "scenario_001",
  "scenario_name": "用户访问控制场景",
  "description": "测试用户访问不同资源的策略执行",
  "test_cases": [
    {
      "case_id": "scenario_001_01",
      "case_name": "管理员访问敏感资源",
      "input": {...},
      "expected_output": {...}
    },
    {
      "case_id": "scenario_001_02",
      "case_name": "普通用户访问敏感资源",
      "input": {...},
      "expected_output": {...}
    }
  ]
}
```

### 3.3 场景覆盖矩阵

| 场景类型 | 必须覆盖 | 说明 |
|----------|----------|------|
| 用户角色 | 所有角色 | 管理员/普通用户/观察者等 |
| 资源类型 | 所有资源 | memory/knowledge/config等 |
| 操作类型 | 所有操作 | read/write/delete等 |
| 时间条件 | 工作时间/非工作时间 | 时间相关策略 |
| 区域条件 | 所有区域 | 区域相关策略 |

## 四、回归测试

### 4.1 回归触发条件

| 触发条件 | 说明 |
|----------|------|
| 策略变更 | 策略代码变更 |
| 依赖变更 | 依赖策略变更 |
| 环境变更 | 部署环境变更 |
| 定期回归 | 定期回归测试 |

### 4.2 回归测试范围

| 范围 | 说明 | 执行频率 |
|------|------|----------|
| 全量回归 | 所有策略测试 | 重大变更 |
| 增量回归 | 变更相关测试 | 每次变更 |
| 冒烟回归 | 核心功能测试 | 每次部署 |
| 定期回归 | 定期全量测试 | 每周 |

### 4.3 回归测试流程

```
回归触发
    ↓
┌─────────────────────────────────────┐
│ 1. 影响分析                          │
│    - 分析变更影响范围                │
│    - 确定回归范围                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 测试选择                          │
│    - 选择相关测试用例                │
│    - 组装测试套件                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 测试执行                          │
│    - 执行测试套件                    │
│    - 收集测试结果                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 结果分析                          │
│    - 分析测试结果                    │
│    - 识别失败原因                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 决策                              │
│    - 通过：继续部署                  │
│    - 失败：阻断部署                  │
└─────────────────────────────────────┘
```

## 五、冲突测试

### 5.1 冲突类型

| 冲突类型 | 说明 | 测试重点 |
|----------|------|----------|
| 规则冲突 | 同一策略内规则冲突 | 规则优先级 |
| 策略冲突 | 不同策略间冲突 | 策略优先级 |
| 作用域冲突 | 不同作用域策略冲突 | 作用域优先级 |
| 例外冲突 | 例外与策略冲突 | 例外处理 |

### 5.2 冲突测试用例

```json
{
  "conflict_test_id": "conflict_001",
  "conflict_type": "policy_conflict",
  "description": "测试访问控制策略与数据治理策略冲突",
  "policies": ["pac_access_001", "pac_data_001"],
  "input": {
    "user_id": "user_001",
    "resource": "sensitive_data",
    "action": "export"
  },
  "expected_resolution": {
    "winning_policy": "pac_data_001",
    "decision": "deny",
    "reason": "数据治理策略优先级更高"
  }
}
```

### 5.3 冲突解决验证

| 验证项 | 说明 |
|--------|------|
| 优先级正确 | 冲突解决优先级正确 |
| 结果一致 | 相同输入结果一致 |
| 无遗漏 | 所有冲突场景覆盖 |
| 可追溯 | 冲突解决过程可追溯 |

## 九、失败阻断规则

### 9.1 阻断条件

| 条件 | 说明 | 阻断级别 |
|------|------|----------|
| 单元测试失败 | 任一单元测试失败 | 阻断 |
| 覆盖率不足 | 覆盖率低于要求 | 阻断 |
| 场景测试失败 | 关键场景测试失败 | 阻断 |
| 回归测试失败 | 回归测试失败 | 阻断 |
| 冲突测试失败 | 冲突测试失败 | 阻断 |

### 9.2 阻断处理

```
测试失败
    ↓
┌─────────────────────────────────────┐
│ 1. 失败记录                          │
│    - 记录失败详情                    │
│    - 通知策略负责人                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 问题分析                          │
│    - 分析失败原因                    │
│    - 定位问题位置                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 问题修复                          │
│    - 修复策略或测试                  │
│    - 重新执行测试                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 验证通过                          │
│    - 测试通过后解除阻断              │
│    - 继续后续流程                    │
└─────────────────────────────────────┘
```

## 十一、异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 环境异常 | 重试 + 告警 |
| 数据异常 | 重置数据 + 重试 |
| 超时异常 | 调整超时 + 重试 |
| 结果不一致 | 分析原因 + 修复 |

| 标准 | 验证方式 |
|------|----------|

| 报告完整 | 报告内容完整 |
| 阻断生效 | 失败正确阻断 |
