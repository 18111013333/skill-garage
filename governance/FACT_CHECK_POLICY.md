# FACT_CHECK_POLICY.md - 事实校验策略

## 目的
定义哪些回答需要事实校验，确保系统不会在不确定时装确定。

## 适用范围
所有涉及事实陈述的回答。

## 事实校验分级

### 直接记忆可答
| 条件 | 说明 | 校验要求 |
|------|------|----------|
| 用户画像信息 | 来自用户明确陈述 | 无需校验 |
| 系统配置信息 | 来自系统配置 | 无需校验 |
| 会话上下文 | 当前会话已确认信息 | 无需校验 |
| 常识知识 | 广为人知的常识 | 无需校验 |

### 需内部校验
| 条件 | 说明 | 校验要求 |
|------|------|----------|
| 记忆检索结果 | 来自长期记忆 | 检查记忆可信度 |
| 推断结论 | 基于已知推断 | 检查推断逻辑 |
| 历史决策 | 基于历史决策 | 检查决策有效性 |

### 需外部校验
| 条件 | 说明 | 校验要求 |
|------|------|----------|
| 时效敏感信息 | 新闻、价格等 | 必须外部检索验证 |
| 事实密集内容 | 数据、统计等 | 多源交叉验证 |
| 高风险领域 | 医疗、法律等 | 权威来源验证 |
| 争议性内容 | 存在争议的内容 | 多方观点验证 |

### 无法确认必须说不确定
| 条件 | 说明 | 处理方式 |
|------|------|----------|
| 信息不足 | 缺乏足够信息 | 明确说"信息不足" |
| 来源冲突 | 来源信息矛盾 | 标注冲突，说"存在不同说法" |
| 超出能力 | 超出系统能力范围 | 明确说"无法确定" |
| 时效过期 | 信息可能过期 | 标注"可能已过期" |

## 来源冲突处理

### 冲突检测
| 情况 | 检测方式 |
|------|----------|
| 数值冲突 | 不同来源数值不同 |
| 事实冲突 | 不同来源事实矛盾 |
| 观点冲突 | 不同来源观点相反 |
| 时间冲突 | 不同来源时间不一致 |

### 冲突处理顺序
| 优先级 | 处理方式 |
|--------|----------|
| 1 | 用户明确输入优先 |
| 2 | 高可信来源优先 |
| 3 | 最新信息优先 |
| 4 | 多数来源一致优先 |
| 5 | 无法判断时标注冲突 |

### 冲突标注格式
```
关于{主题}，存在不同信息：
- 来源A: {信息A} [可信度: 高]
- 来源B: {信息B} [可信度: 中]
当前采用: {采用的信息} (原因: {原因})
```

## 校验流程

```
事实陈述
    ↓
┌─────────────────────────────────────┐
│ 1. 判断校验级别                      │
│    - 直接记忆可答？                  │
│    - 需内部校验？                    │
│    - 需外部校验？                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行校验                          │
│    - 内部校验：检查记忆可信度        │
│    - 外部校验：检索验证              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 处理校验结果                      │
│    - 通过：正常输出                  │
│    - 冲突：标注冲突                  │
│    - 失败：标注不确定                │
└─────────────────────────────────────┘
    ↓
输出结果
```

## 校验结果处理

### 校验通过
```
{事实陈述} [已验证]
```

### 校验冲突
```
{事实陈述} [存在不同说法，详见...]
```

### 校验失败
```
关于{主题}，目前信息不足以确认，建议{建议操作}。
```

## 高风险领域特殊处理

### 医疗领域
| 要求 | 说明 |
|------|------|
| 必须外部校验 | 医疗信息必须权威来源 |
| 免责声明 | 必须添加免责声明 |
| 建议就医 | 建议咨询专业医生 |

### 法律领域
| 要求 | 说明 |
|------|------|
| 必须外部校验 | 法律信息必须权威来源 |
| 免责声明 | 必须添加免责声明 |
| 建议咨询 | 建议咨询专业律师 |

### 财务领域
| 要求 | 说明 |
|------|------|
| 必须外部校验 | 财务数据必须权威来源 |
| 时效标注 | 必须标注数据时效 |
| 风险提示 | 必须添加风险提示 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 校验超时 | 标注"未能完全验证" |
| 无可用来源 | 标注"信息不足" |
| 校验失败 | 标注不确定性 |

## 维护方式
- 新增校验级别: 添加到分级表
- 新增冲突处理: 更新冲突处理顺序
- 新增高风险领域: 添加特殊处理规则
