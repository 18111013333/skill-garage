# METRICS.md - 系统核心指标

## 目的
定义系统核心指标，为后续优化提供明确依据。

## 适用范围
所有系统运行状态的监控和统计。

## 核心指标

### 响应质量指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 首答成功率 | 首次回答满意数 / 总请求数 | ≥ 90% | < 85% |
| 响应准确率 | 正确响应数 / 总响应数 | ≥ 95% | < 90% |
| 响应完整率 | 完整响应数 / 总响应数 | ≥ 90% | < 85% |
| 用户满意度 | 满意评价数 / 总评价数 | ≥ 90% | < 80% |

### 执行效率指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 平均响应时间 | 总响应时间 / 请求数 | ≤ 3s | > 5s |
| P95响应时间 | 95分位响应时间 | ≤ 5s | > 10s |
| 平均步骤数 | 总步骤数 / 任务数 | ≤ 5 | > 10 |
| 平均时延 | 总时延 / 操作数 | ≤ 1s | > 3s |

### 工具与技能指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 工具成功率 | 成功调用数 / 总调用数 | ≥ 98% | < 95% |
| 技能成功率 | 成功执行数 / 总执行数 | ≥ 95% | < 90% |
| 工具错误率 | 失败调用数 / 总调用数 | ≤ 2% | > 5% |
| 技能错误率 | 失败执行数 / 总执行数 | ≤ 5% | > 10% |

### Token与成本指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 平均Token消耗 | 总Token / 请求数 | ≤ 5000 | > 10000 |
| Token预算使用率 | 已用Token / 预算Token | ≤ 80% | > 90% |
| 单次最大Token | 单次请求最大Token | ≤ 50000 | > 80000 |

### 记忆系统指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 记忆命中率 | 命中数 / 查询数 | ≥ 80% | < 60% |
| 记忆冲突率 | 冲突数 / 写入数 | ≤ 5% | > 10% |
| 记忆写入成功率 | 成功写入数 / 写入请求数 | ≥ 99% | < 95% |

### 质量与稳定性指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 追问修正率 | 需追问修正数 / 总请求数 | ≤ 10% | > 20% |
| 幻觉率 | 幻觉数 / 事实陈述数 | ≤ 5% | > 10% |
| 降级触发率 | 降级次数 / 总执行次数 | ≤ 0.1% | > 1% |
| 超时率 | 超时数 / 总请求数 | ≤ 0.5% | > 2% |

### 系统稳定性指标
| 指标名 | 计算方式 | 目标值 | 告警阈值 |
|--------|----------|--------|----------|
| 系统可用率 | 正常运行时间 / 总时间 | ≥ 99.9% | < 99% |
| 错误率 | 错误数 / 总请求数 | ≤ 1% | > 5% |
| 回滚频率 | 回滚次数 / 发布次数 | ≤ 5% | > 20% |

## 指标采集

### 采集频率
| 指标类型 | 采集频率 | 存储周期 |
|----------|----------|----------|
| 实时指标 | 每请求 | 7天 |
| 分钟指标 | 每分钟 | 30天 |
| 小时指标 | 每小时 | 90天 |
| 天指标 | 每天 | 365天 |

### 采集点
```
请求入口 → 记录开始时间
    ↓
任务执行 → 记录执行指标
    ↓
工具调用 → 记录调用指标
    ↓
输出生成 → 记录输出指标
    ↓
请求出口 → 记录结束时间
```

## 告警规则

### 告警级别
| 级别 | 条件 | 通知方式 |
|------|------|----------|
| INFO | 达到目标值 | 记录日志 |
| WARNING | 达到告警阈值 | 日志 + 提示 |
| ERROR | 超过告警阈值 150% | 日志 + 通知 |
| CRITICAL | 超过告警阈值 200% | 日志 + 通知 + 降级 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 指标采集失败 | 使用缓存值 + 标记 |
| 存储失败 | 缓存到内存 + 稍后重试 |
| 告警风暴 | 聚合告警 + 限流 |

## 维护方式
- 新增指标: 添加到指标定义表
- 调整阈值: 修改目标值和告警阈值
- 新增告警: 更新告警规则表
