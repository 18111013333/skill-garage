# MEMORY_CONFLICT_RULES.md - 记忆冲突处理规则

## 目的
定义新旧记忆冲突时的优先级和处理方式。

## 适用范围
所有记忆写入操作中的冲突检测和处理。

## 冲突优先级

### 基本优先级规则
| 优先级 | 规则 | 说明 |
|--------|------|------|
| 1 | 用户明确陈述 > 系统推断 | 用户直接说的优先 |
| 2 | 新信息 > 旧信息 | 时效性优先 |
| 3 | 高置信 > 低置信 | 可信度优先 |
| 4 | 用户自述 > 外部文档推断 | 来源优先 |

### 详细优先级矩阵
| 新信息来源 | 旧信息来源 | 优先级 |
|------------|------------|--------|
| user_explicit | user_inferred | 新信息 |
| user_explicit | system_inferred | 新信息 |
| user_explicit | external_doc | 新信息 |
| user_inferred | user_explicit | 旧信息 |
| user_inferred | system_inferred | 新信息 |
| system_inferred | user_explicit | 旧信息 |
| system_inferred | user_inferred | 旧信息 |
| external_doc | user_explicit | 旧信息 |
| external_doc | user_inferred | 需确认 |

## 冲突类型

### 事实冲突
| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 完全矛盾 | A vs 非A | 标记待确认 |
| 部分矛盾 | A vs A+B | 合并或确认 |
| 数值差异 | 25 vs 30 | 新值覆盖或确认 |
| 时间差异 | 旧时间 vs 新时间 | 新时间优先 |

### 偏好冲突
| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 硬约束冲突 | 必须A vs 必须B | 标记待确认 |
| 软偏好冲突 | 倾向A vs 倾向B | 新偏好覆盖 |
| 格式偏好冲突 | 格式A vs 格式B | 新偏好覆盖 |

### 状态冲突
| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 状态变更 | 进行中 vs 已完成 | 新状态覆盖 |
| 位置变更 | 北京 vs 上海 | 新位置覆盖 |
| 角色变更 | 开发者 vs 设计师 | 新角色覆盖 |

## 冲突处理策略

### 覆盖策略
- **适用条件**: 新信息优先级高于旧信息
- **动作**: 新信息覆盖旧信息，旧信息标记为 stale
- **日志**: 记录覆盖原因

### 并存策略
- **适用条件**: 信息可互补，不矛盾
- **动作**: 两条信息都保留，标记关联
- **日志**: 记录并存原因

### 标记stale策略
- **适用条件**: 旧信息已过期或不再有效
- **动作**: 旧信息标记为 stale，不删除
- **日志**: 记录过期原因

### 待确认策略
- **适用条件**: 无法自动判断优先级
- **动作**: 标记冲突，等待用户确认
- **日志**: 记录冲突详情

## 冲突检测流程

```
新信息写入
    ↓
┌─────────────────────────────────────┐
│ 1. 检索相关记忆                      │
│    - 向量搜索相似记忆                │
│    - 关键词匹配                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 比较内容                          │
│    - 事实比较                        │
│    - 时间比较                        │
│    - 来源比较                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 判断冲突                          │
│    - 无冲突 → 直接写入               │
│    - 有冲突 → 选择处理策略           │
└─────────────────────────────────────┘
    ↓
执行处理策略
```

## 冲突日志格式

```json
{
  "conflictId": "conflict_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "type": "fact_conflict",
  "existing": {
    "id": "mem_001",
    "fact": "用户年龄25岁",
    "source": "user_explicit",
    "confidence": 1.0,
    "createdAt": "2026-01-01T00:00:00+08:00"
  },
  "incoming": {
    "fact": "用户年龄30岁",
    "source": "user_explicit",
    "confidence": 1.0
  },
  "resolution": {
    "strategy": "overwrite",
    "reason": "新信息来源优先级相同，时间更新",
    "result": "新信息覆盖旧信息"
  }
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 无法判断优先级 | 标记待确认 |
| 循环冲突 | 终止处理，记录错误 |
| 批量冲突 | 逐个处理，记录统计 |
