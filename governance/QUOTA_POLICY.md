# QUOTA_POLICY.md - 配额规则

## 目的
定义各层级配额规则，实现成本控制的精细化治理。

## 适用范围
所有租户、团队、项目、工具、模型的资源使用。

## 配额层级

```
┌─────────────────────────────────────────────────────────────┐
│                      配额层级结构                            │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
```

## 租户配额

### 配额类型
| 配额项 | 单位 | 默认值 | 说明 |
|--------|------|--------|------|
| 月度 Token 总量 | Million | 100 | 所有模型调用 |
| 月度工具调用 | Thousand | 500 | 所有工具执行 |
| 并发会话数 | Count | 100 | 同时活跃会话 |
| 存储空间 | GB | 50 | 记忆与数据存储 |
| API 调用 | Thousand | 1000 | 外部 API 调用 |

### 租户配额模板
```json
{
  "tenant_quota": {
    "tenant_id": "tenant-001",
    "quotas": {
      "monthly_tokens": {
        "limit": 100000000,
        "unit": "tokens",
        "period": "monthly"
      },
      "monthly_tool_calls": {
        "limit": 500000,
        "unit": "calls",
        "period": "monthly"
      },
      "concurrent_sessions": {
        "limit": 100,
        "unit": "sessions",
        "period": "concurrent"
      },
      "storage": {
        "limit": 50,
        "unit": "GB",
        "period": "persistent"
      }
    }
  }
}
```

## 团队配额

### 配额类型
| 配额项 | 单位 | 默认值 | 说明 |
|--------|------|--------|------|
| 月度 Token 量 | Million | 20 | 团队模型调用 |
| 月度工具调用 | Thousand | 100 | 团队工具执行 |
| 并发会话数 | Count | 30 | 团队活跃会话 |
| 成员数 | Count | 20 | 团队成员上限 |

### 团队配额继承
```
团队配额上限 ≤ 租户配额 - 其他团队已分配配额
```

## 项目配额

### 配额类型
| 配额项 | 单位 | 默认值 | 说明 |
|--------|------|--------|------|
| 月度 Token 量 | Million | 5 | 项目模型调用 |
| 月度工具调用 | Thousand | 20 | 项目工具执行 |
| 任务数 | Count | 100 | 项目任务上限 |
| 存储空间 | GB | 5 | 项目数据存储 |

### 项目配额继承
```
项目配额上限 ≤ 团队配额 - 其他项目已分配配额
```

## 高风险工具配额

### 风险分级
| 风险等级 | 工具类型 | 配额限制 |
|----------|----------|----------|
| 极高 | 外部写入、数据删除 | 需审批，无默认配额 |
| 高 | 外部发送、批量操作 | 严格限制 |
| 中 | 文件操作、配置修改 | 中等限制 |
| 低 | 读取、查询 | 宽松限制 |

### 高风险工具配额示例
```json
{
  "high_risk_tool_quota": {
    "tool_category": "external_write",
    "quotas": {
      "daily_calls": {
        "limit": 100,
        "unit": "calls",
        "period": "daily"
      },
      "approval_required": true,
      "approval_level": "team_lead"
    }
  }
}
```

## 高成本模型配额

### 成本分级
| 成本等级 | 模型类型 | 配额限制 |
|----------|----------|----------|
| 极高 | 特殊大模型 | 需审批，严格限制 |
| 高 | 高端模型 | 严格限制 |
| 中 | 标准模型 | 中等限制 |
| 低 | 经济模型 | 宽松限制 |

### 高成本模型配额示例
```json
{
  "high_cost_model_quota": {
    "model_tier": "premium",
    "quotas": {
      "monthly_tokens": {
        "limit": 10000000,
        "unit": "tokens",
        "period": "monthly"
      },
      "approval_required": true,
      "approval_level": "department_head"
    }
  }
}
```

## 超额处理策略

### 处理方式
| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 阻断 | 拒绝请求，返回错误 | 硬限制 |
| 降级 | 自动降级到低成本方案 | 可降级场景 |
| 审批 | 需要审批才能继续 | 可审批场景 |
| 计费 | 超额部分按高价计费 | 商业场景 |

### 超额响应配置
```json
{
  "overage_response": {
    "tenant_level": {
      "tokens": "block",
      "tool_calls": "degrade",
      "storage": "block"
    },
    "team_level": {
      "tokens": "approval",
      "tool_calls": "block"
    },
    "project_level": {
      "tokens": "degrade",
      "tool_calls": "block"
    }
  }
}
```

## 配额监控

### 监控指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 使用率 | 已用/配额 | 80% 警告，95% 严重 |
| 预测使用 | 预计周期末使用 | 预计超限告警 |
| 超额次数 | 周期内超额次数 | > 3 次告警 |
| 审批等待 | 等待审批的请求数 | > 10 告警 |

### 配额仪表盘
- 各层级配额使用情况
- 配额趋势预测
- 超额事件记录
- 审批队列状态

## 配额调整

### 调整类型
| 调整类型 | 审批要求 | 生效时间 |
|----------|----------|----------|
| 临时增加 | 团队负责人 | 即时 |
| 永久增加 | 部门负责人 | 下周期 |
| 配额转移 | 双方负责人 | 即时 |
| 紧急扩容 | VP 级别 | 即时 |

### 调整流程
```
申请提交 → 审批 → 配额更新 → 通知相关方 → 记录审计
```
