# CROSS_BORDER_TRANSFER.md - 跨境数据流转规则

## 目的
定义跨境数据流转规则，确保跨境流转具备明确边界和留痕。

## 适用范围
平台所有跨境数据传输、处理、存储。

---

## 一、跨境流转原则

| 原则 | 说明 | 执行要求 |
|------|------|----------|
| 最小跨境 | 最小化跨境数据传输 | 必要时才跨境 |
| 合规优先 | 合规要求高于便利 | 严格遵守法规 |
| 审批留痕 | 跨境需审批并留痕 | 完整记录 |
| 数据保护 | 跨境数据需保护 | 加密 + 保护措施 |
| 用户知情 | 用户知情同意 | 透明告知 |
| 可追溯 | 跨境可追溯 | 完整审计 |

---

## 二、数据跨境分类

### 2.1 禁止跨境数据
| 数据类型 | 禁止原因 | 法规依据 |
|----------|----------|----------|
| 中国个人信息 | 法律禁止 | 个人信息保护法 |
| 中国重要数据 | 安全评估未通过 | 数据安全法 |
| 欧盟敏感个人数据 | 严格限制 | GDPR |
| 健康医疗数据 | 隐私保护 | HIPAA/相关法规 |
| 金融账户数据 | 金融监管 | 金融法规 |
| 国家安全相关 | 安全保护 | 国家安全法 |

### 2.2 条件跨境数据
| 数据类型 | 跨境条件 | 保护措施 | 审批要求 |
|----------|----------|----------|----------|
| 一般个人信息 | 用户同意 + 安全评估 | 加密 + 标准合同 | CISO 审批 |
| 业务数据 | 业务需要 + 脱敏 | 加密传输 | 技术负责人审批 |
| 日志数据 | 合规需要 + 脱敏 | 加密传输 | 安全负责人审批 |
| 配置数据 | 业务需要 | 加密传输 | 技术负责人审批 |

### 2.3 自由跨境数据
| 数据类型 | 说明 | 要求 |
|----------|------|------|
| 公开数据 | 已公开的数据 | 无限制 |
| 脱敏数据 | 完全脱敏的数据 | 脱敏验证 |
| 聚合数据 | 聚合统计数据 | 无个人标识 |
| 非个人数据 | 不涉及个人的数据 | 无限制 |

## 三、跨境场景

### 3.1 跨境传输场景
| 场景 | 数据类型 | 审批要求 | 保护措施 |
|------|----------|----------|----------|
| 全球服务 | 配置数据 | 技术负责人 | 加密传输 |
| 跨区备份 | 业务数据 | CISO | 加密 + 访问控制 |
| 跨区查询 | 脱敏数据 | 无需审批 | 脱敏验证 |
| 第三方集成 | 业务数据 | CISO + 法务 | 合同 + 加密 |
| 数据分析 | 聚合数据 | 无需审批 | 聚合验证 |

### 3.2 跨境处理场景
| 场景 | 数据类型 | 审批要求 | 保护措施 |
|------|----------|----------|----------|
| 远程计算 | 脱敏数据 | 技术负责人 | 脱敏 + 加密 |
| 模型推理 | 对话数据 | CISO | 加密 + 不存储 |
| 数据分析 | 聚合数据 | 无需审批 | 聚合验证 |
| 安全检测 | 日志数据 | 安全负责人 | 脱敏 + 加密 |

### 3.3 跨境存储场景
| 场景 | 数据类型 | 审批要求 | 保护措施 |
|------|----------|----------|----------|
| 异地备份 | 业务数据 | CISO + 法务 | 加密 + 合规评估 |
| 云存储 | 非敏感数据 | 技术负责人 | 加密存储 |
| 归档存储 | 审计数据 | CISO | 加密 + 访问控制 |

## 四、审批流程

### 4.1 审批级别
| 跨境类型 | 数据敏感度 | 审批人 | 审批时限 |
|----------|------------|--------|----------|
| 禁止跨境 | 任何 | 不审批 | - |
| 条件跨境 | 高敏感 | CISO + 法务 + DPO | 7 天 |
| 条件跨境 | 中敏感 | CISO + 法务 | 5 天 |
| 条件跨境 | 低敏感 | CISO | 3 天 |
| 自由跨境 | 任何 | 无需审批 | - |

### 4.2 审批流程
```
跨境申请
    ↓
┌─────────────────────────────────────┐
│ 1. 申请评估                          │
│    - 评估数据类型                    │
│    - 评估跨境必要性                  │
│    - 评估保护措施                    │
│    - 评估合规风险                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 合规审查                          │
│    - 审查法规合规                    │
│    - 审查合同条款                    │
│    - 审查用户同意                    │
│    - 审查保护措施                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 审批决策                          │
│    - 批准：记录审批意见              │
│    - 拒绝：说明拒绝原因              │
│    - 条件批准：附加条件              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 执行跨境                          │
│    - 按批准执行                      │
│    - 记录跨境日志                    │
│    - 监控跨境过程                    │
└─────────────────────────────────────┘
```

## 五、保护措施

### 5.1 技术保护措施
| 措施 | 说明 | 要求 |
|------|------|------|
| 传输加密 | TLS 1.3 加密传输 | 强制 |
| 存储加密 | AES-256 加密存储 | 强制 |
| 访问控制 | 基于角色的访问控制 | 强制 |
| 数据脱敏 | 敏感数据脱敏 | 按需 |
| 数据最小化 | 仅传输必要数据 | 强制 |
| 完整性校验 | 数据完整性验证 | 强制 |

### 5.2 合同保护措施
| 措施 | 说明 | 要求 |
|------|------|------|
| 数据处理协议 | DPA 协议 | 强制 |
| 标准合同条款 | SCC 条款 | 按法规要求 |
| 保密协议 | NDA 协议 | 强制 |
| 审计权利 | 审计条款 | 建议 |
| 删除义务 | 数据删除条款 | 强制 |

### 5.3 组织保护措施
| 措施 | 说明 | 要求 |
|------|------|------|
| 责任人指定 | 指定跨境责任人 | 强制 |
| 培训要求 | 跨境合规培训 | 强制 |
| 审计要求 | 定期审计跨境 | 强制 |
| 事件响应 | 跨境事件响应 | 强制 |

## 九、违规处理

### 9.1 违规类型
| 违规类型 | 说明 | 严重程度 |
|----------|------|----------|
| 未经审批跨境 | 未获审批即跨境 | CRITICAL |
| 超范围跨境 | 超出审批范围 | HIGH |
| 保护措施不足 | 保护措施不符合要求 | HIGH |
| 无用户同意 | 未获用户同意 | HIGH |
| 记录不完整 | 审计记录不完整 | MEDIUM |

### 9.2 违规处理
| 违规程度 | 处理措施 |
|----------|----------|
| CRITICAL | 立即停止 + 调查 + 处分 |
| HIGH | 立即停止 + 整改 + 警告 |
| MEDIUM | 整改 + 培训 |

### 9.3 违规记录
| 记录项 | 记录内容 |
|--------|----------|
| 违规时间 | 违规发生时间 |
| 违规类型 | 违规类型分类 |
| 违规详情 | 违规详细描述 |
| 影响评估 | 违规影响评估 |
| 处理措施 | 采取的处理措施 |
| 整改情况 | 整改完成情况 |
