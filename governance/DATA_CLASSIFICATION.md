# DATA_CLASSIFICATION.md - 数据分级制度

## 目的
定义平台数据的分级制度，确保不同敏感程度的数据得到相应级别的保护和管理。

## 适用范围
平台所有数据的分类、标记、处理、存储、传输、共享、导出、删除。

## 数据等级划分

### 等级定义
| 等级 | 名称 | 说明 | 风险程度 |
|------|------|------|----------|
| L0 | 公开 | 可对外公开，无敏感风险 | 无风险 |
| L1 | 内部 | 仅限内部使用，不对外公开 | 低风险 |
| L2 | 敏感 | 包含敏感信息，需严格控制 | 中风险 |
| L3 | 受限 | 高敏感信息，需最高级别保护 | 高风险 |
| L4 | 禁止 | 法律/合规禁止存储或处理 | 极高风险 |

### 各等级典型示例

#### L0 公开数据
| 数据类型 | 示例 |
|----------|------|
| 产品说明 | 功能介绍、使用指南 |
| 公开配置 | 非敏感系统配置 |
| 统计摘要 | 聚合统计数据（无个体信息） |
| 公开文档 | 技术文档、API说明 |

#### L1 内部数据
| 数据类型 | 示例 |
|----------|------|
| 内部配置 | 系统内部配置参数 |
| 操作日志 | 非敏感操作记录 |
| 性能指标 | 系统性能监控数据 |
| 内部报告 | 内部运营报告 |

#### L2 敏感数据
| 数据类型 | 示例 |
|----------|------|
| 用户行为 | 用户操作记录、偏好设置 |
| 业务数据 | 业务流程数据、交易记录 |
| 会话内容 | 用户与系统交互内容 |
| 模型输出 | AI生成的回答内容 |

#### L3 受限数据
| 数据类型 | 示例 |
|----------|------|
| 个人身份信息 | 姓名、身份证、手机号 |
| 认证信息 | 密码、密钥、Token |
| 财务信息 | 银行账户、支付信息 |
| 健康信息 | 医疗记录、健康数据 |

#### L4 禁止数据
| 数据类型 | 示例 |
|----------|------|
| 违法内容 | 法律禁止存储的内容 |
| 未授权PII | 未经同意收集的个人信息 |
| 敏感政治 | 特定政治敏感信息 |
| 国家秘密 | 涉及国家安全的信息 |

## 分级处理规则

### 读取规则
| 等级 | 允许读取 | 条件 |
|------|----------|------|
| L0 | 所有人 | 无限制 |
| L1 | 内部用户 | 需登录认证 |
| L2 | 授权用户 | 需角色权限 |
| L3 | 特定角色 | 需审批或特定权限 |
| L4 | 禁止读取 | 不得存储 |

### 处理规则
| 等级 | 允许处理 | 条件 |
|------|----------|------|
| L0 | 所有模块 | 无限制 |
| L1 | 内部模块 | 需记录处理日志 |
| L2 | 授权模块 | 需审计追踪 |
| L3 | 特定模块 | 需审批+审计 |
| L4 | 禁止处理 | 不得进入系统 |

### 存储规则
| 等级 | 存储要求 | 加密要求 | 保留期限 |
|------|----------|----------|----------|
| L0 | 普通存储 | 无 | 按需 |
| L1 | 内部存储 | 传输加密 | 按策略 |
| L2 | 加密存储 | 存储加密 | 最小必要 |
| L3 | 隔离存储 | 强加密+访问控制 | 严格限制 |
| L4 | 禁止存储 | N/A | 立即删除 |

### 导出规则
| 等级 | 导出权限 | 审批要求 | 格式限制 |
|------|----------|----------|----------|
| L0 | 所有人 | 无 | 无限制 |
| L1 | 内部用户 | 无 | 标准格式 |
| L2 | 授权用户 | 需审批 | 脱敏可选 |
| L3 | 特定角色 | 强制审批 | 必须脱敏 |
| L4 | 禁止导出 | N/A | N/A |

### 共享规则
| 等级 | 内部共享 | 外部共享 | 跨租户共享 |
|------|----------|----------|------------|
| L0 | 允许 | 允许 | 允许 |
| L1 | 允许 | 需审批 | 禁止 |
| L2 | 需审批 | 禁止 | 禁止 |
| L3 | 特批 | 禁止 | 禁止 |
| L4 | 禁止 | 禁止 | 禁止 |

## 数据分级标记

### 标记格式
```json
{
  "data_id": "data_001",
  "classification": {
    "level": "L2",
    "category": "sensitive",
    "sub_category": "user_behavior",
    "marked_at": "2026-04-06T22:00:00+08:00",
    "marked_by": "system",
    "review_required": true
  },
  "handling_rules": {
    "encryption_required": true,
    "audit_required": true,
    "retention_days": 90,
    "export_approval": true
  }
}
```

### 自动分级规则
```yaml
auto_classification:
  rules:
    - name: "PII检测"
      patterns: ["身份证", "手机号", "邮箱"]
      action: "classify_as_L3"
    
    - name: "敏感词检测"
      patterns: ["密码", "密钥", "token"]
      action: "classify_as_L3"
    
    - name: "用户内容检测"
      patterns: ["会话", "对话", "交互"]
      action: "classify_as_L2"
    
    - name: "内部数据检测"
      patterns: ["配置", "日志", "指标"]
      action: "classify_as_L1"
```

## 分级审计

### 审计要求
| 等级 | 审计频率 | 审计内容 | 保留期限 |
|------|----------|----------|----------|
| L0 | 月度 | 访问统计 | 30天 |
| L1 | 周度 | 访问+处理 | 90天 |
| L2 | 日度 | 全量操作 | 180天 |
| L3 | 实时 | 全量+详情 | 365天 |

### 审计日志格式
```json
{
  "audit_id": "audit_001",
  "timestamp": "2026-04-06T22:00:00+08:00",
  "data_id": "data_001",
  "data_level": "L2",
  "operation": "read",
  "operator": "user_001",
  "operator_role": "analyst",
  "tenant_id": "tenant_001",
  "result": "success",
  "ip_address": "192.168.1.1",
  "user_agent": "OpenClaw/1.0"
}
```

## 租户覆盖规则

### 租户自定义分级
```yaml
tenant_override:
  allowed:
    - 提高数据级别（更严格）
    - 缩短保留期限
    - 增加审批要求
  
  not_allowed:
    - 降低数据级别
    - 延长保留期限
    - 减少审批要求
    - 绕过审计要求
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 分级冲突 | 以更高级别为准 |
| 未分级数据 | 默认L2处理 |
| 分级错误 | 立即纠正+审计 |
| 级别降级请求 | 拒绝或需合规审批 |
