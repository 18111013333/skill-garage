# CREDIT_AND_INVOICE_POLICY.md - 终极鸽子王生态信用额与发票规则

## 目的
定义生态伙伴的信用额度、账期和发票管理规则。

## 适用范围
所有渠道伙伴、转售商的信用与发票管理。

---

## 一、渠道信用额度

### 1.1 信用额度设置
| 伙伴等级 | 信用额度 | 说明 |
|----------|----------|------|
| 注册级 | 0 | 预付模式 |
| 认证级 | 5万 | 月结额度 |
| 高级级 | 20万 | 月结/季结额度 |
| 战略级 | 100万 | 季结/年结额度 |

### 1.2 信用额度管理
| 管理项 | 说明 |
|--------|------|
| 额度监控 | 实时监控额度使用 |
| 额度预警 | 使用达80%时预警 |
| 额度超限 | 超限后需预付 |
| 额度调整 | 可申请调整额度 |

---

## 二、企业账期

### 2.1 账期设置
| 伙伴等级 | 账期 | 说明 |
|----------|------|------|
| 注册级 | 预付 | 无账期 |
| 认证级 | 月结15天 | 月底结算，次月15日付款 |
| 高级级 | 月结30天 | 月底结算，次月月底付款 |
| 战略级 | 季结 | 季度结算 |

### 2.2 账期管理
```
账期管理流程
    │
    ├── 账单生成
    │   └── 周期结束时生成账单
    │
    ├── 账单确认
    │   └── 伙伴确认账单
    │
    ├── 付款提醒
    │   └── 到期前提醒
    │
    └── 逾期处理
        └── 按欠费政策处理
```

---

## 三、发票开具对象

### 3.1 发票类型
| 类型 | 说明 | 开具方 |
|------|------|--------|
| 平台发票 | 平台向伙伴开具 | 平台 |
| 伙伴发票 | 伙伴向客户开具 | 伙伴 |

### 3.2 发票要求
| 要求项 | 说明 |
|--------|------|
| 开票主体 | 与合同主体一致 |
| 开票内容 | 与实际交易一致 |
| 开票时间 | 付款后5个工作日内 |
| 税率 | 按国家税法规定 |

---

## 四、多方结算发票归属

### 4.1 结算场景
| 场景 | 发票归属 |
|------|----------|
| 平台→伙伴 | 平台向伙伴开具 |
| 伙伴→客户 | 伙伴向客户开具 |
| 市场资产 | 平台向购买方开具 |

### 4.2 发票流转
```
发票流转
    │
    ├── 转售模式
    │   ├── 平台 → 伙伴 (厂商价发票)
    │   └── 伙伴 → 客户 (转售价发票)
    │
    └── 市场模式
        └── 平台 → 购买方 (销售发票)
```

---

## 五、争议处理

### 5.1 争议类型
| 争议类型 | 处理方式 |
|----------|----------|
| 金额争议 | 核对账单，协商解决 |
| 发票争议 | 核实信息，重新开具 |
| 账期争议 | 按合同约定处理 |

### 5.2 争议流程
```
争议处理流程
    ↓
┌─────────────────────────────────────┐
│ 1. 争议提交                          │
│    - 说明争议内容                    │
│    - 提供证据材料                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 争议审核                          │
│    - 核实争议内容                    │
│    - 分析争议原因                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 争议解决                          │
│    - 协商解决方案                    │
│    - 执行解决方案                    │
└─────────────────────────────────────┘
```
