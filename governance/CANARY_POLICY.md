# CANARY_POLICY.md - 灰度发布策略

## 目的
定义灰度发布规则，确保新版本安全上线。

## 适用范围
所有系统变更的灰度发布。

## 发布阶段

### 阶段定义
| 阶段 | 流量比例 | 持续时间 | 通过条件 |
|------|----------|----------|----------|
| Canary | 5% | 1小时 | 错误率 < 1% |
| Early | 20% | 2小时 | 错误率 < 2% |
| Mid | 50% | 4小时 | 错误率 < 3% |
| Full | 100% | - | 稳定运行 |

### 阶段转换
```
Canary (5%)
    ↓ 错误率 < 1% 且 持续1小时
Early (20%)
    ↓ 错误率 < 2% 且 持续2小时
Mid (50%)
    ↓ 错误率 < 3% 且 持续4小时
Full (100%)
```

## 发布类型

### 配置变更
| 变更类型 | 灰度策略 | 回滚时间 |
|----------|----------|----------|
| 规则调整 | Canary → Full | 5分钟 |
| 阈值修改 | Canary → Full | 5分钟 |
| 新增规则 | Canary → Full | 10分钟 |

### 技能变更
| 变更类型 | 灰度策略 | 回滚时间 |
|----------|----------|----------|
| 技能更新 | Canary → Full | 10分钟 |
| 新增技能 | Canary → Full | 15分钟 |
| 移除技能 | Canary → Full | 5分钟 |

### 系统升级
| 变更类型 | 灰度策略 | 回滚时间 |
|----------|----------|----------|
| 小版本升级 | Canary → Full | 15分钟 |
| 大版本升级 | Canary → Early → Mid → Full | 30分钟 |
| 架构变更 | Canary → Early → Mid → Full | 1小时 |

## 发布检查

### 发布前检查
| 检查项 | 通过条件 |
|--------|----------|
| 单元测试 | 100%通过 |
| 集成测试 | 100%通过 |
| 回归测试 | 100%通过 |
| 配置验证 | 配置有效 |
| 依赖检查 | 依赖满足 |

### 发布中监控
| 监控项 | 告警阈值 |
|--------|----------|
| 错误率 | > 5% |
| 延迟 | > 2倍基线 |
| 资源使用 | > 90% |
| 用户反馈 | 负面 > 10% |

### 发布后验证
| 验证项 | 通过条件 |
|--------|----------|
| 功能正确 | 核心功能正常 |
| 性能达标 | 性能不下降 |
| 无新错误 | 无新增错误 |
| 用户满意 | 无负面反馈 |

## 回滚策略

### 自动回滚
| 触发条件 | 回滚范围 |
|----------|----------|
| 错误率 > 10% | 立即全量回滚 |
| 错误率 > 5% | 回滚到上一阶段 |
| 严重错误 | 立即全量回滚 |
| 安全问题 | 立即全量回滚 |

### 手动回滚
| 触发条件 | 回滚范围 |
|----------|----------|
| 用户投诉 | 按需回滚 |
| 性能问题 | 按需回滚 |
| 功能异常 | 按需回滚 |

### 回滚流程
```
检测到问题
    ↓
┌─────────────────────────────────────┐
│ 1. 停止发布                          │
│    - 暂停流量切换                    │
│    - 保持当前状态                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 执行回滚                          │
│    - 切换到旧版本                    │
│    - 恢复旧配置                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 验证回滚                          │
│    - 检查系统状态                    │
│    - 验证功能正常                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 记录分析                          │
│    - 记录回滚原因                    │
│    - 分析问题根因                    │
└─────────────────────────────────────┘
```

## 发布审批

### 审批级别
| 变更类型 | 审批要求 |
|----------|----------|
| 配置微调 | 自动审批 |
| 技能更新 | 自动审批 |
| 系统升级 | 人工审批 |
| 架构变更 | 人工审批 + 测试验证 |

### 审批流程
```
发布请求
    ↓
┌─────────────────────────────────────┐
│ 1. 自动检查                          │
│    - 测试通过                        │
│    - 配置有效                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 审批决策                          │
│    - 自动审批                        │
│    - 人工审批                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 执行发布                          │
│    - 灰度发布                        │
│    - 监控验证                        │
└─────────────────────────────────────┘
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 发布失败 | 自动回滚 |
| 监控异常 | 暂停发布 |
| 审批拒绝 | 取消发布 |

## 维护方式
- 调整阶段比例: 修改阶段定义
- 新增发布类型: 添加到类型表
- 调整回滚条件: 更新回滚规则
