# ALERT_POLICY.md - 告警策略

## 目的
定义系统告警规则，确保异常情况及时发现和处理。

## 适用范围
所有需要告警的系统事件。

## 告警级别

### 级别定义
| 级别 | 说明 | 响应时间 | 通知方式 |
|------|------|----------|----------|
| CRITICAL | 严重故障 | 立即 | 全渠道通知 |
| WARNING | 警告 | 1小时内 | 日志 + 消息 |
| INFO | 信息 | 无需响应 | 日志 |

### 级别判定
| 条件 | 级别 |
|------|------|
| 系统不可用 | CRITICAL |
| 数据丢失风险 | CRITICAL |
| 安全违规 | CRITICAL |
| 性能严重下降 | WARNING |
| 错误率升高 | WARNING |
| 资源接近上限 | WARNING |
| 常规操作记录 | INFO |

## 告警规则

### 系统告警
| 规则ID | 条件 | 级别 | 说明 |
|--------|------|------|------|
| SYS001 | 健康度 < 70 | CRITICAL | 系统健康度严重下降 |
| SYS002 | 健康度 < 85 | WARNING | 系统健康度下降 |
| SYS003 | 内存使用 > 90% | WARNING | 内存接近上限 |
| SYS004 | CPU使用 > 80% | WARNING | CPU负载过高 |

### 执行告警
| 规则ID | 条件 | 级别 | 说明 |
|--------|------|------|------|
| EXEC001 | 连续失败 ≥ 3 | WARNING | 执行连续失败 |
| EXEC002 | 错误率 > 50% | CRITICAL | 错误率过高 |
| EXEC003 | 平均延迟 > 60s | WARNING | 执行延迟过高 |
| EXEC004 | 步骤超限 > 10次/h | WARNING | 步骤超限频繁 |

### 安全告警
| 规则ID | 条件 | 级别 | 说明 |
|--------|------|------|------|
| SEC001 | 安全违规 > 0 | CRITICAL | 发生安全违规 |
| SEC002 | 权限拒绝 > 5次/h | WARNING | 权限拒绝频繁 |
| SEC003 | 敏感操作 > 10次/h | WARNING | 敏感操作频繁 |

### 资源告警
| 规则ID | 条件 | 级别 | 说明 |
|--------|------|------|------|
| RES001 | Token预算 > 80% | WARNING | Token预算接近上限 |
| RES002 | 存储空间 < 10% | WARNING | 存储空间不足 |
| RES003 | 并发数 > 80% | WARNING | 并发接近上限 |

## 告警流程

```
事件发生
    ↓
┌─────────────────────────────────────┐
│ 1. 检测告警条件                      │
│    - 匹配告警规则                    │
│    - 计算告警级别                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 生成告警                          │
│    - 创建告警记录                    │
│    - 分配告警ID                      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 发送通知                          │
│    - 按级别选择渠道                  │
│    - 发送告警消息                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 记录日志                          │
│    - 写入告警日志                    │
│    - 更新统计指标                    │
└─────────────────────────────────────┘
```

## 告警通知

### 通知渠道
| 级别 | 渠道 |
|------|------|
| CRITICAL | 即时消息 + 日志 + 邮件 |
| WARNING | 日志 + 定期汇总 |
| INFO | 日志 |

### 通知模板
| 级别 | 模板 |
|------|------|
| CRITICAL | 🚨 [CRITICAL] {rule}: {message} |
| WARNING | ⚠️ [WARNING] {rule}: {message} |
| INFO | ℹ️ [INFO] {rule}: {message} |

### 通知频率限制
| 级别 | 同规则冷却时间 |
|------|----------------|
| CRITICAL | 5分钟 |
| WARNING | 15分钟 |
| INFO | 1小时 |

## 告警处理

### 处理流程
```
告警产生
    ↓
┌─────────────────────────────────────┐
│ 1. 确认告警                          │
│    - 查看告警详情                    │
│    - 评估影响范围                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 分析原因                          │
│    - 查看相关日志                    │
│    - 分析根因                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 采取措施                          │
│    - 自动修复                        │
│    - 手动干预                        │
│    - 升级处理                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 确认恢复                          │
│    - 验证问题解决                    │
│    - 更新告警状态                    │
└─────────────────────────────────────┘
```

### 自动处理
| 告警类型 | 自动处理 |
|----------|----------|
| 内存不足 | 清理缓存 |
| 连续失败 | 进入降级模式 |
| 资源超限 | 自动扩容或限制 |

## 告警统计

### 统计指标
| 指标 | 说明 |
|------|------|
| 告警总数 | 产生的告警数量 |
| 按级别统计 | 各级别告警数量 |
| 按规则统计 | 各规则触发次数 |
| 平均响应时间 | 告警确认时间 |
| 恢复率 | 成功恢复的比例 |

### 统计周期
| 周期 | 统计内容 |
|------|----------|
| 实时 | 当前活跃告警 |
| 每小时 | 告警趋势 |
| 每天 | 告警汇总 |
| 每周 | 告警分析 |

## 告警抑制

### 抑制条件
| 条件 | 说明 |
|------|------|
| 重复告警 | 同规则冷却期内 |
| 已知维护 | 维护窗口期 |
| 依赖告警 | 上游告警未恢复 |

### 抑制规则
```json
{
  "suppression": [
    {
      "condition": "same_rule_within_cooldown",
      "action": "suppress"
    },
    {
      "condition": "maintenance_window",
      "action": "suppress"
    },
    {
      "condition": "dependency_alert_active",
      "action": "suppress"
    }
  ]
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 通知失败 | 重试 + 备用渠道 |
| 日志写入失败 | 缓存后重试 |
| 告警风暴 | 自动抑制 |

## 维护方式
- 新增告警规则: 添加到规则表
- 调整告警级别: 修改级别定义
- 调整通知渠道: 更新通知配置
