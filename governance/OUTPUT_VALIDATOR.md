# OUTPUT_VALIDATOR.md - 输出校验规则

## 目的
定义输出内容的校验规则，确保输出安全、合规、高质量。

## 适用范围
所有返回给用户的输出内容。

## 校验维度

### 格式校验
| 校验项 | 规则 | 失败动作 |
|--------|------|----------|
| JSON格式 | 必须是有效JSON | 返回错误 |
| Markdown格式 | 必须是有效Markdown | 修复格式 |
| 编码 | 必须是UTF-8 | 转换编码 |
| 长度 | 不超过限制 | 截断 |

### 内容校验
| 校验项 | 规则 | 失败动作 |
|--------|------|----------|
| 敏感信息 | 不得包含密码/密钥 | 脱敏处理 |
| 个人隐私 | 不得泄露隐私 | 脱敏处理 |
| 有害内容 | 不得包含有害内容 | 拒绝输出 |
| 幻觉检测 | 不得编造事实 | 标记不确定 |

### 安全校验
| 校验项 | 规则 | 失败动作 |
|--------|------|----------|
| 注入攻击 | 不得包含注入代码 | 清理 |
| XSS | 不得包含XSS脚本 | 清理 |
| 路径泄露 | 不得泄露系统路径 | 脱敏 |
| 权限泄露 | 不得泄露权限信息 | 脱敏 |

## 校验流程

```
输出内容
    ↓
┌─────────────────────────────────────┐
│ 1. 格式校验                          │
│    - JSON/Markdown/编码/长度         │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 内容校验                          │
│    - 敏感信息/隐私/有害/幻觉          │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 安全校验                          │
│    - 注入/XSS/路径/权限              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 质量评分                          │
│    - 完整性/准确性/相关性             │
└─────────────────────────────────────┘
    ↓
返回校验结果
```

## 脱敏规则

### 敏感信息脱敏
| 类型 | 匹配规则 | 脱敏方式 |
|------|----------|----------|
| API密钥 | `[a-zA-Z0-9]{32,}` | `[REDACTED_KEY]` |
| 密码 | `password=xxx` | `[REDACTED_PASSWORD]` |
| 手机号 | `1[3-9]\d{9}` | `138****1234` |
| 邮箱 | `.*@.*\.com` | `a***@example.com` |
| 身份证 | `\d{17}[\dXx]` | `110***********1234` |

### 路径脱敏
| 类型 | 脱敏方式 |
|------|----------|
| 绝对路径 | 保留文件名，隐藏目录 |
| 用户目录 | 替换为 `~` |
| 临时目录 | 替换为 `/tmp` |

## 质量评分

### 评分维度
| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 30% | 是否完整回答问题 |
| 准确性 | 40% | 信息是否准确 |
| 相关性 | 20% | 是否与问题相关 |
| 可读性 | 10% | 是否易于理解 |

### 评分阈值
| 分数范围 | 动作 |
|----------|------|
| 90-100 | 直接输出 |
| 70-89 | 输出 + 提示改进 |
| 50-69 | 重新生成 |
| < 50 | 拒绝输出 |

## 幻觉检测

### 检测规则
| 规则 | 说明 |
|------|------|
| 事实核查 | 关键事实必须有来源 |
| 时间一致性 | 时间信息必须合理 |
| 逻辑一致性 | 逻辑必须自洽 |
| 来源追溯 | 可追溯信息来源 |

### 处理策略
| 检测结果 | 处理 |
|----------|------|
| 确认事实 | 正常输出 |
| 可能幻觉 | 标记"待确认" |
| 确认幻觉 | 拒绝输出或修正 |

## 校验结果格式

```json
{
  "validationId": "val_001",
  "timestamp": "2026-04-06T10:32:00+08:00",
  "input": "原始输出内容",
  "output": "校验后内容",
  "checks": {
    "format": { "passed": true, "details": {} },
    "content": { "passed": true, "details": {} },
    "security": { "passed": true, "details": {} }
  },
  "qualityScore": 85,
  "actions": ["脱敏处理: 手机号"],
  "warnings": []
}
```

## 异常处理

| 异常 | 处理 |
|------|------|
| 校验失败 | 返回错误 + 原因 |
| 脱敏失败 | 拒绝输出 |
| 评分过低 | 重新生成 |

## 维护方式
- 新增校验规则: 更新对应校验表
- 调整脱敏规则: 更新脱敏规则表
- 调整评分权重: 更新评分维度表
