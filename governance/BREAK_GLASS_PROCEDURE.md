# BREAK_GLASS_PROCEDURE.md - 紧急安全响应特权流程

## 目的
定义紧急安全响应特权流程（Break-Glass），确保极端情况下可快速止血，但不会成为长期后门。

## 适用范围
平台所有紧急安全响应、特权访问、应急操作。

---

## 一、Break-Glass 原则

| 原则 | 说明 | 执行要求 |
|------|------|----------|
| 最后手段 | 仅在紧急情况下使用 | 其他方法无效时 |
| 最小权限 | 仅授予必要权限 | 按需授权 |
| 最短时间 | 权限时间最短化 | 自动过期 |
| 强审计 | 所有操作强审计 | 完整记录 |
| 事后复核 | 事后必须复核 | 24 小时内复核 |
| 权限回收 | 使用后立即回收 | 自动回收 |

---

## 二、Break-Glass 触发场景

### 2.1 允许触发的场景
| 场景 | 代码 | 严重程度 | 说明 |
|------|------|----------|------|
| 正在进行的攻击 | BG-ATTACK | CRITICAL | 检测到正在进行的攻击需要紧急阻断 |
| 数据泄露事件 | BG-LEAK | CRITICAL | 检测到数据正在泄露需要紧急阻止 |
| 系统被入侵 | BG-COMPROMISE | CRITICAL | 系统被入侵需要紧急隔离 |
| 恶意代码执行 | BG-MALWARE | CRITICAL | 检测到恶意代码需要紧急终止 |
| 权限被滥用 | BG-ABUSE | HIGH | 检测到权限滥用需要紧急撤销 |
| 服务被劫持 | BG-HIJACK | CRITICAL | 服务被劫持需要紧急恢复 |
| 灾难性故障 | BG-DISASTER | CRITICAL | 灾难性故障需要紧急恢复 |

### 2.2 禁止触发的场景
| 场景 | 说明 | 正确做法 |
|------|------|----------|
| 日常运维 | 日常运维操作 | 使用正常流程 |
| 性能问题 | 性能优化或调试 | 使用正常流程 |
| 功能开发 | 功能开发或测试 | 使用正常流程 |
| 便捷操作 | 为便捷而绕过流程 | 禁止 |
| 个人需求 | 个人特殊需求 | 禁止 |

### 2.3 触发条件判定
| 条件 | 判定标准 | 证据要求 |
|------|----------|----------|
| 紧急性 | 需要立即行动 | 时间敏感证据 |
| 严重性 | 影响重大 | 影响评估 |
| 无替代 | 无其他方法 | 尝试记录 |
| 授权 | 有权触发 | 角色验证 |

## 三、触发权限

### 3.1 有权触发的人员
| 角色 | 触发权限 | 触发范围 | 审批要求 |
|------|----------|----------|----------|
| CISO | 完全权限 | 所有系统 | 自行触发 |
| 安全负责人 | 高级权限 | 安全系统 | 自行触发 |
| 技术负责人 | 中级权限 | 技术系统 | 安全负责人确认 |
| 运维负责人 | 基础权限 | 运维系统 | 技术负责人 + 安全负责人确认 |
| 值班人员 | 紧急权限 | 指定系统 | 事后 1 小时内确认 |

### 3.2 权限级别
| 级别 | 权限范围 | 有效时间 | 审计要求 |
|------|----------|----------|----------|
| L1-基础 | 只读 + 日志 | 1 小时 | 完整审计 |
| L2-中级 | 读取 + 配置变更 | 30 分钟 | 完整审计 + 实时告警 |
| L3-高级 | 读取 + 写入 + 配置 | 15 分钟 | 完整审计 + 实时告警 + 双人确认 |
| L4-完全 | 完全访问 | 10 分钟 | 完整审计 + 实时告警 + 双人确认 + 录屏 |

### 3.3 权限范围限制
| 系统 | L1 | L2 | L3 | L4 |
|------|----|----|----|----|
| 用户数据 | 只读 | 只读 | 脱敏读写 | 完全访问 |
| 系统配置 | 只读 | 读写 | 读写 | 完全访问 |
| 安全配置 | 只读 | 只读 | 读写 | 完全访问 |
| 审计日志 | 只读 | 只读 | 只读 | 完全访问 |
| 密钥凭证 | 无 | 无 | 无 | 受限访问 |

## 四、触发流程

### 4.1 标准触发流程
```
紧急事件发生
    ↓
┌─────────────────────────────────────┐
│ 1. 事件确认                          │
│    - 确认事件类型                    │
│    - 确认严重程度                    │
│    - 确认需要 Break-Glass            │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 权限申请                          │
│    - 选择权限级别                    │
│    - 说明使用目的                    │
│    - 预计使用时间                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 审批确认                          │
│    - 自动验证资格                    │
│    - 如需确认，通知确认人            │
│    - 生成临时凭证                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 权限使用                          │
│    - 使用临时凭证                    │
│    - 执行必要操作                    │
│    - 全程审计记录                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 权限回收                          │
│    - 时间到期自动回收                │
│    - 或主动归还                      │
│    - 凭证立即失效                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 6. 事后复核                          │
│    - 审计记录复核                    │
│    - 操作合理性评估                  │
│    - 改进建议                        │
└─────────────────────────────────────┘
```

### 4.2 紧急触发流程
```
极端紧急情况
    ↓
自行触发（最高权限者）
    ↓
立即执行操作
    ↓
同时通知相关人员
    ↓
操作完成后报告
    ↓
24 小时内完整复核
```

## 五、权限时效

### 5.1 时间限制
| 权限级别 | 最大有效时间 | 自动延长 | 强制回收 |
|----------|--------------|----------|----------|
| L1 | 1 小时 | 不允许 | 到期强制 |
| L2 | 30 分钟 | 不允许 | 到期强制 |
| L3 | 15 分钟 | 需确认 | 到期强制 |
| L4 | 10 分钟 | 不允许 | 到期强制 |

### 5.2 时间监控
| 监控项 | 监控方式 | 告警阈值 |
|--------|----------|----------|
| 剩余时间 | 实时显示 | < 5 分钟告警 |
| 使用时间 | 实时记录 | 超时告警 |
| 操作频率 | 实时监控 | 异常频率告警 |

### 5.3 强制回收
| 触发条件 | 回收动作 | 通知 |
|----------|----------|------|
| 时间到期 | 立即回收 | 通知使用者 |
| 主动归还 | 立即回收 | 记录日志 |
| 异常操作 | 立即回收 | 告警 + 通知 |
| 紧急终止 | 立即回收 | 告警 + 通知 |

## 七、事后复核

### 7.1 复核时限
| 权限级别 | 复核时限 | 复核人 |
|----------|----------|--------|
| L1 | 24 小时 | 安全负责人 |
| L2 | 12 小时 | CISO |
| L3 | 6 小时 | CISO + 法务 |
| L4 | 2 小时 | CISO + 法务 + CEO |

### 7.2 复核内容
| 复核项 | 复核内容 | 判定标准 |
|--------|----------|----------|
| 触发合理性 | 触发是否合理 | 符合触发场景 |
| 权限适当性 | 权限是否适当 | 权限与需求匹配 |
| 操作必要性 | 操作是否必要 | 操作解决问题 |
| 操作合规性 | 操作是否合规 | 无违规操作 |
| 流程完整性 | 流程是否完整 | 流程正确执行 |

### 7.3 复核结果
| 结果 | 说明 | 后续动作 |
|------|------|----------|
| 通过 | 操作合理合规 | 归档记录 |
| 需改进 | 有改进空间 | 制定改进措施 |
| 需调查 | 存在问题 | 启动调查 |
| 违规 | 存在违规 | 纪律处分 |

## 八、权限回收

### 8.1 自动回收
| 触发条件 | 回收动作 | 说明 |
|----------|----------|------|
| 时间到期 | 立即回收 | 自动执行 |
| 操作完成 | 立即回收 | 使用者标记完成 |
| 会话结束 | 立即回收 | 会话断开 |

### 8.2 强制回收
| 触发条件 | 执行人 | 说明 |
|----------|--------|------|
| 异常操作 | 安全团队 | 检测到异常 |
| 滥用嫌疑 | CISO | 怀疑滥用 |
| 紧急终止 | CISO | 紧急情况 |

### 8.3 回收验证
| 验证项 | 验证方法 | 通过标准 |
|--------|----------|----------|
| 凭证失效 | 访问测试 | 无法访问 |
| 权限清除 | 权限检查 | 权限已清除 |
| 会话终止 | 会话检查 | 会话已终止 |
| 日志完整 | 日志检查 | 日志完整 |

## 九、滥用防范

### 9.1 滥用检测
| 检测项 | 检测方法 | 告警条件 |
|--------|----------|----------|
| 频繁触发 | 统计分析 | 短时间多次触发 |
| 无效触发 | 结果分析 | 触发后无实际操作 |
| 越权操作 | 权限检查 | 操作超出授权范围 |
| 敏感操作 | 操作分析 | 访问敏感数据 |

### 9.2 滥用处罚
| 滥用类型 | 处罚措施 |
|----------|----------|
| 无效触发 | 警告 + 培训 |
| 越权操作 | 权限暂停 + 调查 |
| 数据滥用 | 纪律处分 + 法律追责 |
| 恶意使用 | 解雇 + 法律追责 |
