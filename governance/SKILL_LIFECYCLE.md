# SKILL_LIFECYCLE.md - 技能生命周期管理

## 目的
规定技能如何上线、灰度、降级、废弃、合并。

## 适用范围
所有技能的生命周期管理。

## 生命周期阶段

```
开发 → 测试 → 灰度 → 正式 → 降级 → 废弃/合并
```

## 上线流程

### 开发阶段
| 步骤 | 要求 | 说明 |
|------|------|------|
| 1. 设计 | 提交设计文档 | 说明目的、输入输出、依赖 |
| 2. 开发 | 遵循开发规范 | 代码规范、安全规范 |
| 3. 单元测试 | 覆盖率 > 80% | 核心功能测试 |
| 4. 文档 | 完善SKILL.md | 使用说明、示例 |

### 测试阶段
| 步骤 | 要求 | 说明 |
|------|------|------|
| 1. 集成测试 | 通过集成测试 | 与其他技能协作测试 |
| 2. 安全测试 | 通过安全审计 | 无安全漏洞 |
| 3. 性能测试 | 满足性能要求 | 响应时间、资源占用 |
| 4. 回归测试 | 通过回归门禁 | 不影响现有功能 |

### 灰度阶段
| 步骤 | 要求 | 说明 |
|------|------|------|
| 1. 配置灰度 | 设置灰度比例 | 初始 5% |
| 2. 监控指标 | 监控关键指标 | 成功率、延迟、错误 |
| 3. 逐步放量 | 按比例增加 | 5% → 20% → 50% → 100% |
| 4. 全量上线 | 移除灰度配置 | 正式启用 |

## 灰度策略

### 灰度配置
```json
{
  "skillName": "new-skill",
  "grayPercentage": 5,
  "targetUsers": ["user_group_a"],
  "startDate": "2026-04-06",
  "endDate": "2026-04-13",
  "rollbackThreshold": {
    "errorRate": 0.05,
    "latencyIncrease": 0.2
  }
}
```

### 灰度放量规则
| 阶段 | 比例 | 持续时间 | 通过条件 |
|------|------|----------|----------|
| 阶段1 | 5% | 2天 | 错误率 < 5% |
| 阶段2 | 20% | 2天 | 错误率 < 3% |
| 阶段3 | 50% | 2天 | 错误率 < 2% |
| 阶段4 | 100% | - | 正式上线 |

## 降级触发

### 自动降级条件
| 指标 | 阈值 | 动作 |
|------|------|------|
| 成功率 | < 90% | 触发降级 |
| 平均延迟 | > 2倍基线 | 触发降级 |
| 错误率 | > 10% | 触发降级 |
| 用户投诉 | > 5次/天 | 触发降级 |

### 降级动作
1. 禁用技能
2. 启用备选技能
3. 记录降级日志
4. 通知相关人员

## 废弃流程

### 废弃条件
| 条件 | 说明 |
|------|------|
| 成功率持续低 | 连续7天 < 80% |
| 重复度高 | 与其他技能功能重复 > 90% |
| 维护成本高 | 维护成本 > 收益 |
| 使用率低 | 日调用 < 10次 |
| 安全风险 | 存在无法修复的安全问题 |

### 废弃流程
```
评估废弃
    ↓
┌─────────────────────────────────────┐
│ 1. 通知用户                          │
│    - 公告废弃计划                    │
│    - 提供替代方案                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 禁止新调用                        │
│    - 标记为 deprecated               │
│    - 返回替代建议                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 迁移现有用户                      │
│    - 提供迁移工具                    │
│    - 协助迁移数据                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 正式下线                          │
│    - 从目录移除                      │
│    - 归档代码和文档                  │
└─────────────────────────────────────┘
```

## 合并流程

### 合并条件
| 条件 | 说明 |
|------|------|
| 功能相似 | 多个技能功能相似 |
| 依赖相同 | 依赖相同的底层服务 |
| 维护统一 | 由同一团队维护 |

### 合并流程
```
评估合并
    ↓
确定主技能和从技能
    ↓
合并功能到主技能
    ↓
迁移从技能用户
    ↓
废弃从技能
    ↓
更新路由表
```

## 技能状态

| 状态 | 说明 | 可用性 |
|------|------|--------|
| development | 开发中 | 不可用 |
| testing | 测试中 | 测试环境可用 |
| canary | 灰度中 | 部分用户可用 |
| active | 正式运行 | 全部可用 |
| deprecated | 已废弃 | 禁止新调用 |
| disabled | 已禁用 | 不可用 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 上线失败 | 回滚到上一版本 |
| 灰度异常 | 自动回滚 |
| 废弃失败 | 重新评估 |
| 合并失败 | 恢复独立运行 |

## 维护方式
- 新增状态: 更新状态表
- 调整灰度规则: 更新灰度策略
- 调整废弃条件: 更新废弃条件表
