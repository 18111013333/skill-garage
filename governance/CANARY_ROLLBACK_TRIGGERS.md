# CANARY_ROLLBACK_TRIGGERS.md - 金丝雀回滚触发规则

## 目的
定义金丝雀发布的回滚触发条件和回滚执行规则。

## 适用范围
所有金丝雀发布的自动回滚和手动回滚。

## 回滚触发分类

### 自动触发
| 触发类型 | 触发条件 | 检测方式 |
|----------|----------|----------|
| 错误率触发 | 错误率 > 阈值 | 自动检测 |
| 延迟触发 | 响应时间 > 阈值 | 自动检测 |
| 告警触发 | P0/P1 告警 | 自动检测 |
| 业务指标触发 | 业务指标下降 | 自动检测 |

### 手动触发
| 触发类型 | 触发条件 | 决策者 |
|----------|----------|--------|
| 问题发现 | 发现严重问题 | 发布负责人 |
| 用户反馈 | 用户报告问题 | 发布负责人 |
| 依赖问题 | 外部依赖问题 | 发布负责人 |
| 决策需要 | 需要人工判断 | 发布负责人 |

## 自动回滚触发条件

### 错误率触发
| 条件 | 阈值 | 说明 |
|------|------|------|
| 绝对阈值 | 错误率 > 5% | 超过绝对阈值 |
| 相对阈值 | 新版本 > 旧版本 × 2 | 相对旧版本 |
| 趋势阈值 | 错误率持续上升 | 趋势恶化 |

### 延迟触发
| 条件 | 阈值 | 说明 |
|------|------|------|
| 绝对阈值 | P95 > 5s | 超过绝对阈值 |
| 相对阈值 | 新版本 > 旧版本 × 2 | 相对旧版本 |
| 趋势阈值 | 延迟持续上升 | 趋势恶化 |

### 告警触发
| 告警级别 | 触发动作 | 说明 |
|----------|----------|------|
| P0 告警 | 立即回滚 | 严重问题 |
| P1 告警 | 立即回滚 | 重要问题 |
| P2 告警 | 评估后决定 | 需要评估 |
| P3 告警 | 记录观察 | 仅记录 |

### 业务指标触发
| 指标 | 阈值 | 说明 |
|------|------|------|
| 转化率 | 下降 > 20% | 业务影响 |
| 成功率 | 下降 > 10% | 业务影响 |
| 用户投诉 | 增加 > 50% | 用户影响 |

## 回滚执行

### 自动回滚流程
```
触发条件满足
    ↓
立即停止放量
    ↓
执行回滚操作
    ↓
恢复旧版本
    ↓
验证恢复结果
    ↓
通知相关方
    ↓
记录回滚原因
```

### 手动回滚流程
```
发现问题
    ↓
评估回滚必要性
    ↓
获取回滚审批
    ↓
执行回滚操作
    ↓
验证恢复结果
    ↓
通知相关方
    ↓
记录回滚原因
```

## 回滚验证

### 验证内容
| 验证项 | 说明 | 方法 |
|--------|------|------|
| 版本确认 | 确认已回滚到旧版本 | 版本检查 |
| 功能验证 | 确认功能正常 | 功能测试 |
| 指标验证 | 确认指标恢复 | 指标对比 |
| 告警清除 | 确认告警已清除 | 告警检查 |

### 验证标准
| 指标 | 回滚后标准 |
|------|------------|
| 错误率 | ≤ 旧版本 × 1.1 |
| 响应时间 | ≤ 旧版本 × 1.1 |
| 告警 | 无 P0/P1 告警 |
| 业务指标 | 恢复正常 |

## 回滚后处理

### 问题分析
| 分析项 | 说明 |
|--------|------|
| 根因分析 | 分析回滚原因 |
| 影响评估 | 评估影响范围 |
| 改进措施 | 制定改进措施 |
| 重新发布计划 | 制定重新发布计划 |
