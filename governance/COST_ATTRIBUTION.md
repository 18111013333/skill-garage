# COST_ATTRIBUTION.md - 成本归因规则

## 目的
定义成本归因规则，确保成本能追溯到责任主体。

## 适用范围
所有平台资源消耗、模型调用、工具执行、存储使用。

## 成本归因层级

```
┌─────────────────────────────────────────────────────────────┐
│                      成本归因层级                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Level 1: 租户 (Tenant)                                       │
│ - 最高层级归因对象                                           │
│ - 租户总成本汇总                                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Level 2: 团队 (Team)                                         │
│ - 租户下的组织单元                                           │
│ - 团队成本汇总                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Level 3: 项目 (Project)                                      │
│ - 团队下的工作单元                                           │
│ - 项目成本汇总                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Level 4: 任务 (Task)                                         │
│ - 项目下的具体任务                                           │
│ - 任务成本明细                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Level 5: Agent/集成 (Agent/Integration)                      │

└─────────────────────────────────────────────────────────────┘
```

## 成本类型

### 直接成本
| 成本类型 | 归因规则 | 计量单位 |
|----------|----------|----------|
| 模型调用 | 按调用归因 | Token 数 |
| 工具执行 | 按执行归因 | 执行次数 |
| 存储使用 | 按存储归因 | GB-月 |
| 网络流量 | 按流量归因 | GB |
| API 调用 | 按调用归因 | 调用次数 |

### 间接成本
| 成本类型 | 分摊规则 | 分摊依据 |
|----------|----------|----------|
| 平台基础设施 | 按使用量分摊 | 资源消耗比例 |
| 共享服务 | 按调用量分摊 | 调用次数比例 |
| 管理开销 | 按租户分摊 | 租户规模比例 |
| 安全审计 | 按审计量分摊 | 审计记录比例 |

## 归因规则

### 租户归因
```json
{
  "tenant_id": "tenant-001",
  "attribution_rules": {
    "direct_costs": "全额归因",
    "shared_infrastructure": "按使用量比例",
    "platform_services": "按调用量比例"
  }
}
```

### 团队归因
```json
{
  "team_id": "team-001",
  "parent_tenant": "tenant-001",
  "attribution_rules": {
    "team_direct": "全额归因",
    "team_shared": "按成员数比例",
    "cross_team": "按实际使用"
  }
}
```

### 项目归因
```json
{
  "project_id": "project-001",
  "parent_team": "team-001",
  "attribution_rules": {
    "project_direct": "全额归因",
    "project_shared": "按任务数比例",
    "cross_project": "按实际使用"
  }
}
```

### 任务归因
```json
{
  "task_id": "task-001",
  "parent_project": "project-001",
  "attribution_rules": {
    "model_cost": "按实际 Token",
    "tool_cost": "按实际执行",
    "retry_cost": "归因到原任务",
    "validation_cost": "归因到触发任务"
  }
}
```

### Agent/集成归因
```json
{
  "agent_id": "agent-001",
  "parent_task": "task-001",
  "attribution_rules": {
    "inference_cost": "按实际调用",
    "tool_cost": "按实际执行",
    "memory_cost": "按实际读写",
    "external_api_cost": "按实际调用"
  }
}
```

## 共享成本分摊

### 分摊方法
| 方法 | 适用场景 | 计算公式 |
|------|----------|----------|
| 按使用量 | 资源消耗 | 成本 × (使用量/总使用量) |
| 按调用量 | 服务调用 | 成本 × (调用量/总调用量) |
| 按成员数 | 团队共享 | 成本 × (成员数/总成员数) |
| 按任务数 | 项目共享 | 成本 × (任务数/总任务数) |
| 平均分摊 | 无法区分 | 成本 / 分摊对象数 |

### 分摊示例
```
共享基础设施成本: ¥10,000/月

租户 A 使用量: 60%
租户 B 使用量: 30%
租户 C 使用量: 10%

分摊结果:
- 租户 A: ¥6,000
- 租户 B: ¥3,000
- 租户 C: ¥1,000
```

## 无法归因处理

### 默认池
| 情况 | 处理方式 |
|------|----------|
| 无法识别租户 | 归入平台默认池 |
| 无法识别团队 | 归入租户默认池 |
| 无法识别项目 | 归入团队默认池 |
| 无法识别任务 | 归入项目默认池 |

### 默认池分配
| 默认池类型 | 分配周期 | 分配规则 |
|------------|----------|----------|
| 平台默认池 | 月度 | 按租户规模比例 |
| 租户默认池 | 月度 | 按团队规模比例 |
| 团队默认池 | 月度 | 按项目规模比例 |
| 项目默认池 | 月度 | 按任务规模比例 |

## 成本报告

### 报告层级
| 报告类型 | 内容 | 频率 |
|----------|------|------|
| 租户报告 | 租户总成本 | 日/周/月 |
| 团队报告 | 团队成本明细 | 周/月 |
| 项目报告 | 项目成本明细 | 周/月 |
| 任务报告 | 任务成本明细 | 按需 |

### 报告内容
- 各层级成本汇总
- 成本趋势分析
- 成本构成分析
- 异常成本预警
- 成本优化建议

## 联动机制

### 与 Usage Schema 联动
- 使用量记录自动归因
- 归因规则自动应用
- 成本计算自动执行

### 与 Optimization 联动
- 成本数据驱动优化
- 优化效果成本归因
- 成本预警触发优化

### 与 Audit 联动
- 成本归因记录审计
- 异常成本审计追踪
- 成本报告审计存档

### 与 Portfolio 联动
- 项目成本纳入组合视图
- 组合成本汇总分析
- 投资回报成本关联
