# ANSWER_VALIDATOR.md - 答案校验规则

## 目的
定义最终答案生成前的检查项，确保每个回答在输出前都有一轮轻校验。

## 适用范围
所有返回给用户的最终答案。

## 校验检查项

### 答非所问检查
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 问题覆盖 | 是否回答了用户的问题 | 重新生成 |
| 意图匹配 | 是否符合用户意图 | 重新生成 |
| 主题一致 | 是否偏离主题 | 修正或重新生成 |

### 硬约束检查
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 格式约束 | 是否满足格式要求 | 格式修正 |
| 长度约束 | 是否满足长度要求 | 截断或扩展 |
| 内容约束 | 是否包含必须内容 | 补充内容 |
| 禁止内容 | 是否包含禁止内容 | 移除或重新生成 |

### 逻辑冲突检查
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 内部一致 | 回答内部是否矛盾 | 修正矛盾 |
| 与已知一致 | 是否与已知信息矛盾 | 标注或修正 |
| 因果合理 | 因果关系是否合理 | 修正逻辑 |

### 无依据扩展检查
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 事实依据 | 事实是否有依据 | 添加依据或标注不确定 |
| 推断标注 | 推断是否标注 | 添加推断标记 |
| 数据来源 | 数据是否有来源 | 添加来源 |

### 格式符合检查
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 格式正确 | 是否符合要求格式 | 格式修正 |
| 结构完整 | 结构是否完整 | 补充结构 |
| 可读性 | 是否易于阅读 | 优化排版 |

## 校验流程

```
答案生成
    ↓
┌─────────────────────────────────────┐
│ 1. 答非所问检查                      │
│    - 问题覆盖检查                    │
│    - 意图匹配检查                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 硬约束检查                        │
│    - 格式约束                        │
│    - 内容约束                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 逻辑冲突检查                      │
│    - 内部一致性                      │
│    - 与已知一致性                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 4. 无依据扩展检查                    │
│    - 事实依据                        │
│    - 推断标注                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 5. 格式符合检查                      │
│    - 格式正确性                      │
│    - 可读性                          │
└─────────────────────────────────────┘
    ↓
通过/修正后输出
```

## 校验结果

### 通过
```json
{
  "validationId": "val_001",
  "result": "pass",
  "checks": {
    "question_coverage": "pass",
    "constraint_satisfaction": "pass",
    "logic_consistency": "pass",
    "evidence_support": "pass",
    "format_compliance": "pass"
  }
}
```

### 需修正
```json
{
  "validationId": "val_002",
  "result": "needs_fix",
  "issues": [
    {
      "type": "format_error",
      "description": "用户要求表格格式，实际为文本",
      "fix": "转换为表格格式"
    }
  ]
}
```

### 需重新生成
```json
{
  "validationId": "val_003",
  "result": "regenerate",
  "issues": [
    {
      "type": "off_topic",
      "description": "回答偏离用户问题",
      "action": "重新生成"
    }
  ]
}
```

## 校验阈值

| 检查项 | 通过阈值 | 说明 |
|--------|----------|------|
| 问题覆盖 | ≥ 80% | 至少覆盖80%问题点 |
| 约束满足 | 100% | 硬约束必须全部满足 |
| 逻辑一致 | 无矛盾 | 不能有内部矛盾 |
| 依据支持 | ≥ 70% | 事实需有依据支持 |

## 异常处理

| 异常 | 处理 |
|------|------|
| 校验超时 | 跳过部分检查，标注警告 |
| 校验失败 | 按需修正或重新生成 |
| 多次失败 | 标注问题，请求用户确认 |

## 维护方式
- 新增检查项: 添加到校验检查项表
- 调整阈值: 修改校验阈值表
- 新增处理方式: 更新校验结果格式
